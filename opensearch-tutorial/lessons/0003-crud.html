CRUD în OpenSearch

Creează index, indexează documente, citește, actualizează, șterge

În această lecție folosim clientul low-level (friendsofcat/opensearch-client) pentru a înțelege exact ce se întâmplă la nivel HTTP. Ai nevoie de conexiunea configurată din Lecția 2.

1. Service de bază pentru CRUD

Extindem service-ul din Lecția 2 cu toate operațiile CRUD:

// app/Services/OpenSearchService.php
namespace App\Services;

use OpenSearch\Laravel\Client\ClientBuilderInterface;

class OpenSearchService
{
    protected $client;

    public function __construct(ClientBuilderInterface $builder)
    {
        $this->client = $builder->default();
    }
}

2. Creare index

Un index trebuie creat înainte de a indexa documente. Îl creezi o singură dată — de obicei printr-o comandă Artisan sau migration echivalent. 

public function createIndex(string $indexName): array
{
    return $this->client->indices()->create([
        'index' => $indexName,
        'body'  => [
            'settings' => [
                'number_of_shards'   => 1,   // un shard pentru dev
                'number_of_replicas' => 0,   // zero replici pentru dev
            ],
        ],
    ]);
}

// Verifici dacă indexul există deja
public function indexExists(string $indexName): bool
{
    return $this->client->indices()->exists([
        'index' => $indexName,
    ]);
}

// Ștergi indexul (ATENȚIE: șterge toate documentele!)
public function deleteIndex(string $indexName): array
{
    return $this->client->indices()->delete([
        'index' => $indexName,
    ]);
}
// Exemplu de utilizare
$search = app(OpenSearchService::class);

if (!$search->indexExists('articles')) {
    $search->createIndex('articles');
}

3. Indexare document (Create)

„A indexa un document" = a-l salva în OpenSearch. Poți specifica un ID sau lăsa OpenSearch să genereze unul automat. 

// Cu ID specificat manual (recomandat — îl legi de _id din MongoDB)
public function indexDocument(string $index, string $id, array $data): array
{
    return $this->client->index([
        'index' => $index,
        'id'    => $id,
        'body'  => $data,
    ]);
}

// Fără ID — OpenSearch generează automat un ID unic
public function indexDocumentAutoId(string $index, array $data): array
{
    return $this->client->index([
        'index' => $index,
        'body'  => $data,
    ]);
}
// Exemplu concret: indexezi un articol după ce l-ai salvat în MongoDB
class ArticleService
{
    public function create(array $data): void
    {
        // 1. Salvezi în MongoDB (sursa de adevăr)
        $article = Article::create($data);

        // 2. Indexezi în OpenSearch (pentru search)
        $this->search->indexDocument('articles', (string) $article->_id, [
            'titlu'       => $article->titlu,
            'continut'    => $article->continut,
            'autor'       => $article->autor,
            'publicat_la' => $article->created_at->toIso8601String(),
            'taguri'      => $article->taguri,
        ]);
    }
}
Pattern cheie: ID-ul din MongoDB = ID-ul din OpenSearch Folosind același ID în ambele baze de date, poți recupera detalii complete din MongoDB după ce ai obținut ID-urile din OpenSearch prin search. Simplu și eficient.

4. Citire document (Read)

// Citire după ID
public function getDocument(string $index, string $id): array
{
    return $this->client->get([
        'index' => $index,
        'id'    => $id,
    ]);
}

5. Actualizare document (Update)

OpenSearch are două abordări de update. Partial update (cu doc) actualizează doar câmpurile specificate — celelalte rămân neatinse. 

// Partial update — actualizează doar câmpurile trimise
public function updateDocument(string $index, string $id, array $fields): array
{
    return $this->client->update([
        'index' => $index,
        'id'    => $id,
        'body'  => [
            'doc' => $fields,   // doar câmpurile de actualizat
        ],
    ]);
}

// Re-indexare completă — înlocuiește tot documentul
public function replaceDocument(string $index, string $id, array $data): array
{
    return $this->client->index([   // același client->index() ca la creare
        'index' => $index,
        'id'    => $id,
        'body'  => $data,
    ]);
}
// Exemplu: utilizatorul a editat articolul
public function update(string $id, array $data): void
{
    // 1. Actualizezi în MongoDB
    Article::where('_id', $id)->update($data);

    // 2. Actualizezi în OpenSearch (doar câmpurile searchable)
    $this->search->updateDocument('articles', $id, [
        'titlu'    => $data['titlu']    ?? null,
        'continut' => $data['continut'] ?? null,
    ]);
}

6. Ștergere document (Delete)

// Ștergere după ID
public function deleteDocument(string $index, string $id): array
{
    return $this->client->delete([
        'index' => $index,
        'id'    => $id,
    ]);
}
// Exemplu: ștergi articolul din ambele baze de date
public function delete(string $id): void
{
    Article::where('_id', $id)->delete();          // MongoDB
    $this->search->deleteDocument('articles', $id); // OpenSearch
}

7. Indexare în masă (Bulk)

Dacă ai multe documente de indexat (migrare inițială, resync), bulk este mult mai rapid decât indexarea individuală. 

public function bulkIndex(string $index, array $documents): array
{
    $body = [];

    foreach ($documents as $doc) {
        // Fiecare document necesită 2 linii în bulk: action + data
        $body[] = ['index' => ['_index' => $index, '_id' => $doc['id']]];
        $body[] = $doc['data'];
    }

    return $this->client->bulk(['body' => $body]);
}

// Utilizare: resync toate articolele din MongoDB în OpenSearch
Article::chunk(500, function ($articles) use ($search) {
    $docs = $articles->map(fn($a) => [
        'id'   => (string) $a->_id,
        'data' => [
            'titlu'       => $a->titlu,
            'continut'    => $a->continut,
            'autor'       => $a->autor,
            'publicat_la' => $a->created_at->toIso8601String(),
        ],
    ])->toArray();

    $search->bulkIndex('articles', $docs);
});
