Lecția 06 · OpenSearch + Laravel

Filtre, sortare și paginație

bool query, term, range, sort, from/size — construiești query-uri complexe

În practică, nu faci niciodată doar un simplu match. Utilizatorul caută „laravel" și vrea doar articolele publicate, și sortate după dată. Aceasta este lecția în care piesele se leagă.

1. bool query — inima oricărui query complex

bool combină mai multe condiții cu logică AND/OR/NOT. Are 4 clauze, fiecare cu un rol diferit:

Clauza Logică Afectează scorul? Când o folosești
must AND — trebuie să fie adevărat Da Termeni de căutare principali
filter AND — trebuie să fie adevărat Nu (mai rapid!) Filtre: status, dată, categorie
should OR — îmbunătățește scorul dacă e adevărat Da Criterii opționale de relevanță
must_not NOT — trebuie să fie fals Nu Excludere: nu vreau status=draft
filter vs must — diferența practică must calculează un scor de relevanță — util pentru câmpuri de căutare.
filter nu calculează scor și e cached de OpenSearch — mai rapid pentru filtre fixe (status, categorie, dată). Folosește filter pentru orice condiție care nu influențează relevanța.

2. Exemple practice de bool query

Scenariu A: căutare + filtru simplu

Caută „laravel", dar returnează doar articolele publicate:

public function searchPublished(string $query): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'query' => [
                'bool' => [
                    'must' => [
                        ['multi_match' => [
                            'query'  => $query,
                            'fields' => ['titlu^3', 'continut'],
                        ]],
                    ],
                    'filter' => [
                        ['term' => ['status' => 'published']],
                    ],
                ],
            ],
        ],
    ]);

    return $this->extractHits($response);
}

Scenariu B: filtre multiple + range pe dată

Articole publicate, dintr-o anumită categorie, din ultimul an:

public function searchWithFilters(string $query, string $tag, string $autor = null): array
{
    $filters = [
        ['term'  => ['status' => 'published']],
        ['term'  => ['taguri' => $tag]],          // filtru exact pe keyword
        ['range' => ['publicat_la' => [
            'gte' => 'now-1y/d',   // de acum 1 an
            'lte' => 'now/d',      // până azi
        ]]],
    ];

    // Adaugă filtru opțional pe autor
    if ($autor) {
        $filters[] = ['term' => ['autor.keyword' => $autor]];
    }

    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'query' => [
                'bool' => [
                    'must'   => [['multi_match' => [
                        'query'  => $query,
                        'fields' => ['titlu^3', 'continut'],
                    ]]],
                    'filter' => $filters,
                ],
            ],
        ],
    ]);

    return $this->extractHits($response);
}

3. Filtre atomice: term, terms, range, exists

// term — egalitate exactă (pentru keyword, boolean, integer)
['term' => ['status' => 'published']]
['term' => ['este_publicat' => true]]
['term' => ['autor.keyword' => 'Paul Maxineanu']]

// terms — oricare din valorile din array (echivalent IN() din SQL)
['terms' => ['status' => ['published', 'featured']]]
['terms' => ['taguri'  => ['php', 'laravel']]]

// range — interval numeric sau de date
['range' => ['vizualizari' => ['gte' => 100]]]                   // >= 100
['range' => ['pret'         => ['gte' => 10, 'lte' => 50]]]      // intre 10 si 50
['range' => ['publicat_la'  => ['gte' => '2024-01-01']]]        // de la 2024
['range' => ['publicat_la'  => ['gte' => 'now-30d/d']]]        // ultimele 30 zile

// exists — câmpul are valoare (nu e null / lipsă)
['exists' => ['field' => 'publicat_la']]

4. Sortare

public function searchSorted(string $query, string $sortField = '_score', string $order = 'desc'): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'query' => ['bool' => [
                'must'   => [['multi_match' => [
                    'query'  => $query,
                    'fields' => ['titlu^3', 'continut'],
                ]]],
                'filter' => [['term' => ['status' => 'published']]],
            ]],

            'sort' => [
                // Sortare primară după câmpul ales
                [$sortField => ['order' => $order]],
                // Sortare secundară după scor (dacă sortField este dată)
                ['_score'  => ['order' => 'desc']],
            ],
        ],
    ]);

    return $this->extractHits($response);
}

// Utilizare:
// $search->searchSorted('laravel', '_score', 'desc')         — după relevanță
// $search->searchSorted('laravel', 'publicat_la', 'desc')    — cele mai recente
// $search->searchSorted('laravel', 'vizualizari', 'desc')    — cele mai citite
Atenție la sortare pe câmpuri text Nu poți sorta după un câmp de tip text. Dacă vrei să sortezi după autor, folosești sub-câmpul autor.keyword (definit în mapping în Lecția 5).

5. Paginație

OpenSearch folosește from și size — exact ca LIMIT și OFFSET în SQL. 

public function searchPaginated(string $query, int $page = 1, int $perPage = 15): array
{
    $from = ($page - 1) * $perPage;

    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'size'  => $perPage,
            'from'  => $from,
            'query' => ['bool' => [
                'must'   => [['multi_match' => [
                    'query'  => $query,
                    'fields' => ['titlu^3', 'continut'],
                ]]],
                'filter' => [['term' => ['status' => 'published']]],
            ]],
            'sort'  => [['_score' => ['order' => 'desc']]],
        ],
    ]);

    $total = $response['hits']['total']['value'];

    return [
        'total'        => $total,
        'per_page'     => $perPage,
        'current_page' => $page,
        'last_page'    => (int) ceil($total / $perPage),
        'results'      => collect($response['hits']['hits'])
            ->map(fn($h) => array_merge(
                ['_id' => $h['_id'], '_score' => $h['_score']],
                $h['_source']
            ))->toArray(),
    ];
}

// Utilizare în controller:
$results = $search->searchPaginated('laravel', page: 2, perPage: 10);
// $results['total']        = 42
// $results['last_page']    = 5
// $results['current_page'] = 2
// $results['results']      = [...]
Limita de paginație deep from + size nu poate depăși 10.000 (implicit). Pentru paginație adâncă (pagina 500+), folosești search_after — subiect avansat. Pentru cazuri normale (e-commerce, blog), from/size este perfect.

6. Query complet — căutare reală din proiectul tău

// Metodă completă, parametrizabilă — acoperă 90% din cazuri
public function searchArticles(array $params): array
{
    $query   = $params['q']       ?? '';
    $tag     = $params['tag']     ?? null;
    $autor   = $params['autor']   ?? null;
    $sortBy  = $params['sort']    ?? '_score';
    $page    = $params['page']    ?? 1;
    $perPage = $params['per_page'] ?? 15;

    // Filtre fixe — întotdeauna aplicate
    $filters = [
        ['term' => ['status' => 'published']],
    ];

    if ($tag)   $filters[] = ['term' => ['taguri'       => $tag]];
    if ($autor) $filters[] = ['term' => ['autor.keyword' => $autor]];

    // Query principal
    $must = $query
        ? [['multi_match' => [
            'query'     => $query,
            'fields'    => ['titlu^3', 'continut', 'taguri^2'],
            'fuzziness' => 'AUTO',
          ]]]
        : [['match_all' => (object)[]]];   // dacă nu e query, returnează toate

    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'size'  => $perPage,
            'from'  => ($page - 1) * $perPage,
            'query' => ['bool' => ['must' => $must, 'filter' => $filters]],
            'sort'  => [[$sortBy => ['order' => 'desc']]],
        ],
    ]);

    $total = $response['hits']['total']['value'];
    return [
        'total'        => $total,
        'current_page' => $page,
        'last_page'    => (int) ceil($total / $perPage),
        'results'      => collect($response['hits']['hits'])
            ->map(fn($h) => array_merge(['_id' => $h['_id']], $h['_source']))->toArray(),
    ];
}
