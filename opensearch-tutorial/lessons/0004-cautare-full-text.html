Query DSL — match, multi_match, match_phrase și structura răspunsului

Aceasta este lecția centrală. Căutarea este motivul pentru care există OpenSearch. Vei înțelege cum funcționează Query DSL — limbajul de interogare JSON al OpenSearch — și vei scrie primele căutări reale din Laravel.

1. Ce este Query DSL

Query DSL (Domain Specific Language) este formatul JSON prin care descrii ce vrei să cauți. Îl trimiți în câmpul body al cererii de search. 

// Structura generală a unui request de search
$client->search([
    'index' => 'articles',
    'body'  => [
        // --- Ce returnezi ---
        'size' => 10,          // câte rezultate (implicit 10)
        'from' => 0,           // offset pentru paginare

        // --- Cum cauți ---
        'query' => [
            // ... query DSL vine aici
        ],

        // --- Cum sortezi ---
        'sort' => [
            // ... sortare vine aici
        ],
    ],
]);

2. match — căutare într-un singur câmp

match este interogarea de bază pentru text. OpenSearch analizează textul de căutat: îl tokenizează, îl normalizează (lowercase, diacritice), și caută documente care conțin acei termeni. 

// Caută articole care conțin „laravel" în titlu
public function searchByTitle(string $query): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'size'  => 10,
            'query' => [
                'match' => [
                    'titlu' => [
                        'query'     => $query,
                        'operator'  => 'or',    // 'or' (implicit) sau 'and'
                        'fuzziness' => 'AUTO', // toleranță typo-uri
                    ],
                ],
            ],
        ],
    ]);

    return $this->extractHits($response);
}
operator: 'or' vs 'and'
  • or (implicit): „laravel tutorial" → documente care conțin „laravel" SAU „tutorial" → mai multe rezultate
  • and: „laravel tutorial" → documente care conțin „laravel" ȘI „tutorial" → rezultate mai precise
fuzziness: 'AUTO' permite typo-uri: „laravell" va găsi tot „laravel".

3. multi_match — căutare în mai multe câmpuri

Cel mai frecvent folosit în practică: caută același termen în mai multe câmpuri deodată. Poți boosta un câmp (titlul contează mai mult decât conținutul). 

// Caută în titlu, continut și taguri — titlul are prioritate
public function search(string $query): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'size'  => 10,
            'query' => [
                'multi_match' => [
                    'query'  => $query,
                    'fields' => [
                        'titlu^3',     // ^3 = de 3× mai important
                        'continut',    // relevanță normală
                        'taguri^2',    // de 2× mai important
                        'autor',       // relevanță normală
                    ],
                    'type'       => 'best_fields',  // scorul celui mai bun câmp
                    'fuzziness'  => 'AUTO',
                ],
            ],
        ],
    ]);

    return $this->extractHits($response);
}
Tipuri de multi_match
  • best_fields (implicit): scorul vine de la câmpul cu cea mai mare relevanță
  • most_fields: cumulează scoruri din toate câmpurile — bun pentru text duplicat în variante
  • cross_fields: tratează câmpurile ca un singur câmp — bun pentru nume/prenume
  • phrase: match_phrase în toate câmpurile — bun pentru expresii exacte

4. match_phrase — expresie exactă

Caută expresia exact în ordinea cuvintelor. „laravel tutorial" va găsi doar documente cu aceste cuvinte unul lângă altul. 

public function searchPhrase(string $phrase): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'query' => [
                'match_phrase' => [
                    'continut' => $phrase,
                ],
            ],
        ],
    ]);

    return $this->extractHits($response);
}

5. Structura răspunsului — cum citești rezultatele

Orice query de search returnează același format JSON. Trebuie să știi unde să te uiți. 

// Răspunsul complet de la OpenSearch:
[
    'took'      => 5,         // milisecunde cât a durat
    'timed_out' => false,
    'hits' => [
        'total' => [
            'value'    => 42,       // numărul TOTAL de documente găsite
            'relation' => 'eq',
        ],
        'max_score' => 1.8734,    // scorul celui mai relevant document
        'hits' => [               // lista de documente returnate (max 'size')
            [
                '_index'  => 'articles',
                '_id'     => '64f1a2b3...',
                '_score'  => 1.8734,   // relevanța acestui document
                '_source' => [          // datele tale
                    'titlu'    => 'Introducere în Laravel',
                    'autor'    => 'Paul',
                    'continut' => '...',
                ],
            ],
            // ... restul documentelor
        ],
    ],
]
// Metodă helper pentru a extrage datele utile
protected function extractHits(array $response): array
{
    return [
        'total'   => $response['hits']['total']['value'],
        'results' => collect($response['hits']['hits'])
            ->map(fn($hit) => array_merge(
                ['_id' => $hit['_id'], '_score' => $hit['_score']],
                $hit['_source']
            ))
            ->toArray(),
    ];
}

// Utilizare în controller:
$results = $search->search('laravel tutorial');
// $results['total']   = 42
// $results['results'] = [['_id' => '...', 'titlu' => '...', '_score' => 1.8], ...]

6. match_all — returnează toate documentele

// Returnează toate documentele (util pentru listare)
public function getAll(int $size = 10, int $from = 0): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'size'  => $size,
            'from'  => $from,
            'query' => ['match_all' => (object)[]],  // obiect gol
        ],
    ]);

    return $this->extractHits($response);
}
