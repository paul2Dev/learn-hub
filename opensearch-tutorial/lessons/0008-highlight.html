Highlight — evidențierea termenilor căutați

Marchezi în rezultate exact unde a găsit OpenSearch ce a căutat utilizatorul

1. Ce este highlight și de ce îl vrei

Fără highlight, utilizatorul vede o listă de titluri și fragmente de text și nu înțelege de ce a apărut acel rezultat. Cu highlight, OpenSearch îți returnează fragmentele relevante cu termenii marcați.

Ce face OpenSearch de fapt OpenSearch caută fragmentele din document care conțin termenii căutați, le extrage și le întoarce cu tag-urile HTML pe care le specifici tu (<em>, <mark>, sau orice altceva). Tu controlezi complet cum arată marcajul.

2. Adaugi highlight la orice query de search

Highlight se adaugă ca o cheie paralelă cu query în body-ul cererii. Funcționează pe orice tip de query — match, multi_match, bool. 

public function searchWithHighlight(string $query): array
{
    $response = $this->client->search([
        'index' => 'articles',
        'body'  => [
            'size'  => 10,

            'query' => [
                'bool' => [
                    'must'   => [['multi_match' => [
                        'query'  => $query,
                        'fields' => ['titlu^3', 'continut'],
                    ]]],
                    'filter' => [['term' => ['status' => 'published']]],
                ],
            ],

            // Highlight — adăugat lângă query, nu înăuntrul lui
            'highlight' => [
                'pre_tags'  => ['<em class="highlight">'],  // tag de deschidere
                'post_tags' => ['</em>'],                   // tag de închidere
                'fields'    => [
                    'titlu'    => (object)[],   // highlight în titlu
                    'continut' => [
                        'fragment_size'       => 150,  // lungimea fragmentului (caractere)
                        'number_of_fragments' => 3,    // max fragmente returnate
                    ],
                ],
            ],
        ],
    ]);

    return $this->extractHighlightedHits($response);
}

3. Structura răspunsului cu highlight

Fiecare document din hits.hits primește o cheie suplimentară highlight cu fragmentele marcate. 

// Răspuns de la OpenSearch pentru un hit:
[
    '_index'  => 'articles',
    '_id'     => '64f1a2b3...',
    '_score'  => 2.14,
    '_source' => [
        'titlu'    => 'Introducere în Laravel',
        'continut' => 'Laravel este un framework PHP modern...',
        'autor'    => 'Paul',
    ],
    'highlight' => [               // cheie nouă cu fragmentele marcate
        'titlu'    => [
            'Introducere în <em class="highlight">Laravel</em>',
        ],
        'continut' => [
            '...<em class="highlight">Laravel</em> este un framework PHP modern...',
            '...primul pas este <em class="highlight">instalarea</em> Composer...',
        ],
    ],
]
highlight vs _source _source.titlu conține titlul original, nemodificat.
highlight.titlu conține titlul cu tag-uri HTML injectate — folosit pentru afișare.
Dacă nu există highlight pentru un câmp (termenul nu a fost găsit acolo), cheia lipsește din highlight.

4. Helper pentru extragerea rezultatelor cu highlight

protected function extractHighlightedHits(array $response): array
{
    return [
        'total'   => $response['hits']['total']['value'],
        'results' => collect($response['hits']['hits'])
            ->map(fn($hit) => [
                '_id'    => $hit['_id'],
                '_score' => $hit['_score'],

                // Date originale din _source
                ...$hit['_source'],

                // Titlu: cu highlight dacă există, altfel original
                'titlu_display' => $hit['highlight']['titlu'][0]
                    ?? $hit['_source']['titlu'],

                // Fragmente din conținut: array de fragmente marcate
                'fragmente' => $hit['highlight']['continut']
                    ?? [substr($hit['_source']['continut'] ?? '', 0, 150) . '...'],
            ])
            ->toArray(),
    ];
}

5. Afișare în Blade — HTML safe

Fragmentele de highlight conțin HTML (<em>). Trebuie să folosești {!! !!} în Blade (nu {{ }}) pentru a nu escapa tag-urile. 

{{-- resources/views/search/results.blade.php --}}
@foreach ($results['results'] as $item)
<article class="search-result">

    <h3>
        <a href="/articles/{{ $item['slug'] }}">
            {!! $item['titlu_display'] !!}   {{-- HTML marcat, safe din OpenSearch --}}
        </a>
    </h3>

    @foreach ($item['fragmente'] as $fragment)
        <p class="fragment">...{!! $fragment !!}...</p>
    @endforeach

    <small>{{ $item['autor'] }} · {{ $item['publicat_la'] }}</small>

</article>
@endforeach
/* CSS pentru tag-ul de highlight */
em.highlight {
    font-style: normal;
    background: #fff3cd;
    color: #856404;
    padding: 1px 3px;
    border-radius: 2px;
    font-weight: 700;
}
Securitate — XSS Fragmentele cu highlight vin de la OpenSearch, nu direct de la utilizator, deci conținutul este din documentele tale indexate — safe pentru {!! !!}. Nu concatena niciodată input-ul utilizatorului direct în HTML fără escape.

6. Parametri utili de highlight

'highlight' => [
    // Tag-uri personalizate (poți folosi orice HTML)
    'pre_tags'  => ['<mark>'],
    'post_tags' => ['</mark>'],

    // Setări globale (aplicate tuturor câmpurilor dacă nu specifici per câmp)
    'fragment_size'       => 200,  // lungime fragment în caractere
    'number_of_fragments' => 2,    // max fragmente per câmp

    'fields' => [
        'titlu' => [
            'number_of_fragments' => 0,   // 0 = returnează titlul complet, nu fragment
        ],
        'continut' => [
            'fragment_size'       => 150,
            'number_of_fragments' => 3,
            'no_match_size'       => 100,  // dacă nu e match, returnează primele N chars
        ],
    ],
],
Parametru Valoare implicită Ce face
pre_tags <em> Tag HTML înainte de termenul găsit
post_tags </em> Tag HTML după termenul găsit
fragment_size 100 Lungimea unui fragment în caractere
number_of_fragments 5 Câte fragmente per câmp. 0 = câmpul complet
no_match_size 0 (nimic) Dacă nu e match, returnează primele N caractere ca fallback
