Lecția 02 · OpenSearch + Laravel

Instalare și conectare din Laravel

Docker + pachet PHP + configurare .env — de la zero la prima conexiune

1. Pornești OpenSearch cu Docker

Cel mai simplu mod de a rula OpenSearch local este prin Docker. Creează un fișier docker-compose.yml în rădăcina proiectului Laravel: 

# docker-compose.yml
version: '3.8'
services:

  opensearch:
    image: opensearchproject/opensearch:2.17.0
    container_name: opensearch
    environment:
      - cluster.name=opensearch-cluster
      - node.name=opensearch-node1
      - discovery.type=single-node          # mod development, un singur nod
      - bootstrap.memory_lock=true
      - OPENSEARCH_JAVA_OPTS=-Xms512m -Xmx512m
      - DISABLE_SECURITY_PLUGIN=true        # dezactivat pentru dev local
      - OPENSEARCH_INITIAL_ADMIN_PASSWORD=Admin@1234  # necesar din v2.12+
    ulimits:
      memlock:
        soft: -1
        hard: -1
      nofile:
        soft: 65536
        hard: 65536
    ports:
      - "9200:9200"
      - "9600:9600"
    networks:
      - app-network

  opensearch-dashboards:
    image: opensearchproject/opensearch-dashboards:2.17.0
    container_name: opensearch-dashboards
    ports:
      - "5601:5601"
    environment:
      - OPENSEARCH_HOSTS=["http://opensearch:9200"]
      - DISABLE_SECURITY_DASHBOARDS_PLUGIN=true
    depends_on:
      - opensearch
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
Windows / WSL — mapare memorie Pe Windows cu WSL, dacă OpenSearch nu pornește, rulează în PowerShell ca administrator: wsl -d docker-desktop sysctl -w vm.max_map_count=262144

Pornire și verificare

# Pornești containerele
docker-compose up -d

# Verifici că OpenSearch răspunde (după ~30 secunde)
curl http://localhost:9200

# Răspuns așteptat:
{
  "name": "opensearch-node1",
  "cluster_name": "opensearch-cluster",
  "version": { "number": "2.17.0", ... }
}
OpenSearch Dashboards Interfața vizuală este disponibilă la http://localhost:5601. E opțională, dar utilă pentru a vedea indexurile și a testa query-uri interactiv.

2. Două opțiuni de integrare Laravel

Există două pachete principale. Alege în funcție de nevoie:

Pachet Abordare Când îl folosești
pdphilip/opensearch Eloquent ORM complet — modele, relații, query builder Recomandat — vrei să lucrezi ca cu Eloquent normal
friendsofcat/opensearch-client Clientul PHP oficial înregistrat ca serviciu Laravel Vrei control low-level total, sau query DSL pur

Vom folosi pdphilip/opensearch ca abordare principală (Lecțiile 7+), și friendsofcat/opensearch-client pentru a înțelege CRUD și query DSL pur (Lecțiile 3–6). Astfel înveți ambele straturi.

3. Instalare clientul low-level (Lecțiile 3–6)

composer require friendsofcat/opensearch-client
# Publică fișierul de configurare
php artisan vendor:publish --provider="OpenSearch\Laravel\Client\ServiceProvider"

Editează config/opensearch.client.php:

// config/opensearch.client.php
return [
    'default' => env('OPENSEARCH_CONNECTION', 'default'),
    'connections' => [
        'default' => [
            'hosts' => [
                env('OPENSEARCH_HOST', 'http://localhost:9200'),
            ],
            'httpClientOptions' => [
                'timeout' => 5,
            ],
        ],
    ],
];

Adaugă în .env:

OPENSEARCH_CONNECTION=default
OPENSEARCH_HOST=http://localhost:9200

4. Creează un Service Laravel pentru OpenSearch

Cel mai bun pattern în Laravel este să înregistrezi un Service class care primește clientul prin dependency injection. 

// app/Services/OpenSearchService.php
namespace App\Services;

use OpenSearch\Laravel\Client\ClientBuilderInterface;

class OpenSearchService
{
    protected $client;

    public function __construct(ClientBuilderInterface $builder)
    {
        $this->client = $builder->default();
    }

    public function ping(): bool
    {
        return $this->client->ping();
    }
}

Înregistrează-l în AppServiceProvider sau folosește-l direct prin DI:

// Injectare directă în Controller
use App\Services\OpenSearchService;

class ArticleController extends Controller
{
    public function __construct(
        private OpenSearchService $search
    ) {}

    public function index()
    {
        $ok = $this->search->ping();
        return response()->json(['connected' => $ok]);
    }
}

5. Verifici conexiunea cu Artisan

Cel mai simplu test — o comandă Artisan:

// app/Console/Commands/OpenSearchPing.php
namespace App\Console\Commands;

use Illuminate\Console\Command;
use OpenSearch\Laravel\Client\ClientBuilderInterface;

class OpenSearchPing extends Command
{
    protected $signature   = 'opensearch:ping';
    protected $description = 'Testează conexiunea la OpenSearch';

    public function handle(ClientBuilderInterface $builder): int
    {
        $client = $builder->default();

        if ($client->ping()) {
            $this->info('✓ OpenSearch este disponibil!');
            $info = $client->info();
            $this->line('Versiune: ' . $info['version']['number']);
            return 0;
        }

        $this->error('✗ Nu pot contacta OpenSearch!');
        return 1;
    }
}
php artisan opensearch:ping
# ✓ OpenSearch este disponibil!
# Versiune: 2.17.0
