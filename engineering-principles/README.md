# Engineering Principles — Aplicate în Dezvoltarea Asistată de AI

Un curs practic pentru dezvoltatorii web care vor să-și ascută judecata inginerească în era AI. Pe măsură ce AI preia din ce în ce mai multă implementare, valoarea ta se mută spre design, arhitectură și decizii de calitate pe care AI nu le poate lua în locul tău.

## Structura cursului

| # | Lecție | Subiecte |
|---|--------|----------|
| 01 | Mentalitatea Arhitectului | inginer vs. coder, rolul AI, noua ta pârghie |
| 02 | Principiile SOLID | SRP, OCP, LSP, ISP, DIP cu exemple web |
| 03 | YAGNI, KISS, DRY | principii ale simplității, capcane ale supraingineriei |
| 04 | Gândire Sistemică | descompunere, limite, cuplaj, coeziune |
| 05 | Principii de Design API | contracte, fundamente REST, versionare |
| 06 | Ingineria Cerințelor | traducerea nevoilor de business în specificații tehnice |
| 07 | Managementul Datoriei Tehnice | recunoaștere, cuantificare și rambursare |
| 08 | Strategia de Testare | testare bazată pe risc, ce nu poate testa AI pentru tine |
| 09 | Securitate prin Design | mentalitate OWASP, modelare amenințări, apărare în profunzime |
| 10 | Code Review în Era AI | evaluarea output-ului AI, rolul tău ca poartă de calitate |

## Cum se folosește

Deschide `index.html` direct în browser — nu este nevoie de server.

```bash
# Windows
start index.html

# macOS / Linux
open index.html
```

Fiecare lecție conține:
- Explicație conceptuală cu diagrame
- Exemple complete, gata de copiat
- Quiz interactiv cu feedback instant

## Tehnologii acoperite

- **Principii de Inginerie** — SOLID, YAGNI, KISS, DRY, OWASP
- **Flux de Lucru Asistat de AI** — prompting cu intenție, revizuirea output-ului, menținerea porților de calitate
- **Design de Sisteme** — descompunere, contracte API, cuplaj, coeziune

## Flux tipic de lucru

```
problemă de business → specificație cerințe → design sistem → implementare (AI) → revizuire & poartă de calitate
```
