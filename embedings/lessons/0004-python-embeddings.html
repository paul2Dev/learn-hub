Python: embeddings cu OpenAI și Sentence Transformers

Cod complet, rulabil imediat: calculezi embeddings, măsori similaritatea și faci un prim semantic search.

Setup mediu Python

Ai nevoie de Python 3.9+ instalat. Creează un mediu virtual și instalează dependențele: 

# Creează un folder pentru proiect
mkdir embeddings-lab && cd embeddings-lab

# Creează mediu virtual
python -m venv venv

# Activează (Windows)
venv\Scripts\activate

# Activează (macOS/Linux)
source venv/bin/activate

# Instalează librăriile
pip install openai sentence-transformers numpy scipy
Dacă nu vrei să plătești API
Sari peste pip install openai și folosește doar sentence-transformers — modelele locale sunt complet gratuite. Lecția acoperă ambele abordări.

Varianta 1: OpenAI API

Obții un API key de pe platform.openai.com, pui în variabila de mediu OPENAI_API_KEY și rulezi: 

import os
from openai import OpenAI
import numpy as np

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

def get_embedding(text: str, model: str = "text-embedding-3-small") -> list[float]:
    text = text.replace("\n", " ")
    response = client.embeddings.create(input=[text], model=model)
    return response.data[0].embedding

# Test
embedding = get_embedding("Pisica mea doarme pe canapea.")
print(f"Dimensiuni vector: {len(embedding)}")   # 1536
print(f"Primele 5 valori: {embedding[:5]}")

Funcția get_embedding este tot ce ai nevoie. Restul este aplicarea ei.

Varianta 2: Sentence Transformers (local, gratuit)

Prima rulare descarcă modelul (~90MB pentru MiniLM). Ulterior este cached local. 

from sentence_transformers import SentenceTransformer
import numpy as np

# Modelul multilingv — funcționează și în română
model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')

def get_embedding_local(text: str) -> np.ndarray:
    return model.encode(text)

# Test
embedding = get_embedding_local("Pisica mea doarme pe canapea.")
print(f"Dimensiuni vector: {len(embedding)}")   # 384
print(f"Tip: {embedding.dtype}")                # float32
Diferență față de OpenAI
sentence-transformers returnează un numpy.ndarray, nu o listă Python. Asta este convenabil — numpy are calculul cosine similarity inclus. OpenAI returnează o list[float] — trebuie să o convertești cu np.array().

Calculul similarității

from scipy.spatial.distance import cosine

def cosine_similarity(vec_a, vec_b) -> float:
    # scipy.cosine returnează DISTANȚA (0=identic, 2=opus)
    # convertiți la similarity (1=identic, -1=opus)
    return 1 - cosine(vec_a, vec_b)

# Exemplu complet cu modele locale
sentences = [
    "Pisica mea doarme pe canapea.",
    "Felina mea se odihnește pe fotoliu.",
    "Câinele aleargă în parc.",
    "Factura de electricitate a crescut.",
]

embeddings = [get_embedding_local(s) for s in sentences]

query_idx = 0  # "Pisica mea doarme pe canapea."
query_emb = embeddings[query_idx]

print(f"Query: {sentences[query_idx]}\n")
for i, (sent, emb) in enumerate(zip(sentences, embeddings)):
    if i == query_idx:
        continue
    sim = cosine_similarity(query_emb, emb)
    print(f"  [{sim:.4f}] {sent}")

Output așteptat:

Query: Pisica mea doarme pe canapea.

  [0.8921] Felina mea se odihnește pe fotoliu.
  [0.6234] Câinele aleargă în parc.
  [0.1823] Factura de electricitate a crescut.

Exact ordinea așteptată: parafraza este cea mai similară, subiect din același domeniu e la mijloc, subiect complet diferit are similarity mică.

Semantic Search în ~20 de linii

Ai un corpus de documente, vectorizezi totul, apoi la fiecare query returnezi top-K rezultate: 

from sentence_transformers import SentenceTransformer
from scipy.spatial.distance import cosine

model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')

# Corpus de documente (în producție: citite din DB)
corpus = [
    "Cum resetez parola contului meu?",
    "Ghid de instalare a aplicației mobile.",
    "Politica de retur și refund pentru comenzi online.",
    "Cum schimb adresa de livrare după plasarea comenzii?",
    "Suport pentru plăți cu cardul în aplicație.",
    "Erori frecvente și soluțiile lor.",
]

# Vectorizăm corpus-ul O SINGURĂ DATĂ
corpus_embeddings = model.encode(corpus)

def semantic_search(query: str, top_k: int = 3) -> list[tuple[float, str]]:
    query_emb = model.encode(query)
    scores = [1 - cosine(query_emb, doc_emb) for doc_emb in corpus_embeddings]
    ranked = sorted(zip(scores, corpus), reverse=True)
    return ranked[:top_k]

# Test
results = semantic_search("vreau să returnez un produs")
for score, doc in results:
    print(f"  [{score:.3f}] {doc}")

Output:

  [0.812] Politica de retur și refund pentru comenzi online.
  [0.634] Cum schimb adresa de livrare după plasarea comenzii?
  [0.521] Cum resetez parola contului meu?
În producție
Nu stochezi embeddings în memorie și nu calculezi distanța față de fiecare document la fiecare query. Folosești o vector database (Lecția 6) care face asta eficient la scară de milioane de documente.
Exercițiu — Rulează și extinde
  1. Copiază codul semantic search de mai sus într-un fișier search.py și rulează-l.
  2. Adaugă 5 documente proprii în corpus (din domeniul tău: PHP, Laravel, web).
  3. Testează cu 3 query-uri diferite — observă că găsește documentul potrivit chiar și cu alte cuvinte.
  4. Modifică top_k=1 — observă că returnează mereu cel mai relevant document.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Câte dimensiuni are un vector produs de text-embedding-3-small (OpenAI)?
Corect. text-embedding-3-small produce vectori de 1536 dimensiuni. Modelul local paraphrase-multilingual-MiniLM-L12-v2 produce 384 dimensiuni — mai compact, gratuit, dar ușor mai puțin precis.
Întrebarea 2 din 5 Ce returnează scipy.spatial.distance.cosine(a, b)?
Corect. scipy.cosine returnează distanța (0 = identic, 2 = opus), nu similarity. Pentru a obține similarity, calculezi 1 - cosine(a, b). Confuzia asta este o sursă frecventă de bug-uri.
Întrebarea 3 din 5 Când ar trebui să vectorizezi corpus-ul de documente pentru semantic search?
Corect. Embedding-urile documentelor nu se schimbă dacă textul nu se schimbă. Le calculezi o dată și le stochezi. Re-calculezi doar când adaugi sau modifici documente. Calculul la fiecare query ar fi extrem de lent și costisitor.
Întrebarea 4 din 5 Ce tip de date returnează model.encode(text) din sentence-transformers?
Corect. model.encode() returnează numpy.ndarray de tip float32. Asta permite să folosești direct funcțiile NumPy/SciPy fără conversie. OpenAI returnează list[float] — trebuie np.array() pentru compatibilitate.
Întrebarea 5 din 5 De ce semantic search în memorie (lista de vectori + loop) nu este potrivit pentru producție cu milioane de documente?
Corect. Un loop peste 1M vectori la fiecare query durează secunde. Vector databases (Lecția 6) folosesc algoritmi aproximativi (HNSW, IVF) care reduc complexitatea la O(log n) sau sub-lineară — returnând top-K în milisecunde.
