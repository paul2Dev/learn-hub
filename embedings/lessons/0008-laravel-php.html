Strategia 1: Client OpenAI în Laravel

composer require openai-php/client
// .env
OPENAI_API_KEY=sk-proj-...

// config/services.php
'openai' => [
    'api_key' => env('OPENAI_API_KEY'),
],
// app/Services/EmbeddingService.php
namespace App\Services;

use OpenAI;

class EmbeddingService
{
    private \OpenAI\Client $client;

    public function __construct()
    {
        $this->client = OpenAI::client(config('services.openai.api_key'));
    }

    /**
     * Returnează vectorul embedding pentru un text.
     * @return float[]
     */
    public function embed(string $text, string $model = 'text-embedding-3-small'): array
    {
        $response = $this->client->embeddings()->create([
            'model' => $model,
            'input' => str_replace("\n", ' ', $text),
        ]);

        return $response->embeddings[0]->embedding;
    }

    /**
     * Vectorizează mai multe texte simultan (mai eficient API).
     * @return float[][]
     */
    public function embedBatch(array $texts): array
    {
        $response = $this->client->embeddings()->create([
            'model' => 'text-embedding-3-small',
            'input' => $texts,
        ]);

        return array_map(fn($item) => $item->embedding, $response->embeddings);
    }
}
Batch embedding
Dacă ai 100 de documente de indexat, trimite-le pe toate în același request cu embedBatch. API-ul OpenAI suportă array de input-uri — este mult mai rapid și mai ieftin decât 100 de request-uri separate.

Stocarea vectorilor cu pgvector

Dacă folosești PostgreSQL, pgvector este cel mai simplu: nu adaugi niciun serviciu nou. 

-- Activează extensia (o singură dată, ca root)
CREATE EXTENSION IF NOT EXISTS vector;

-- Sau în migration Laravel
public function up(): void
{
    DB::statement('CREATE EXTENSION IF NOT EXISTS vector');
}
// Migration
Schema::create('documents', function (Blueprint $table) {
    $table->id();
    $table->text('content');
    $table->string('category')->nullable();
    $table->timestamps();
});

// Coloana vector se adaugă cu raw SQL (Blueprint nu o suportă nativ)
DB::statement('ALTER TABLE documents ADD COLUMN embedding vector(1536)');
DB::statement('CREATE INDEX ON documents USING hnsw (embedding vector_cosine_ops)');
// Model Eloquent
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Document extends Model
{
    protected $fillable = ['content', 'category', 'embedding'];

    // Cast custom pentru array de float-uri
    protected $casts = [
        'embedding' => 'array',
    ];

    /**
     * Salvează un document cu embedding calculat automat.
     */
    public static function createWithEmbedding(string $content, string $category = null): self
    {
        $service = app(EmbeddingService::class);
        $embedding = $service->embed($content);

        $doc = new self();
        $doc->content = $content;
        $doc->category = $category;
        // pgvector acceptă format '[0.1, 0.2, ...]'
        $doc->embedding = $embedding;
        $doc->save();

        // Updatează vectorul cu sintaxa pgvector
        $vector = '[' . implode(',', $embedding) . ']';
        \DB::statement(
            "UPDATE documents SET embedding = ?::vector WHERE id = ?",
            [$vector, $doc->id]
        );

        return $doc;
    }
}

Semantic Search cu query raw în Eloquent

// app/Services/SearchService.php
namespace App\Services;

use App\Models\Document;
use Illuminate\Support\Facades\DB;

class SearchService
{
    public function __construct(private EmbeddingService $embedder) {}

    /**
     * Semantic search: returnează documentele cele mai similare cu query-ul.
     */
    public function search(string $query, int $limit = 5, ?string $category = null): \Illuminate\Support\Collection
    {
        $queryEmbedding = $this->embedder->embed($query);
        $vector = '[' . implode(',', $queryEmbedding) . ']';

        $sql = "
            SELECT id, content, category,
                   1 - (embedding <=> ?::vector) AS similarity
            FROM documents
            " . ($category ? "WHERE category = ?" : "") . "
            ORDER BY embedding <=> ?::vector
            LIMIT ?
        ";

        $bindings = $category
            ? [$vector, $category, $vector, $limit]
            : [$vector, $vector, $limit];

        return collect(DB::select($sql, $bindings));
    }
}

// Utilizare în Controller
class SearchController extends Controller
{
    public function search(Request $request, SearchService $search)
    {
        $results = $search->search(
            query: $request->input('q'),
            limit: 5,
            category: $request->input('category'),
        );

        return response()->json($results);
    }
}
Operatorul <=> în pgvector
<=> este distanța cosinus — valori mai mici = mai similar. Expresia 1 - (embedding <=> vector) o convertește la similarity (0–1). Există și <-> (L2/euclidiană) și <#> (inner product negativ).

RAG simplu în Laravel

// app/Services/RagService.php
namespace App\Services;

class RagService
{
    public function __construct(
        private SearchService $search,
        private \OpenAI\Client $openai,
    ) {}

    public function ask(string $question, ?string $category = null): string
    {
        // 1. Retrieval — găsim contextul relevant
        $results = $this->search->search($question, limit: 3, category: $category);
        $context = $results->pluck('content')->implode("\n\n");

        if ($results->isEmpty()) {
            return 'Nu am găsit informații relevante pentru această întrebare.';
        }

        // 2. Construim promptul
        $prompt = "Ești un asistent de customer support. Răspunde DOAR pe baza contextului de mai jos.
Dacă informația nu este prezentă în context, spune că nu știi.

Context:
{$context}

Întrebarea clientului: {$question}

Răspuns:";

        // 3. LLM generează răspunsul
        $response = $this->openai->chat()->create([
            'model'       => 'gpt-4o-mini',
            'messages'    => [['role' => 'user', 'content' => $prompt]],
            'temperature' => 0,
            'max_tokens'  => 500,
        ]);

        return $response->choices[0]->message->content;
    }
}

// Route
Route::post('/support/ask', function (Request $request, RagService $rag) {
    return response()->json([
        'answer' => $rag->ask($request->input('question')),
    ]);
});

Bonus: Microserviciu Python pentru modele locale

Dacă vrei modele locale (gratuite, fără API key), creează un microserviciu Python cu FastAPI: 

# embed_service.py
from fastapi import FastAPI
from pydantic import BaseModel
from sentence_transformers import SentenceTransformer

app = FastAPI()
model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')

class EmbedRequest(BaseModel):
    text: str | list[str]

@app.post("/embed")
def embed(req: EmbedRequest):
    embeddings = model.encode(req.text)
    if isinstance(req.text, str):
        return {"embedding": embeddings.tolist()}
    return {"embeddings": [e.tolist() for e in embeddings]}

# uvicorn embed_service:app --port 8001
// Laravel — apelează microserviciul
$response = Http::post('http://localhost:8001/embed', [
    'text' => $content,
]);

$embedding = $response->json('embedding'); // array PHP de float-uri

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Ce face metoda embedBatch(array $texts) față de embed(string $text) în termeni de eficiență?
Corect. API-ul OpenAI acceptă un array de input-uri și returnează un array de embeddings. Un singur HTTP request pentru 100 texte este mult mai eficient decât 100 request-uri separate — reduce latența de rețea și costul.
Întrebarea 2 din 5 Ce operează <=> în PostgreSQL cu extensia pgvector?
Corect. <=> este distanța cosinus (0 = identic, 2 = opus). Nu similarity. Sortezi ASC pentru cei mai apropiați. Conversia la similarity: 1 - (embedding <=> query).
Întrebarea 3 din 5 Care este avantajul microserviciului Python față de API-ul OpenAI pentru calculul embeddings în Laravel?
Corect. Un microserviciu Python cu FastAPI + sentence-transformers rulează local — zero cost API, zero date trimise în cloud (important pentru GDPR sau date confidențiale). Trade-off: adaugi complexitate de infrastructură.
Întrebarea 4 din 5 De ce indexul USING hnsw (embedding vector_cosine_ops) este important în pgvector?
Corect. HNSW (Hierarchical Navigable Small World) este un algoritm ANN care reduce complexitatea căutării. Fără index, Postgres face un scan complet (O(n)) la fiecare query — imposibil la mii de documente. Cu HNSW, sub-milisecundă chiar și la milioane.
Întrebarea 5 din 5 Ai o aplicație Laravel cu PostgreSQL și vrei să adaugi semantic search la 10.000 documente. Care este abordarea cu cel mai mic overhead de infrastructură?
Corect. pgvector extinde Postgres-ul pe care îl ai deja. Nu adaugi niciun serviciu nou, nicio configurație Docker, niciun cont cloud. La 10.000 documente, Postgres + HNSW index este mai mult decât suficient — Postgres gestionează milioane de vectori.
