Ce sunt Embeddings?

Cum transformi text, imagini sau orice altceva în numere cu sens — și de ce asta schimbă totul.

Problema: computerele nu înțeleg cuvintele

Un computer poate compara "câine" și "pisică" și să spună că sunt diferite — dar nu poate ști că sunt mai similare între ele decât cu "avion". Compararea de string-uri este binară: egale sau diferite. Nu există nuanță.

Pentru a rezolva asta, avem nevoie să transformăm cuvintele în numere — dar nu oricum. Numerele trebuie să captureze sensul: cuvintele cu sens similar trebuie să producă numere similare.

Intuiție cheie
Un embedding este un punct într-un spațiu multidimensional. Cuvintele cu sens similar ajung aproape unele de altele în acel spațiu. „Câine" și „pisică" sunt vecini. „Avion" este departe de amândouă.

Ce este un embedding

Un embedding este o listă de numere zecimale (un vector) care reprezintă un text, imagine sau orice altă dată. Modelele moderne de embedding produc vectori cu sute sau mii de dimensiuni.

De exemplu, cuvântul "rege" ar putea fi reprezentat ca un vector de 1536 de numere:

Exemplu vector (simplificat la 8 dimensiuni)
"rege" 0.231 -0.847 0.512 0.093 -0.421 0.768 -0.134 0.659
"regină" 0.228 -0.831 0.498 0.087 -0.409 0.751 -0.121 0.643
"avion" -0.712 0.234 -0.089 0.891 0.302 -0.445 0.677 -0.213

„Rege" și „regină" au valori aproape identice — sunt aproape în spațiu. „Avion" are valori complet diferite.

Valorile individuale nu au un sens explicit — nu știi ce „înseamnă" dimensiunea a 3-a. Dar combinația lor capturează semantica: relații, concepte, tonul textului.

Cum se produce un embedding

Un model de embedding este o rețea neurală antrenată special să producă vectori cu proprietăți semantice. Nu antrenezi tu modelul — folosești unul deja antrenat, prin API sau local.

1

Input: text (sau imagine, audio)

Dai modelului un string: o propoziție, un paragraf, un document.

string
2

Tokenizare

Textul este descompus în token-uri (bucăți de cuvinte). Modelul lucrează cu token-uri, nu cu cuvinte.

tokenizer
3

Rețea neurală (Transformer)

Token-urile trec printr-un model Transformer care „înțelege" contextul și relațiile semantice.

transformer
4

Output: vector de float-uri

Modelul returnează un vector de 768, 1024, sau 1536 de numere zecimale. Acesta este embedding-ul.

float[]

Procesul durează milisecunde pentru text scurt și poți face mii de embeddings pe minut cu un API cloud (OpenAI, Cohere, etc.) sau cu un model local.

Cum măsori similaritatea: cosine similarity

Odată ce ai doi vectori, poți măsura cât de similari sunt. Metrica standard este cosine similarity — măsoară unghiul dintre vectori, nu distanța.

  • 1.0 = vectori identici (același text)
  • 0.9+ = texte foarte similare (parafraze, sinonime)
  • 0.7–0.9 = texte cu aceeași temă
  • 0.0 = texte fără legătură
  • -1.0 = texte cu sens opus (rar în practică)
De ce cosine, nu distanță euclidiană?
Distanța euclidiană este afectată de lungimea vectorului — un text mai lung va produce un vector „mai mare". Cosine similarity normalizează această diferență: compară direcția, nu magnitudinea. Doi vectori care „pointează în aceeași direcție" sunt similari, indiferent de lungime.

Formula pentru cosine similarity este simplă și o vei vedea în toate librăriile:

# Python — formula manuală
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Valoarea returnată este un float între -1 și 1
similarity = cosine_similarity(embedding_a, embedding_b)
print(f"Similaritate: {similarity:.4f}")

Exercițiu mental

Înainte de cod, gândește-te la câteva perechi de texte și estimează similaritatea lor:

Exercițiu — Ordonează după similaritate

Față de propoziția „Pisica mea doarme pe canapea", care propoziție crezi că are similaritate mai mare?

  1. „Felina mea se odihnește pe fotoliu" — aceeași idee, alte cuvinte
  2. „Câinele aleargă în parc" — același domeniu (animale), altă acțiune
  3. „Factura de electricitate a crescut cu 20%" — subiect complet diferit

Răspuns așteptat: 1 > 2 > 3. Modelele de embedding vor produce exact acest rezultat — fără reguli hardcodate, ci din structura spațiului semantic.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Ce returnează un model de embedding când îi dai un text ca input?
Corect. Un embedding este un vector — o listă de float-uri (de obicei 768, 1024 sau 1536 numere). Nu are sens literal citit, dar combinația lor capturează semantica textului.
Întrebarea 2 din 5 Două propoziții cu sens similar vor produce embeddings cu cosine similarity aproape de:
Corect. Cosine similarity are valori între -1 și 1. Texte similare semantic produc vectori care „pointează în aceeași direcție" → valoare apropiată de 1.0. Valoarea exactă depinde de model.
Întrebarea 3 din 5 De ce cosine similarity este preferată față de distanța euclidiană pentru compararea embeddings?
Corect. Distanța euclidiană este influențată de magnitudine — texte mai lungi produc vectori mai mari, denaturând comparația. Cosine măsoară unghiul, nu lungimea, deci normalizează automat.
Întrebarea 4 din 5 Trebuie să antrenezi tu un model de embedding de la zero pentru a-l folosi?
Corect. Modelele de embedding sunt preantrenate pe cantități uriașe de text. Tu doar le folosești — trimiți textul, primești vectorul. Fine-tuning-ul este opțional și necesar doar în cazuri specializate.
Întrebarea 5 din 5 Ce etapă are loc ÎNAINTE ca textul să intre în rețeaua neurală a modelului de embedding?
Corect. Tokenizarea este primul pas: textul este descompus în token-uri (sub-cuvinte) înainte de a fi procesat. Fiecare model are propriul tokenizer — acesta influențează și limita de input (ex. max 8192 token-uri).
