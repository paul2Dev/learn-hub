lecția 3 din 8 ~15 min Embeddings — Curs

Alegerea limbajului și tooling-ului

Python, JavaScript sau PHP — când să folosești fiecare și ce librării să alegi.

Principiu: modelul este independent de limbaj

Un model de embedding este accesat printr-un API HTTP sau rulat local ca librărie. Orice limbaj poate apela un API HTTP — deci teoretic poți folosi embeddings în PHP, Rust sau Go. Diferența este în ecosistemul disponibil: tooluri, librării, comunitate și exemple.

Regula practică
Dacă vrei modele locale sau ecosistem ML complet → Python. Dacă lucrezi pe Node.js sau în browser → JavaScript/TypeScript. Dacă aplicația ta principală e Laravel → PHP cu API calls sau un microserviciu Python.

Python — ecosistemul dominant

Python este alegerea implicită pentru orice proiect ML/AI. Nu pentru că e cel mai rapid sau cel mai simplu, ci pentru că toate librăriile importante există în Python și apar acolo primele.

1

openai (pip)

Client oficial OpenAI. Acces la text-embedding-3-small și text-embedding-3-large. Necesită API key plătit.

cloud API
2

sentence-transformers (pip)

Modele locale, gratuite. Cele mai bune modele open-source: all-MiniLM-L6-v2, paraphrase-multilingual-MiniLM-L12-v2.

local / gratuit
3

numpy / scipy

Calculul cosine similarity și operații pe vectori. Incluse automat cu scipy.

math
4

chromadb / qdrant-client (pip)

Vector databases — stochezi și cauți în milioane de vectori eficient.

vector DB

Când să alegi Python: prototipare rapidă, data science, modele locale, proiecte unde Python este deja în stack, sau când vrei cel mai larg ecosistem disponibil.

JavaScript/TypeScript — pentru aplicații web

Dacă lucrezi cu Node.js (Express, Next.js, NestJS), JavaScript este alegerea naturală — nu introduci un limbaj nou în stack.

1

openai (npm)

Client oficial OpenAI pentru Node.js și browser. Identic ca funcționalitate cu clientul Python.

cloud API
2

@xenova/transformers (npm)

Port JavaScript al Hugging Face Transformers. Rulează modele local, inclusiv în browser. Bazat pe ONNX Runtime.

local / browser
3

langchain / llamaindex (npm)

Framework-uri RAG complete. Abstractizează embeddings, vector stores și LLM-uri într-un API unificat.

RAG framework

Când să alegi JavaScript: aplicații Node.js existente, proiecte Next.js/Nuxt, când vrei embeddings în browser (privacy, offline), sau când echipa nu cunoaște Python.

PHP/Laravel — integrare via API

PHP nu are un ecosistem ML propriu, dar asta nu contează: modelele de embedding sunt accesate prin HTTP. Laravel poate apela orice API REST. 

// Laravel — apelul direct la OpenAI Embeddings API
use Illuminate\Support\Facades\Http;

$response = Http::withToken(env('OPENAI_API_KEY'))
    ->post('https://api.openai.com/v1/embeddings', [
        'input' => 'Textul de vectorizat',
        'model' => 'text-embedding-3-small',
    ]);

$embedding = $response->json('data.0.embedding');
// $embedding este un array de 1536 float-uri

Alternativ, există pachete Composer care wrappează API-urile ML: openai-php/client este clientul oficial OpenAI pentru PHP.

Limitare PHP
Nu poți rula modele locale în PHP fără extensii native (ex. FFI sau o extensie C++). Dacă ai nevoie de modele locale (fără cost API), creează un microserviciu Python cu FastAPI și apelează-l din Laravel.

Comparație modele de embedding

Model Dimensiuni Cost Când să-l alegi
text-embedding-3-small 1536 $0.02/1M tokens Produção, calitate bună, preț mic
text-embedding-3-large 3072 $0.13/1M tokens Precizie maximă, documente specializate
all-MiniLM-L6-v2 384 Gratuit (local) Development, text în engleză, rapid
paraphrase-multilingual-MiniLM-L12-v2 384 Gratuit (local) Text în română sau multilingv, local
Recomandare pentru start
Dacă ești la început: folosește text-embedding-3-small via OpenAI API (Python sau JS). Costul este neglijabil în development — 1 milion de token-uri = câteva sute de pagini de text = $0.02. Schimbi la modele locale ulterior dacă costul devine o problemă.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 De ce Python are ecosistemul dominant pentru embeddings, chiar dacă alte limbaje pot apela același API?
Corect. Viteza limbajului nu contează — bottleneck-ul este rețeaua (API call) sau inferența modelului. Python domină pentru că ecosistemul de librării (numpy, sklearn, huggingface, langchain) a apărut acolo și rămâne cel mai complet.
Întrebarea 2 din 5 Care librărie JavaScript permite rularea modelelor de embedding în browser, fără server?
Corect. @xenova/transformers (acum @huggingface/transformers) rulează modele ONNX direct în browser prin WebAssembly. Asta înseamnă embeddings fără server — útil pentru privacy sau aplicații offline.
Întrebarea 3 din 5 Ai o aplicație Laravel și vrei embeddings cu modele locale (fără cost API). Care este abordarea recomandată?
Corect. PHP nu poate rula modele Transformer local fără extensii native complexe. Un microserviciu Python cu FastAPI expune un endpoint REST — Laravel îl apelează ca orice API. Aceasta separă concerns și permite scaling independent.
Întrebarea 4 din 5 Costul aproximativ pentru a vectoriza 1 milion de token-uri cu text-embedding-3-small este:
Corect. La prețul OpenAI din 2024-2025, text-embedding-3-small costă $0.02/1M tokens. 1 milion de token-uri reprezintă aproximativ 750.000 de cuvinte — câteva cărți întregi. Ideal pentru development și producție cu volum mediu.
Întrebarea 5 din 5 Ce model local recomandat funcționează bine pentru text în română?
Corect. paraphrase-multilingual-MiniLM-L12-v2 este antrenat pe 50+ limbi inclusiv română și produce vectori de 384 dimensiuni — compact și rapid. all-MiniLM-L6-v2 funcționează mai slab pe limbi non-engleze.
