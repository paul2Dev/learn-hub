referință Embeddings — Curs

Glosar Embeddings

Termenii cheie din toate cele 8 lecții, în ordine alfabetică.

A
ANN — Approximate Nearest Neighbor
Algoritm de căutare care returnează cei mai apropiați vecini ai unui vector fără a parcurge tot indexul. Este mai rapid decât căutarea exactă cu un cost mic de precizie (~1–5%).
Exemple de algoritmi ANN: HNSW, IVF, FAISS. → Lecția 6
C
ChromaDB
Vector database open-source, simplu de pornit local. Ideal pentru prototipuri și proiecte mici. Suportă storage în memorie sau persistent pe disc.
→ Lecția 6
Chunking
Procesul de fragmentare a documentelor lungi în bucăți mai mici (chunk-uri) înainte de vectorizare. Necesar deoarece modelele de embedding au o limită de token-uri și chunk-urile mici produc retrieval mai precis.
Valori tipice: 200–500 cuvinte cu overlap 50. → Lecția 7
Clustering
Gruparea automată a punctelor (documente, texte) similare fără etichete predefinite. Pe embeddings, k-means sau DBSCAN grupează textele cu sens similar în clustere.
→ Lecția 2
Cosine Distance
Complementul cosine similarity: 1 - cosine_similarity(a, b). Valori mai mici = mai similar. Folosit de unele librării (scipy, ChromaDB cu metrica default) în loc de similarity.
Atenție: scipy.cosine() returnează distanța, nu similarity! → Lecția 4
Cosine Similarity
Măsura similarității dintre doi vectori bazată pe unghiul dintre ei. Valori între -1 și 1: 1 = identici, 0 = fără legătură, -1 = opuși. Preferată față de distanța euclidiană pentru embeddings deoarece nu e afectată de magnitudinea vectorului.
Formula: dot(a,b) / (norm(a) × norm(b)). → Lecțiile 1, 4, 5
E
Embedding
O reprezentare numerică (vector de float-uri) a unui text, imagine sau alt conținut. Capturează semantica astfel încât texte cu sens similar produc vectori apropiați în spațiu.
Dimensiuni tipice: 384 (MiniLM), 768 (BERT), 1536 (text-embedding-3-small). → Lecția 1
Embedding Model
O rețea neurală Transformer antrenată special să producă vectori cu proprietăți semantice. Nu o antrenezi tu — folosești unul preantrenat via API (OpenAI, Cohere) sau local (sentence-transformers).
→ Lecțiile 1, 3
F
Fine-tuning
Re-antrenarea unui model existent pe date specifice domeniului tău. Modifică weights-urile modelului. Alternativă la RAG — mai scump și mai lent de actualizat, dar poate produce rezultate mai bune în cazuri specializate.
În 95% din cazuri, RAG este preferabil fine-tuning-ului. → Lecția 7
Float32 / float[]
Tipul de date al valorilor dintr-un vector de embedding. Fiecare dimensiune este un număr zecimal pe 32 biți. Un vector de 1536 dimensiuni ocupă 1536 × 4 bytes = 6KB.
→ Lecțiile 1, 4
H
Hallucination
Fenomenul în care un LLM generează informații false dar plauzibile din training data, în loc să recunoască că nu știe. RAG cu instrucțiuni de grounding reduce semnificativ hallucinations.
→ Lecția 7
HNSW — Hierarchical Navigable Small World
Algoritmul ANN cel mai popular pentru vector databases. Construiește o structură de grafuri ierarhică care permite căutarea nearest neighbor în timp sub-liniar.
Suportat de pgvector, Qdrant, ChromaDB. → Lecțiile 6, 8
I
Indexare (vector indexing)
Faza offline în RAG: calculezi embeddings pentru toate documentele și le stochezi în vector database. Se face o singură dată și se repetă doar la adăugarea sau modificarea documentelor.
→ Lecțiile 6, 7
K
Knowledge Cutoff
Data până la care un LLM a fost antrenat. Nu știe nimic despre evenimente sau documente apărute după acea dată. RAG rezolvă asta injectând informații actuale în context.
→ Lecția 7
k-NN — k Nearest Neighbors
Algoritmul de căutare a celor mai apropiați k vecini ai unui vector. Exact: compară cu toți vectorii (O(n)). Aproximativ (ANN): structuri de date eficiente reduc la O(log n).
→ Lecțiile 4, 6
L
LLM — Large Language Model
Model de limbaj de mari dimensiuni (GPT-4, Claude, Gemini). Generează text, răspunde la întrebări, rezumă. În RAG, LLM-ul generează răspunsul pe baza contextului furnizat de retrieval.
→ Lecțiile 2, 7
M
Metadata Filtering
Combinarea căutării semantice cu filtre structurate (categorie, dată, user ID). Permite query-uri de tipul „similare cu X, dar DOAR din categoria Y".
Suportat de ChromaDB, Qdrant, pgvector (WHERE clause). → Lecția 6
MiniLM
Familie de modele de embedding compacte și rapide: all-MiniLM-L6-v2 (engleză, 384 dim) și paraphrase-multilingual-MiniLM-L12-v2 (50+ limbi, 384 dim). Disponibile gratuit pe Hugging Face.
→ Lecțiile 3, 4
N
Normalizare vector
Scalarea unui vector la lungime unitară (norma = 1). Când vectorii sunt normalizați, cosine similarity se reduce la produsul scalar. Parametrul normalize=True în sentence-transformers face asta automat.
→ Lecția 5
O
Overlap (chunking)
Numărul de cuvinte sau token-uri comune între chunk-uri consecutive. Previne pierderea contextului la granița dintre chunk-uri. Valoare tipică: 50–100 cuvinte.
→ Lecția 7
P
pgvector
Extensie PostgreSQL care adaugă tipul de date vector(n) și operatori pentru cosine distance (<=>), distanță euclidiană (<->) și inner product (<#>). Permite semantic search direct în Postgres.
→ Lecțiile 6, 8
Pinecone
Vector database cloud fully-managed. Zero infrastructure de gestionat. Mai scump la scară mare, dar simplu de integrat prin API REST. Popular în aplicații SaaS.
→ Lecția 6
Q
Qdrant
Vector database open-source scris în Rust. Performant, self-hosted sau managed cloud. Are client PHP oficial, util pentru integrare Laravel.
→ Lecțiile 6, 8
Query Embedding
Vectorul calculat pentru întrebarea/query-ul utilizatorului la momentul căutării. Se compară cu embeddings-urile documentelor din index pentru a găsi cele mai similare.
→ Lecțiile 4, 5, 6, 7
R
RAG — Retrieval Augmented Generation
Paradigmă care combină semantic search (retrieval) cu generarea de text (LLM). Embeddings găsesc documentele relevante → context injectat în prompt → LLM generează răspunsul. Permite LLM-ului să „știe" despre documentele tale fără re-antrenare.
→ Lecțiile 2, 7, 8
Retrieval
În contextul RAG, procesul de găsire a documentelor relevante semantic pentru o întrebare. Implică vectorizarea query-ului și nearest neighbor search în vector database.
→ Lecțiile 6, 7
S
Semantic Search
Căutare bazată pe înțelesul textului, nu pe potrivirea exactă de cuvinte cheie. Query-ul și documentele sunt vectorizate; se returnează documentele cu similarity maximă față de query. Funcționează cross-lingual.
→ Lecțiile 2, 4, 5, 6, 8
Sentence Transformers
Librărie Python (sbert.net) pentru modele de embedding locale. Oferă zeci de modele preantrenate, inclusiv multilingve. pip install sentence-transformers. Zero cost de API.
→ Lecțiile 3, 4
Spațiu vectorial (Vector Space)
Spațiul n-dimensional în care trăiesc vectorii de embedding. Texte semantice similare sunt „aproape" în acest spațiu. Nu are o vizualizare intuitivă la 1536 dimensiuni, dar proprietățile geometrice (distanță, unghi) funcționează matematic.
→ Lecția 1
T
Temperature (LLM)
Parametru care controlează aleatoria în generarea textului. Temperature = 0: determinist, același input → același output. Recomandat pentru RAG factual. Temperature = 1: variabilitate naturală.
→ Lecția 7
text-embedding-3-small / text-embedding-3-large
Modelele de embedding OpenAI din generația 3. small: 1536 dim, $0.02/1M tokens. large: 3072 dim, $0.13/1M tokens. Ambele suportă reducerea dimensiunilor fără pierderi mari de calitate.
→ Lecțiile 3, 4, 8
Token
Unitatea de bază de procesare a modelelor NLP. Aproximativ ~4 caractere sau 3/4 dintr-un cuvânt în engleză. Modelele de embedding au limite de token-uri per input (ex. 8192 pentru text-embedding-3-small).
→ Lecțiile 1, 7
Tokenizare
Procesul de descompunere a textului în token-uri înainte de procesarea de către model. Fiecare model are propriul tokenizer. Afectează numărul de token-uri dintr-un text și limita de input.
→ Lecția 1
Transformer
Arhitectura de rețea neurală la baza modelelor moderne de NLP (BERT, GPT, etc.). Folosește mecanisme de „atenție" pentru a înțelege relațiile contextuale dintre cuvinte. Modelele de embedding sunt Transformers specializați.
→ Lecția 1
Transformers.js (@xenova/transformers)
Port JavaScript al librăriei Hugging Face Transformers. Rulează modele ONNX local în Node.js sau browser (via WebAssembly/WebGPU). Permite embeddings fără server și fără cost API.
→ Lecția 5
V
Vector
O listă ordonată de numere. În contextul embeddings, un vector de n numere float reprezintă un punct în spațiul n-dimensional. Proprietățile geometrice (distanță, unghi) capturează relații semantice.
→ Lecția 1
Vector Database
Bază de date specializată în stocarea și căutarea eficientă a vectorilor. Folosește algoritmi ANN pentru nearest neighbor search sub-liniar. Exemple: ChromaDB, Qdrant, Pinecone, Weaviate, pgvector.
→ Lecția 6
