Greploop — Bucla Autonomă Review-Fix-Push

Skill-ul Claude Code care iterează automat până când Greptile dă 5/5 cu zero comentarii nerezolvate.

Ce este Greploop

Greploop este un skill open-source pentru Claude Code, creat de echipa Greptile. Este un developer loop care nu se oprește până când codul tău trece review-ul complet. Automatizează complet ciclul obositor de: primești feedback → aplici fix → re-pushezi → aștepți review din nou.

🔄 Greploop Skill Claude Code · open source · necesită Greptile

Ce face: Declanșează review Greptile → citește feedback → aplică fiecare fix → push → repetă

Condiție de stop: Greptile dă scor 5/5 cu zero comentarii nerezolvate, sau după 5 iterații (limită anti-runaway)

Platforme: GitHub, GitLab, Perforce

Cerință: Greptile instalat și indexat pe repository (Lecția 6)

Maxim 5 iterații per rulare — nu poate rula la infinit

Instalare Greploop

Greploop se instalează ca orice skill Claude Code — ca fișier SKILL.md descoperit automat de Claude Code:

# Creează directorul de skills dacă nu există
mkdir -p ~/.claude/skills/greploop

# Descarcă skill-ul din repository-ul oficial Greptile
# (urmează instrucțiunile de pe claudskills.com/skills/greploop/)

# Sau clonează repository-ul de skills Greptile
git clone https://github.com/greptileai/skills ~/.claude/skills/greptile-skills

Odată instalat, Claude Code descoperă automat skill-ul și îl poți invoca cu:

/greploop

Ce face și ce nu face Greploop

Face Nu face
Aplică fix-urile sugerate de Greptile Nu face merge în main — tu decizi
Commit și push automat după fiecare fix Nu rezolvă conflicte de merge
Iterează maxim 5 ori per rulare Nu rulează dacă Greptile nu e instalat
Funcționează pe GitHub, GitLab, Perforce Nu poate fixa buguri pe care Greptile nu le detectează
Revizuiește commit-urile Greploop
Greploop face commit-uri automate. Înainte de a da merge, citește ce a schimbat în fiecare iterație — git log --oneline și git diff HEAD~N pentru a vedea schimbările. Un agent care aplică fix-uri poate introduce regresia dacă fix-ul e incorect.

Limita de 5 iterații — design deliberat

Greploop se oprește după maxim 5 iterații chiar dacă nu a ajuns la 5/5. Aceasta este o decizie de design deliberată: dacă după 5 iterații codul tot nu trece review-ul, problema necesită intervenție umană — nu mai mult agent.

Când Greploop se oprește la limită, verifică:

  • Finding-urile Greptile care au rămas nerezolvate
  • De ce agentul nu le-a putut aplica (poate necesită decizie de arhitectură)
  • Dacă finding-urile sunt false positives — le poți marca ca atare în Greptile
Exercițiu — Rulează Greploop pe un PR real
  1. Asigură-te că ai Greptile instalat pe repository (Lecția 6) și un PR deschis.
  2. Instalează skill-ul Greploop urmând instrucțiunile de pe claudskills.com/skills/greploop/
  3. Rulează /greploop și observă iterațiile în timp real.
  4. La final, rulează git log --oneline -10 pentru a vedea commit-urile create de Greploop.
  5. Revizuiește fiecare schimbare cu git show <commit-hash> înainte de merge.

Quiz — Verifică înțelegerea

Întrebarea 1 din 5 Care este condiția exactă în care Greploop se oprește cu succes?
Corect. Condiția de succes este strictă: scor 5/5 de la Greptile și zero comentarii active nerezolvate. Altfel, Greploop mai iterează sau se oprește la limita de 5 iterații.
Întrebarea 2 din 5 De ce limita de 5 iterații este un design deliberat și nu o limitare tehnică?
Exact. Dacă un agent nu poate rezolva problema în 5 iterații, înseamnă că problema depășește capacitatea de fix automat — poate necesită o decizie de arhitectură sau contextul pe care numai tu îl ai.
Întrebarea 3 din 5 Ce trebuie să ai instalat ÎNAINTE de a putea folosi Greploop?
Corect. Greploop este un orchestrator peste Greptile — fără Greptile instalat și indexat, Greploop nu are cu ce serviciu să interacționeze. Instalează Greptile mai întâi (Lecția 6).
Întrebarea 4 din 5 Ce ar trebui să faci ÎNAINTE de a da merge după ce Greploop termină?
Exact. Greploop a creat commit-uri automate — trebuie să verifici ce a schimbat. git log --oneline arată commit-urile, git show <hash> arată diff-ul fiecăruia. Merge-ul rămâne decizia ta.
Întrebarea 5 din 5 În ce moment din workflow rulezi /greploop față de crearea PR-ului?
Corect. Greploop necesită un PR deschis pentru a putea declanșa review-ul Greptile și a posta fix-urile înapoi. Ordinea: cod → /verify → /code-review → push → gh pr create → /greploop → merge.
