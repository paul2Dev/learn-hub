Code Review cu Claude

Trei skill-uri pentru review înainte de PR: /code-review, /security-review, și /code-review ultra pentru review profund în cloud.

De ce review local înainte de PR?

Un PR deschis cu buguri sau vulnerabilități costă mai mult decât unul deschis după review. Review-ul local cu Claude este rapid (secunde–minute) și prins înainte ca problema să devină vizibilă pentru orice serviciu extern sau coleg.

/code-review — Review rapid al diff-ului

Skill-ul standard de review. Analizează diff-ul față de branch-ul de bază și raportează:

  • Buguri de corectitudine (logică greșită, null pointer, race conditions)
  • Oportunități de reuse și simplificare (dar nu le aplică — pentru asta e /simplify)
  • Probleme de eficiență evidente

Invocă-l simplu:

/code-review

Sau cu nivel de efort specificat:

/code-review low      # Câteva finding-uri, high-confidence
/code-review medium   # Mai multe finding-uri (default)
/code-review high     # Acoperire mai largă, poate include finding-uri incerte
/code-review max      # Tot ce poate găsi — include finding-uri speculative

Flag-uri utile

/code-review --comment   # Postează finding-urile ca inline comments pe PR
/code-review --fix       # Aplică fix-urile direct în working tree
Când să folosești --fix
Folosește --fix când ai încredere că finding-urile sunt corecte și vrei să le aplici rapid. Revizuiește întotdeauna ce a schimbat înainte de commit — chiar și un agent face greșeli.

/security-review — Scanare pentru vulnerabilități

Skill-ul /security-review face o analiză de securitate a schimbărilor de pe branch-ul curent. Caută:

  • Injection — SQL injection, command injection, XSS
  • Autentificare și autorizare — endpoint-uri neprotejate, logică de access control greșită
  • Secrets expuse — credențiale hardcodate, API keys în cod
  • Dependențe vulnerabile — pachete cu CVE-uri cunoscute
  • OWASP Top 10 — categoriile principale de vulnerabilități web

Cum se invocă:

/security-review
Nu înlocuiește un audit complet
/security-review este un prim filtru, nu o garanție de securitate. Pentru aplicații care gestionează date sensibile sau tranzacții financiare, un audit de securitate profesional rămâne necesar.

/code-review ultra — Review profund în cloud

Cel mai puternic instrument de review din Claude Code. Lansează un review multi-agent în cloud — agenți multipli analizează codul din unghiuri diferite și sintetizează finding-urile.

/code-review ultra          # Review branch-ul curent
/code-review ultra 42       # Review PR #42 de pe GitHub

Ce face diferit față de /code-review standard:

  • Citește fișierele întregi, nu doar excerpts — nu ratează context din afara diff-ului
  • Agenți multipli cu perspective diferite (corectitudine, securitate, arhitectură)
  • Poate posta finding-urile direct ca inline comments pe PR: /code-review ultra --comment
  • Durează câteva minute (față de secunde la /code-review standard)
Cost și disponibilitate
/code-review ultra este facturat și necesită un repository git. Folosit strategic pe PR-uri mari sau înainte de release-uri importante — nu pe fiecare commit.

Când să folosești ce

Situație Skill recomandat
Fix mic, schimbare simplă /code-review low
Feature nou, schimbări medii /code-review (medium)
Cod cu autentificare sau date sensibile /security-review în plus
PR mare sau înainte de release /code-review ultra
Vrei să postezi finding-urile pe PR /code-review --comment
Exercițiu — Fă review pe ultima schimbare a ta
  1. Asigură-te că ești pe un branch cu schimbări față de master: git status
  2. Rulează /code-review și citește fiecare finding cu atenție.
  3. Identifică cel puțin un finding pe care l-ai fi ratat în review-ul manual.
  4. Dacă ai cod care gestionează input de la utilizator, rulează și /security-review.
  5. Aplicați fix-urile relevante cu /code-review --fix sau manual, după caz.

Quiz — Verifică înțelegerea

Întrebarea 1 din 5 Ce diferențiază /code-review ultra față de /code-review standard?
Corect. Ultra înseamnă multi-agent în cloud + citire fișiere întregi. Nu ratează context din afara diff-ului și sintetizează perspective diferite — corectitudine, securitate, arhitectură.
Întrebarea 2 din 5 Ce flag adaugi la /code-review dacă vrei ca finding-urile să apară ca inline comments pe PR?
Corect. /code-review --comment (sau /code-review ultra --comment) postează finding-urile direct ca inline comments pe PR-ul deschis în GitHub.
Întrebarea 3 din 5 Ce caută /security-review dar /code-review standard nu se focusează pe asta?
Exact. /security-review are un focus explicit pe vulnerabilități de securitate: injection attacks, secrets hardcodate, endpoint-uri neprotejate, dependențe cu CVE-uri. /code-review face review general de corectitudine.
Întrebarea 4 din 5 Când este recomandat să folosești /code-review max în loc de /code-review standard?
Corect. Nivelul `max` include finding-uri pe care agentul nu e sigur 100% că sunt probleme reale. Util pentru PR-uri critice unde preferi false positives față de finding-uri ratate.
Întrebarea 5 din 5 Ce skill aplică fix-urile de calitate direct în codul tău, pe lângă raportarea lor?
Corect. /simplify aplică. /code-review raportează (sau cu --fix aplică specific). Diferența: /simplify e pentru calitate și claritate; /code-review cu --fix e pentru buguri identificate de review.
