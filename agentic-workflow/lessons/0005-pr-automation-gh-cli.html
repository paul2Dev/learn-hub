PR Automation cu gh CLI

Claude Code poate crea PR-uri cu descrieri complete — tu doar dai click pe Merge.

Ce este gh CLI și de ce îl folosim

gh este CLI-ul oficial GitHub. Permite interacțiunea cu GitHub din terminal: creare PR-uri, adăugare labels, citire comentarii, merge. Claude Code îl folosește nativ — nu are nevoie de setup special din partea ta.

Instalare
Dacă nu ai gh instalat: winget install GitHub.cli (Windows) sau brew install gh (macOS). Autentificare: gh auth login.

Anatomia unui PR bun

Un PR util conține mai mult decât codul. Conține context pentru reviewer (uman sau AI):

📋 PR bun — structură template

Titlu: concis, max 70 caractere, prefix tip (feat/fix/refactor)

Summary: 1–3 bullet points — ce face această schimbare

Test plan: ce ai verificat, cum să testeze reviewer-ul

Context: de ce această abordare, tradeoff-uri, alternative considerate

Claude Code creează PR-uri automat

Claude Code știe să creeze PR-uri cu structură completă. Îi poți cere explicit sau el poate face asta ca parte din workflow:

# Cere lui Claude Code să creeze PR-ul
"Creează un PR pentru branch-ul curent cu descriere completă"

# Sau Claude Code face asta automat după /code-review
# — el generează titlul, body-ul, și îl creează cu gh pr create

Comanda reală pe care Claude Code o rulează:

gh pr create --title "feat: add user authentication flow" --body "$(cat <<'EOF'
## Summary
- Adaugă endpoint POST /auth/login cu JWT
- Implementează middleware de autentificare pentru rute protejate
- Adaugă refresh token cu expirare 7 zile

## Test plan
- [ ] Login cu credențiale valide returnează JWT valid
- [ ] Login cu credențiale greșite returnează 401
- [ ] Endpoint protejat fără token returnează 403
- [ ] Refresh token funcționează după expirare JWT

🤖 Generated with Claude Code
EOF
)"

Comenzi esențiale gh CLI

Comenzile pe care le vei folosi cel mai des:

# Listează PR-urile deschise
gh pr list

# Verifică statusul PR-ului curent
gh pr status

# Deschide PR-ul în browser
gh pr view --web

# Mergeach PR cu squash (un singur commit în main)
gh pr merge --squash

# Adaugă label pe PR
gh pr edit --add-label "needs-review"

# Citește comentariile de review pe PR
gh api repos/{owner}/{repo}/pulls/{number}/comments

# Verifică statusul check-urilor CI pe PR-ul curent
gh pr checks

Workflow complet cu gh CLI

1

Claude Code scrie cod pe branch nou: git checkout -b feature/xyz

↓
2

Claude Code rulează /verify și /code-review

↓
3

Claude Code face commit și push: git push origin feature/xyz

↓
4

Claude Code creează PR: gh pr create --title "..." --body "..."

↓
5

Tu deschizi PR-ul în GitHub, citești review-ul AI (Greptile/CodeRabbit), și dai click pe Merge

Verificarea statusului PR din terminal

Nu trebuie să deschizi GitHub în browser pentru a vedea ce se întâmplă cu PR-ul:

# Vezi toate check-urile CI și statusul lor
gh pr checks

# Output exemplu:
# ✓ build         pass  1m30s
# ✓ lint          pass  45s
# ✗ tests         fail  2m15s  — click pentru detalii
Claude Code și gh pr merge
Nu instrui Claude Code să ruleze gh pr merge automat. Merge-ul în main este decizia ta — cel care înțelege contextul de business și a citit review-ul complet.

Labels utile pentru triaj rapid

Adaugă labels ca să filtrezi rapid PR-urile în GitHub:

# Creează labels (o singură dată per repo)
gh label create "needs-review" --color "#0052CC"
gh label create "ready-to-merge" --color "#0E8A16"
gh label create "work-in-progress" --color "#E4E669"

# Adaugă label pe PR-ul curent
gh pr edit --add-label "needs-review"
Exercițiu — Creează primul PR automat cu gh CLI
  1. Creează un branch nou: git checkout -b feature/test-gh-cli
  2. Adaugă un fișier simplu: echo "test" > test-gh.txt și fă commit.
  3. Push pe branch: git push origin feature/test-gh-cli
  4. Creează PR cu gh CLI urmând structura de mai sus (sau roagă Claude Code să creeze PR-ul).
  5. Rulează gh pr status și gh pr view --web pentru a vedea PR-ul.

Quiz — Verifică înțelegerea

Întrebarea 1 din 5 Ce informații esențiale ar trebui să conțină body-ul unui PR bun?
Corect. Un PR bun conține context pentru reviewer: ce face, cum s-a testat, și de ce această abordare. GitHub arată automat fișierele modificate — nu trebuie listate manual.
Întrebarea 2 din 5 Ce comandă gh CLI verifici statusul check-urilor CI ale PR-ului curent?
Corect. gh pr checks arată statusul tuturor GitHub Actions care rulează pe PR-ul curent — build, lint, teste — cu timpul de rulare și link-uri la log-uri în caz de eșec.
Întrebarea 3 din 5 De ce nu ar trebui să instrui Claude Code să ruleze `gh pr merge` automat?
Exact. Modelul hibrid păstrează merge-ul la om. Agentul creează PR-ul, serviciile AI fac review — tu citești tot și iei decizia. Merge-ul automat elimină această ultimă verificare.
Întrebarea 4 din 5 Care comandă gh CLI deschide PR-ul curent direct în browser?
Corect. gh pr view --web deschide PR-ul curent în browser-ul implicit — util când vrei să citești comentariile de review sau să dai click pe Merge.
Întrebarea 5 din 5 Ce face `gh pr merge --squash` față de `gh pr merge` fără flag?
Corect. --squash combină toate commit-urile de pe branch într-un singur commit în main — rezultă un git log mai curat, unde fiecare feature este un singur commit descriptiv.
