Ce este un workflow agentic

Fluxul complet de la idee la producție — și de ce contează ordinea pașilor.

Problema workflow-ului manual

Când scrii cod cu un agent AI (cum ar fi Claude Code) dar restul workflow-ului rămâne manual, apar trei tipuri de probleme:

  • Risc de orbire — nu știi exact ce a schimbat agentul înainte de push
  • Review superficial — citești diff-ul singur, fără context deplin al codebase-ului
  • Lipsa feedback loop-ului — nu ai dovadă că schimbarea funcționează înainte de merge

Un workflow agentic rezolvă toate trei: agentul face treaba de rutină, tu controlezi deciziile cu impact real.

Un workflow agentic matur arată astfel — fiecare pas este fie automat, fie cu aprobare explicită:

Modelul hibrid — de ce nu totul automat?

Există o tentație să automatizezi complet — inclusiv merge-ul. Aceasta este o greșeală pentru proiecte cu cod viu în producție:

Risc
Un agent poate introduce un bug subtil care trece de toate check-urile automate dar strică UX-ul pentru utilizatori reali. Omul rămâne ultima linie de apărare.

Modelul recomandat este hibrid: automatizezi tot ce este repetitiv și fără risc (review, formatare, descriere PR, smoke tests), și păstrezi controlul uman pe deciziile ireversibile (merge în main, deploy în producție).

Ce vei construi în acest curs

La finalul celor 10 lecții, vei avea un workflow în care:

  • Niciun commit nu poate intra în master fără un PR aprobat
  • Fiecare PR primește automat un review AI detaliat
  • GitHub Actions verifică build-ul și lint-ul la fiecare PR
  • Claude Code face review local înainte de push
  • Tu decizi merge-ul — cu toate informațiile în față

Quiz — Verifică înțelegerea

Întrebarea 1 din 5 Care este riscul principal al unui workflow în care agentul AI poate face merge automat în main?
Corect. Automatizarea completă elimină ultima linie de apărare. Modelul hibrid păstrează decizia de merge la om — exact pentru a prinde ce scapă tuturor check-urilor automate.
Întrebarea 2 din 5 Ce este rolul lui Greptile sau CodeRabbit în workflow-ul agentic?
Exact. Aceste servicii rulează în cloud și indexează întregul codebase — pot detecta buguri pe care un reviewer uman le-ar rata pentru că nu țin în cap toate dependențele.
Întrebarea 3 din 5 Care pas din fluxul de 6 pași este singurul care necesită decizie umană?
Corect. Toți ceilalți pași sunt automatizați. Merge-ul este singurul moment în care omul intervine deliberat — și de aceea contează să ai toate informațiile (review AI, CI checks) înainte de acel click.
Întrebarea 4 din 5 Ce face GitHub Actions în workflow-ul agentic descris în această lecție?
Corect. GitHub Actions este stratul de CI — nu scrie cod și nu face review semantic, dar verifică că build-ul nu e rupt și că testele trec, la fiecare PR, automat.
Întrebarea 5 din 5 De ce se numește modelul recomandat „hibrid" și nu „complet automat"?
Exact. „Hibrid" înseamnă: tot ce este repetitiv și reversibil — automatizezi. Tot ce este ireversibil și cu impact în producție (merge, deploy) — omul decide. Costul unui click este mic; costul unui bug în producție este mare.
