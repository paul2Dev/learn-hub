Glosar — Workflow Agentic Complet

Termenii, tool-urile și conceptele din tot cursul — pentru referință rapidă.

A — Tool-uri Claude Code

/verify Skill Claude Code

Skill care pornește aplicația, testează golden path-ul și edge case-urile, și raportează ce funcționează și ce nu. Se rulează înainte de orice commit — mai ales după ce un agent a scris cod.

/simplify Skill Claude Code

Skill care revizuiește codul schimbat (doar diff-ul, nu tot codebase-ul) și aplică direct fix-uri de calitate: eliminare duplicări, reuse, claritate. Nu vânează buguri — pentru asta există /code-review.

/code-review Skill Claude Code

Skill de review al diff-ului curent. Raportează buguri de corectitudine, probleme de eficiență, oportunități de simplificare. Opțiuni de efort: low / medium / high / max. Flag-uri: --comment (postează pe PR), --fix (aplică fix-urile).

/code-review ultra Skill Claude Code · Cloud

Varianta cloud multi-agent a /code-review. Citește fișierele întregi (nu doar excerpts), folosește agenți multipli cu perspective diferite. Durează câteva minute; cel mai profund review disponibil în Claude Code.

/security-review Skill Claude Code

Analiză de securitate a branch-ului curent față de main. Caută vulnerabilități OWASP Top 10, injection attacks, secrets expuse în cod, endpoint-uri neprotejate, dependențe cu CVE-uri.

/greploop Skill Claude Code · necesită Greptile

Skill open-source creat de Greptile. Automatizează bucla review-fix-push: declanșează Greptile, citește feedback, aplică fix-uri, push, repetă. Se oprește la scor 5/5 sau după 5 iterații (anti-runaway).

B — Tool-uri Externe de Review

Greptile Serviciu Cloud · $30/dev/month

Serviciu de review PR care indexează întregul codebase și construiește un graf semantic. Face multi-hop investigation la fiecare PR — urmărește dependențele prin fișiere și git history. Bug catch rate: 82%. Suportă GitHub și GitLab.

CodeRabbit Serviciu Cloud · $24/dev/month

Serviciu de review PR automat. Review în ~30 secunde, bug catch rate 46%, mai puține false positives decât Greptile. Funcționalități distinctive: Autofix 1-click, generare teste, Issue Planner. Suportă GitHub, GitLab, Azure DevOps, Bitbucket.

C — GitHub și CI/CD

Branch Protection GitHub Settings

Regulă GitHub care blochează push-urile directe pe un branch protejat (master/main) și impune că orice schimbare să treacă printr-un PR aprobat. Fundația oricărui workflow agentic serios.

GitHub Actions CI/CD Cloud

Sistem de automatizare integrat în GitHub. Rulează workflow-uri YAML la evenimente (PR deschis, push, @claude mention). Folosit pentru: build, lint, teste, și invocarea Claude Code din PR-uri.

gh CLI Tool CLI

CLI-ul oficial GitHub. Permite interacțiunea cu GitHub din terminal: creare PR-uri (gh pr create), verificare status (gh pr status), checks CI (gh pr checks), merge (gh pr merge --squash). Claude Code îl folosește nativ.

Status Check GitHub Concept

Job de CI (build, lint, test) care trebuie să treacă înainte de a permite merge-ul unui PR. Se configurează în branch protection → „Require status checks to pass before merging".

HTTP Hooks Claude Code Config

Comenzi locale care se execută la evenimente Claude Code (PostToolUse, Stop). Se configurează în .claude/settings.json. Exemple: rulare lint după fiecare Edit, raport la final de sesiune.

ANTHROPIC_API_KEY GitHub Secret

Cheia API Anthropic necesară pentru integrarea Claude Code în GitHub Actions. Se salvează ca GitHub Repository Secret (Settings → Secrets → Actions) — niciodată în cod sau fișiere commise.

D — Concepte de Workflow

Workflow Agentic Concept

Un workflow de development în care un agent AI (Claude Code) automatizează pașii repetitivi (scriere cod, review, creare PR, fix-uri), iar omul controlează deciziile ireversibile (merge în main, deploy în producție).

Model Hibrid Concept

Filozofie de workflow: automatizează tot ce este repetitiv și reversibil; păstrează controlul uman pe deciziile ireversibile cu impact real. Opusul: „complet automat" (risc) sau „complet manual" (lent).

Multi-hop Investigation Concept Greptile

Tehnica Greptile de a urmări lead-urile prin fișiere, dependențe și git history — nu stă la diff. Dacă o funcție se schimbă, Greptile găsește toți caller-ii din tot codebase-ul și verifică impactul.

Golden Path Concept /verify

Calea principală pe care un utilizator real ar urma-o în aplicație. /verify testează golden path-ul înainte de edge cases — dacă golden path-ul nu funcționează, edge cases nu mai contează.

Edge Case Concept /verify

Cazuri limită sau neașteptate: input gol, valori extrem de mari, caractere speciale, request fără autentificare. /verify testează edge cases după ce golden path-ul este confirmat.

Bug Catch Rate Metrică

Procentul de buguri reale dintr-un set de test pe care tool-ul le detectează. Greptile: 82%. CodeRabbit: 46%. Benchmark-uri independente 2026. Rată mai mare = mai profund, dar și mai multe false positives.

False Positive Concept review

Finding raportat de un tool de review care nu este o problemă reală. Greptile are mai multe false positives decât CodeRabbit — prețul profunzimii. Se marchează ca atare în dashboard-ul Greptile.
