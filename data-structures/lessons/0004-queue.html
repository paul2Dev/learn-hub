Queue (Coadă)

Structura FIFO — First In, First Out. Fundamentală pentru job queues, BFS, sisteme de mesagerie și procesarea task-urilor în ordine.

Principiul FIFO

O Coadă funcționează ca o coadă la casă: primul venit, primul servit. Elementele se adaugă la coadă (enqueue) și se scot din față (dequeue) — First In, First Out. 

Coada vizualizată (față ← elemente → spate):

enqueue(A) → [ A ]
enqueue(B) → [ A, B ]
enqueue(C) → [ A, B, C ]
dequeue()  → returnează A, coada = [ B, C ]
dequeue()  → returnează B, coada = [ C ]
front()    → returnează C (fără să scoată)

Operațiile unei cozi:

  • enqueue(val) — adaugă la spate — O(1)
  • dequeue() — scoate din față — O(1) cu implementare corectă
  • front() — citește primul element fără a-l scoate — O(1)
  • isEmpty() — verifică dacă coada e goală — O(1)
Atenție — implementare cu array simplu
array_shift() în PHP și shift() în JS sunt O(n) — deplasează toate elementele. Pentru o coadă eficientă cu volume mari, folosește SplQueue (PHP) sau o implementare cu indice de start.

Implementare în PHP

// Variantă 1: SplQueue — implementare nativă optimizată O(1)
$queue = new SplQueue();
$queue->enqueue('task1');
$queue->enqueue('task2');
$queue->enqueue('task3');

echo $queue->dequeue();  // task1
echo $queue->bottom();   // task2 (front, fără scoatere)
echo $queue->count();    // 2

// Variantă 2: clasă custom bazată pe array asociativ
class Queue {
    private array $items = [];
    private int $head = 0;

    public function enqueue(mixed $value): void {
        $this->items[] = $value;
    }

    public function dequeue(): mixed {
        if ($this->isEmpty()) throw new UnderflowException('Queue is empty');
        $value = $this->items[$this->head];
        unset($this->items[$this->head]);
        $this->head++;
        return $value;
    }

    public function front(): mixed {
        if ($this->isEmpty()) throw new UnderflowException('Queue is empty');
        return $this->items[$this->head];
    }

    public function isEmpty(): bool {
        return $this->head >= count($this->items) + $this->head;
    }

    public function size(): int {
        return count($this->items);
    }
}

Implementare în JavaScript

class Queue {
    #items = {};
    #head  = 0;
    #tail  = 0;

    enqueue(value) {
        this.#items[this.#tail] = value;
        this.#tail++;
    }

    dequeue() {
        if (this.isEmpty()) throw new Error('Queue is empty');
        const value = this.#items[this.#head];
        delete this.#items[this.#head];
        this.#head++;
        return value;
    }

    front() {
        if (this.isEmpty()) throw new Error('Queue is empty');
        return this.#items[this.#head];
    }

    isEmpty() { return this.#tail === this.#head; }
    get size() { return this.#tail - this.#head; }
}

const q = new Queue();
q.enqueue('A');
q.enqueue('B');
q.enqueue('C');
console.log(q.dequeue()); // A
console.log(q.front());   // B
console.log(q.size);      // 2

Aplicații reale

1. Job Queue (Laravel / Node.js)

// Conceptual — cum funcționează Laravel Queues internaly
class JobQueue {
    private Queue $queue;

    public function dispatch(callable $job): void {
        $this->queue->enqueue($job);
    }

    public function process(): void {
        while (!$this->queue->isEmpty()) {
            $job = $this->queue->dequeue();
            $job();  // execută job-ul
        }
    }
}

$jq = new JobQueue(new Queue());
$jq->dispatch(fn() => sendEmail('user@example.com'));
$jq->dispatch(fn() => resizeImage('photo.jpg'));
$jq->dispatch(fn() => generatePDF('report'));
$jq->process();  // procesează în ordinea adăugării

2. BFS — Breadth First Search

// Parcurgerea unui graf nivel cu nivel folosind o coadă
function bfs(array $graph, string $start): array {
    $visited = [];
    $queue   = new SplQueue();
    $queue->enqueue($start);
    $visited[$start] = true;

    while (!$queue->isEmpty()) {
        $node = $queue->dequeue();
        echo $node . ' ';

        foreach ($graph[$node] as $neighbor) {
            if (!isset($visited[$neighbor])) {
                $visited[$neighbor] = true;
                $queue->enqueue($neighbor);
            }
        }
    }
    return array_keys($visited);
}

$graph = ['A' => ['B', 'C'], 'B' => ['D'], 'C' => ['D', 'E'], 'D' => [], 'E' => []];
bfs($graph, 'A');  // A B C D E

3. Rate Limiting cu sliding window

// Permite maxim N request-uri în ultimele M secunde
class RateLimiter {
    private SplQueue $timestamps;
    private int $limit;
    private int $windowSeconds;

    public function __construct(int $limit, int $windowSeconds) {
        $this->timestamps    = new SplQueue();
        $this->limit         = $limit;
        $this->windowSeconds = $windowSeconds;
    }

    public function allow(): bool {
        $now = time();
        // Elimină timestamp-urile vechi din fereastră
        while (!$this->timestamps->isEmpty()
               && $this->timestamps->bottom() < $now - $this->windowSeconds) {
            $this->timestamps->dequeue();
        }
        if ($this->timestamps->count() < $this->limit) {
            $this->timestamps->enqueue($now);
            return true;
        }
        return false;  // limita atinsă
    }
}

$limiter = new RateLimiter(5, 60);  // 5 req/minut
echo $limiter->allow() ? 'OK' : 'Rate limited';

Verifică-ți înțelegerea

Întrebarea 1 din 5 Ce înseamnă FIFO în contextul unei cozi?
Corect. FIFO înseamnă că elementele se extrag în aceeași ordine în care au fost adăugate — ca o coadă la casă. Primul client venit este primul servit. Contrastează cu LIFO (Stack) unde ultimul venit iese primul.
Întrebarea 2 din 5 De ce array_shift() nu e ideal pentru implementarea unui Queue cu volume mari?
Corect. array_shift() elimină primul element și reindexează tot array-ul de la 0 — O(n). Pe 1 milion de elemente în coadă, fiecare dequeue mută un milion de referințe. SplQueue sau implementarea cu indice de start rezolvă asta în O(1).
Întrebarea 3 din 5 De ce algoritmul BFS (Breadth First Search) folosește o coadă și nu un stack?
Corect. BFS explorează toate nodurile de la distanța 1, apoi toate de la distanța 2 etc. Coada (FIFO) asigură că procesăm nodurile în ordinea descoperirii. Un stack (LIFO) ar produce DFS — explorare în adâncime, nu în lățime.
Întrebarea 4 din 5 Care este principala diferență de comportament dintre un Stack și o Queue?
Corect. Aceasta este diferența fundamentală: Stack = LIFO (undo, call stack, DFS), Queue = FIFO (job processing, BFS, rate limiting). Ambele au operații O(1) cu implementarea corectă, deci diferența nu e de performanță ci de semantică.
Întrebarea 5 din 5 Ce clasă PHP nativă oferă o implementare eficientă O(1) pentru Queue?
Corect. SplQueue din Standard PHP Library este implementată intern ca o listă dublu înlănțuită — enqueue și dequeue sunt ambele O(1), indiferent de dimensiune. Este soluția recomandată pentru producție cu volume mari.
