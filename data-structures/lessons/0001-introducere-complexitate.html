Introducere și Complexitate

Ce sunt structurile de date, de ce contează alegerea lor și cum măsurăm eficiența cu notația Big O.

Ce este o structură de date?

O structură de date este o modalitate organizată de a stoca și accesa date în memorie. Alegerea structurii potrivite poate face diferența dintre un program care rulează în 1 ms și unul care rulează în 10 minute pe același set de date.

ARR

Array (tablou)

Colecție ordonată de elemente, acces prin index. Rapid la citire, lent la inserare la mijloc.

O(1) citire
STK

Stack / Queue

Colecții cu acces restricționat — LIFO sau FIFO. Simplu de implementat, esențial în algoritmi.

O(1) push/pop
MAP

Hash Map

Perechi cheie-valoare cu acces ultra-rapid. PHP arrays sunt internaly hash maps.

O(1) căutare
TRE

Arbori și Grafuri

Structuri ierarhice și relaționale — baze de date, rute, sisteme de fișiere, rețele sociale.

O(log n) BST

Notația Big O — măsurarea eficienței

Big O descrie cum crește timpul de execuție sau memoria în funcție de mărimea input-ului n. Nu măsoară secunde — măsoară ritmul de creștere.

// O(1) — constant: nu contează câte elemente are array-ul
$first = $arr[0];           // PHP
const first = arr[0];       // JS

// O(n) — linear: parcurgem fiecare element o dată
foreach ($arr as $item) {   // PHP
    echo $item;
}
arr.forEach(item => console.log(item));  // JS

// O(n²) — pătratic: for în for (evită pe date mari!)
for ($i = 0; $i < count($arr); $i++) {     // PHP
    for ($j = 0; $j < count($arr); $j++) {
        // ...
    }
}
Regulă practică
O(n²) devine problematic după ~10 000 elemente. Pe 100 000 elemente, un algoritm O(n²) face 10 miliarde de operații — câteva minute de execuție. Un O(n log n) face doar ~1.7 milioane.

Complexitate spațială vs temporală

// Complexitate temporală O(n), spațială O(1) — nu alocăm memorie extra
function findMax(array $arr): int {
    $max = PHP_INT_MIN;
    foreach ($arr as $val) {
        if ($val > $max) $max = $val;
    }
    return $max;
}

// Complexitate temporală O(n), spațială O(n) — alocăm un array nou
function doubleAll(array $arr): array {
    return array_map(fn($x) => $x * 2, $arr);
    // array nou de aceeași dimensiune = O(n) spațiu
}

Există adesea un trade-off între timp și spațiu: poți cumpăra viteză cu memorie (memoization, cache) sau poți economisi memorie cu algoritmi mai lenți.

  2. O funcție care caută un element printr-o buclă simplă — O(?)

Verifică-ți înțelegerea

Întrebarea 1 din 5 Ce descrie notația Big O?
Corect. Big O descrie comportamentul asimptotic — cum crește complexitatea pe măsură ce n tinde spre infinit. Nu depinde de hardware sau de constantele mici; se concentrează pe termenul dominant.
Întrebarea 2 din 5 Care este complexitatea accesului la un element dintr-un array prin index?
Corect. Array-urile stochează elemente în memorie contiguă. Adresa elementului la index i este adresa_baza + i * dimensiune_element — calculat direct, indiferent de mărimea array-ului.
Întrebarea 3 din 5 Un algoritm O(n²) pe 1000 elemente face aproximativ câte operații?
Corect. O(n²) cu n=1000 înseamnă 1000² = 1 000 000 operații. Acesta este motivul pentru care algoritmii pătratici (bubble sort, selection sort) devin impractici pe seturi mari de date.
Întrebarea 4 din 5 Ce este trade-off-ul clasic între complexitate temporală și spațială?
Corect. Memoization (cache pentru rezultate calculate) reduce complexitatea temporală de la O(2ⁿ) la O(n) dar crește spațiul la O(n). Inversul: poți recalcula în loc să stochezi, economisind memorie cu costul timpului.
Întrebarea 5 din 5 Array-urile PHP sunt internaly ce structură de date?
Corect. Array-urile PHP sunt hash map-uri ordonate care suportă și chei întregi (comportament de array) și chei string (comportament de dicționar). Aceasta le face extrem de flexibile dar și ușor mai lente decât array-urile fixe din C sau Java.
