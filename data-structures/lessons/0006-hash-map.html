Hash Map (Dicționar)

Căutare, inserare și ștergere în O(1) — cea mai folosită structură de date în aplicații web. PHP arrays și JS objects sunt hash maps.

Cum funcționează un Hash Map

Un Hash Map transformă o cheie în un index de array printr-o funcție hash. Indexul indică "bucket-ul" unde e stocată valoarea — acces direct, O(1): 

hash("name")  → 3  → bucket[3] = "Paul"
hash("age")   → 7  → bucket[7] = 30
hash("email") → 1  → bucket[1] = "paul@example.com"

get("name") → hash("name") = 3 → bucket[3] → "Paul"  // O(1)

Coliziuni

Două chei diferite pot produce același hash — coliziune. Soluții comune: chaining (fiecare bucket e o listă înlănțuită) sau open addressing (caută un bucket liber). PHP folosește chaining; cu un factor de încărcare mic, coliziunile sunt rare.

Hash Map în PHP — arrays asociative

// PHP arrays sunt hash maps ordonate native
$user = [
    'name'  => 'Paul',
    'age'   => 30,
    'email' => 'paul@example.com',
];

// Acces — O(1)
echo $user['name'];          // Paul
echo $user['nonexistent'] ?? 'default';  // null-safe

// Inserare / actualizare — O(1)
$user['city'] = 'Bucharest';

// Ștergere — O(1)
unset($user['age']);

// Verificare existență cheie — O(1)
if (isset($user['email'])) { /* ... */ }
if (array_key_exists('email', $user)) { /* ... */ }  // mai precis (detectează null)

// Parcurgere — O(n)
foreach ($user as $key => $value) {
    echo "$key: $value\n";
}

// Cheie calculată — memoization
function fibonacci(int $n, array &$memo = []): int {
    if ($n <= 1) return $n;
    if (isset($memo[$n])) return $memo[$n];   // O(1) cache lookup
    return $memo[$n] = fibonacci($n - 1, $memo) + fibonacci($n - 2, $memo);
}

echo fibonacci(50);  // instant — fără memo ar dura ani

Hash Map în JavaScript — Object și Map

// Object — hash map simplu (chei string/Symbol)
const user = {
    name:  'Paul',
    age:   30,
    email: 'paul@example.com',
};
user.city = 'Bucharest';     // inserare O(1)
delete user.age;             // ștergere O(1)
'email' in user;             // verificare O(1)

// Map — hash map adevărat (orice tip de cheie)
const map = new Map();
map.set('name', 'Paul');      // O(1)
map.set(42, 'cheie numerică');
map.set({id: 1}, 'cheie obiect');

map.get('name');              // 'Paul' — O(1)
map.has('name');              // true  — O(1)
map.delete('name');           // O(1)
map.size;                     // numărul de intrări

// Iterare Map — păstrează ordinea inserării
for (const [key, value] of map) {
    console.log(key, value);
}

// Set — colecție de valori unice
const set = new Set([1, 2, 3, 2, 1]);
console.log([...set]);        // [1, 2, 3]
set.has(2);                   // true — O(1)
set.add(4);
set.delete(1);
Object vs Map în JavaScript
Folosește Object pentru date structurate cu chei string cunoscute (config, DTO, JSON). Folosește Map când: cheile nu sunt string-uri, adaugi/ștergi frecvent, ai nevoie de .size direct, sau ordinea inserării contează și vrei garantare explicită.

Aplicații clasice

1. Frecvența elementelor — O(n)

// PHP
function countFrequency(array $arr): array {
    $freq = [];
    foreach ($arr as $val) {
        $freq[$val] = ($freq[$val] ?? 0) + 1;
    }
    return $freq;
}
$words = ['ana', 'are', 'mere', 'ana', 'are', 'ana'];
print_r(countFrequency($words));
// ['ana' => 3, 'are' => 2, 'mere' => 1]

// JS
function countFrequency(arr) {
    return arr.reduce((map, val) => {
        map.set(val, (map.get(val) ?? 0) + 1);
        return map;
    }, new Map());
}

2. Two Sum — O(n) cu hash map vs O(n²) brute force

// Găsește doi indecși cu suma = target
function twoSum(array $nums, int $target): array {
    $seen = [];  // valoare => index
    foreach ($nums as $i => $num) {
        $complement = $target - $num;
        if (isset($seen[$complement])) {
            return [$seen[$complement], $i];
        }
        $seen[$num] = $i;
    }
    return [];
}
print_r(twoSum([2, 7, 11, 15], 9));  // [0, 1] — 2+7=9

3. Grouping / indexare — O(n)

// Grupează useri după țară
function groupBy(array $users, string $key): array {
    $groups = [];
    foreach ($users as $user) {
        $groups[$user[$key]][] = $user;
    }
    return $groups;
}

$users = [
    ['name' => 'Ana',  'country' => 'RO'],
    ['name' => 'John', 'country' => 'US'],
    ['name' => 'Paul', 'country' => 'RO'],
];
$byCountry = groupBy($users, 'country');
// ['RO' => [Ana, Paul], 'US' => [John]]

Verifică-ți înțelegerea

Întrebarea 1 din 5 Ce este o coliziune într-un Hash Map?
Corect. Hash-ul transformă chei de orice dimensiune în indici dintr-un spațiu finit — inevitabil, chei diferite pot produce același index. Soluția clasică (chaining) stochează o listă de perechi (cheie, valoare) în fiecare bucket; la acces se compară cheile exact.
Întrebarea 2 din 5 De ce array_key_exists() e mai precis decât isset() în PHP?
Corect. $arr = ['key' => null] — isset($arr['key']) returnează false (valoarea e null), dar array_key_exists('key', $arr) returnează true (cheia există). Când null e o valoare legitimă, folosești array_key_exists().
Întrebarea 3 din 5 De ce algoritmul Two Sum cu Hash Map este O(n) față de O(n²) brute force?
Corect. Brute force: pentru fiecare element, caută complementul în toată lista — O(n) × O(n) = O(n²). Cu Hash Map: stochezi elementele văzute; la fiecare element nou, verifici complementul în O(1). Un singur parcurs = O(n) total.
Întrebarea 4 din 5 Când preferi Map față de Object în JavaScript?
Corect. Map acceptă orice tip ca cheie (obiecte, numere, funcții), expune .size direct, garantează ordinea inserării și e optimizată pentru operații frecvente de add/delete. Object e mai convenabil pentru date JSON-like cu chei string cunoscute. Notă: Map nu se serializează cu JSON.stringify.
Întrebarea 5 din 5 Ce structură de date folosești pentru a elimina duplicatele dintr-un array eficient?
Corect. Set stochează numai valori unice prin definiție. [...new Set(arr)] în JS sau iterare cu $seen[$val] = true în PHP elimină duplicatele în O(n) cu O(n) spațiu extra. Alternativa sortare+comparare e O(n log n) dar O(1) spațiu.
