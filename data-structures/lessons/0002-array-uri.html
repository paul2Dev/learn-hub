Array-uri și Operații

Cea mai fundamentală structură de date — cum funcționează în memorie, operațiile esențiale și diferențele dintre PHP și JavaScript.

Cum funcționează un array în memorie

Un array clasic stochează elemente în memorie contiguă (bloc continuu). Știind adresa primului element și dimensiunea fiecăruia, calculezi adresa oricărui element în O(1): 

// Element la index i:
adresa = adresa_baza + (i × dimensiune_element)

// Ex: int de 4 bytes, adresă bază 1000
// arr[0] → 1000
// arr[1] → 1004
// arr[5] → 1020  (indiferent câte elemente are array-ul)

PHP și JavaScript au array-uri dinamice — redimensionabile automat. Internaly, când se umple capacitatea, se alocă un bloc nou (de obicei dublul), datele se copiază, iar vechiul bloc se eliberează. Aceasta face append amortizat O(1).

Operații esențiale — PHP

// Creare
$arr = [1, 2, 3, 4, 5];
$assoc = ['name' => 'Paul', 'age' => 30];

// Citire
$first = $arr[0];                  // 1
$name  = $assoc['name'];           // 'Paul'

// Adăugare la final — O(1)
$arr[] = 6;
array_push($arr, 7, 8);

// Adăugare la început — O(n) — deplasează tot!
array_unshift($arr, 0);

// Ștergere la final — O(1)
$last = array_pop($arr);

// Ștergere la început — O(n)
$first = array_shift($arr);

// Inserare la poziție — O(n)
array_splice($arr, 2, 0, [99]);    // inserează 99 la index 2

// Căutare liniară — O(n)
$pos = array_search(99, $arr);     // returnează index sau false

// Slice (subset) — O(k) unde k = lungimea slice-ului
$subset = array_slice($arr, 1, 3); // elemente de la index 1, lungime 3

// Sortare — O(n log n)
sort($arr);
usort($arr, fn($a, $b) => $a - $b);

Operații esențiale — JavaScript

// Creare
const arr = [1, 2, 3, 4, 5];

// Citire
const first = arr[0];    // 1
const last  = arr.at(-1); // 5  (ES2022 — index negativ)

// Adăugare la final — O(1)
arr.push(6, 7);

// Adăugare la început — O(n)
arr.unshift(0);

// Ștergere la final — O(1)
const popped = arr.pop();

// Ștergere la început — O(n)
const shifted = arr.shift();

// Inserare / ștergere la poziție — O(n)
arr.splice(2, 0, 99);     // inserează 99 la index 2
arr.splice(2, 1);         // șterge 1 element la index 2

// Căutare — O(n)
const idx = arr.indexOf(99);
const found = arr.find(x => x > 3);

// Slice (nu modifică array-ul original) — O(k)
const subset = arr.slice(1, 4);   // elemente de la 1 la 3 (exclusiv 4)

// Sortare — O(n log n)
arr.sort((a, b) => a - b);        // IMPORTANT: comparator numeric!

// Transformări funcționale (returnează array nou)
const doubled  = arr.map(x => x * 2);       // O(n)
const evens    = arr.filter(x => x % 2 === 0); // O(n)
const sum      = arr.reduce((acc, x) => acc + x, 0); // O(n)
Atenție — JavaScript sort implicit
arr.sort() fără comparator convertește elementele la string înainte de a le compara. [10, 9, 2].sort() returnează [10, 2, 9] — nu [2, 9, 10]. Folosește mereu sort((a, b) => a - b) pentru numere.

Căutare binară — O(log n) pe array sortat

// PHP
function binarySearch(array $arr, int $target): int {
    $left  = 0;
    $right = count($arr) - 1;

    while ($left <= $right) {
        $mid = intdiv($left + $right, 2);
        if ($arr[$mid] === $target)  return $mid;
        if ($arr[$mid] < $target)    $left  = $mid + 1;
        else                         $right = $mid - 1;
    }
    return -1;
}

// JS
function binarySearch(arr, target) {
    let left = 0, right = arr.length - 1;
    while (left <= right) {
        const mid = Math.floor((left + right) / 2);
        if (arr[mid] === target) return mid;
        if (arr[mid] < target)   left  = mid + 1;
        else                     right = mid - 1;
    }
    return -1;
}

Verifică-ți înțelegerea

Întrebarea 1 din 5 De ce inserarea unui element la mijlocul unui array este O(n)?
Corect. Memoria contiguă înseamnă că nu poți "face loc" fără a deplasa elementele existente. Dacă inserezi la index 0, toate n elemente se mută cu o poziție — O(n) operații.
Întrebarea 2 din 5 Ce returnează [10, 9, 2].sort() în JavaScript fără comparator?
Corect. .sort() convertește elementele la string: "10" < "2" lexicografic (pentru că "1" < "2"). Rezultatul este [10, 2, 9]. Folosește mereu .sort((a, b) => a - b) pentru numere.
Întrebarea 3 din 5 Căutarea binară are o condiție obligatorie — care este aceasta?
Corect. Căutarea binară funcționează prin eliminarea jumătății din spațiul de căutare la fiecare pas, bazându-se pe faptul că dacă elementul din mijloc e mai mic decât target, tot ce e la stânga e irelevant. Fără sortare, această presupunere nu mai e validă.
Întrebarea 4 din 5 Care este diferența dintre slice() și splice() în JavaScript?
Corect. slice(start, end) = non-destructiv, returnează un sub-array nou. splice(start, deleteCount, ...items) = destructiv, modifică array-ul original, returnează elementele șterse. Confuzia dintre ele este o sursă frecventă de bug-uri.
Întrebarea 5 din 5 De ce append la finalul unui array dinamic e considerat O(1) amortizat?
Corect. Când capacitatea se umple, se alocă un bloc de 2× mai mare și se copiază datele — costul e O(n). Dar asta se întâmplă rar: după n append-uri ieftine, o singură copiere costă O(n). Costul total al n operații = O(n), deci per operație = O(1) amortizat.
