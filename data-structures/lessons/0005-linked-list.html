Linked List (Listă Înlănțuită)

O colecție de noduri conectate prin pointeri — inserare și ștergere O(1) la extremități, dar fără acces aleator.

Structura unui nod

O listă înlănțuită este formată din noduri. Fiecare nod conține: o valoare și un pointer (referință) către nodul următor. Ultimul nod are pointer-ul null. 

Node(1) → Node(2) → Node(3) → Node(4) → null
  ↑ head                        ↑ tail

Singly Linked List:   fiecare nod știe numai nodul următor
Doubly Linked List:   fiecare nod știe nodul anterior ȘI cel următor
                      null ← Node(1) ↔ Node(2) ↔ Node(3) → null

Array vs Linked List — comparație

OperațieArrayLinked List
Acces prin indexO(1)O(n) — trebuie parcurs
Inserare la începutO(n) — deplasareO(1) — schimb de pointer
Inserare la finalO(1) amortizatO(1) cu pointer tail
Inserare la mijlocO(n)O(n) căutare + O(1) inserare
Ștergere la începutO(n) — deplasareO(1) — schimb de pointer
MemorieContiguă, eficientăExtra memorie pentru pointeri
Cache performanceExcelent (localism)Slab (noduri împrăștiate)

Implementare în PHP

class Node {
    public function __construct(
        public mixed $value,
        public ?Node $next = null
    ) {}
}

class LinkedList {
    private ?Node $head = null;
    private ?Node $tail = null;
    private int   $size = 0;

    // Adaugă la final — O(1)
    public function append(mixed $value): void {
        $node = new Node($value);
        if ($this->tail === null) {
            $this->head = $this->tail = $node;
        } else {
            $this->tail->next = $node;
            $this->tail = $node;
        }
        $this->size++;
    }

    // Adaugă la început — O(1)
    public function prepend(mixed $value): void {
        $node = new Node($value, $this->head);
        $this->head = $node;
        if ($this->tail === null) $this->tail = $node;
        $this->size++;
    }

    // Șterge primul nod — O(1)
    public function removeFirst(): mixed {
        if ($this->head === null) throw new UnderflowException('List is empty');
        $value = $this->head->value;
        $this->head = $this->head->next;
        if ($this->head === null) $this->tail = null;
        $this->size--;
        return $value;
    }

    // Caută o valoare — O(n)
    public function contains(mixed $value): bool {
        $current = $this->head;
        while ($current !== null) {
            if ($current->value === $value) return true;
            $current = $current->next;
        }
        return false;
    }

    // Conversie la array — O(n)
    public function toArray(): array {
        $result  = [];
        $current = $this->head;
        while ($current !== null) {
            $result[] = $current->value;
            $current  = $current->next;
        }
        return $result;
    }

    public function size(): int { return $this->size; }
}

$list = new LinkedList();
$list->append(1);
$list->append(2);
$list->append(3);
$list->prepend(0);
print_r($list->toArray());   // [0, 1, 2, 3]
echo $list->removeFirst();   // 0

Implementare în JavaScript

class Node {
    constructor(value, next = null) {
        this.value = value;
        this.next  = next;
    }
}

class LinkedList {
    #head = null;
    #tail = null;
    #size = 0;

    append(value) {
        const node = new Node(value);
        if (!this.#tail) {
            this.#head = this.#tail = node;
        } else {
            this.#tail.next = node;
            this.#tail = node;
        }
        this.#size++;
    }

    prepend(value) {
        const node = new Node(value, this.#head);
        this.#head = node;
        if (!this.#tail) this.#tail = node;
        this.#size++;
    }

    removeFirst() {
        if (!this.#head) throw new Error('List is empty');
        const value = this.#head.value;
        this.#head  = this.#head.next;
        if (!this.#head) this.#tail = null;
        this.#size--;
        return value;
    }

    contains(value) {
        let current = this.#head;
        while (current) {
            if (current.value === value) return true;
            current = current.next;
        }
        return false;
    }

    toArray() {
        const result = [];
        let current  = this.#head;
        while (current) {
            result.push(current.value);
            current = current.next;
        }
        return result;
    }

    get size() { return this.#size; }
}

const list = new LinkedList();
list.append(1); list.append(2); list.append(3);
list.prepend(0);
console.log(list.toArray());    // [0, 1, 2, 3]
console.log(list.removeFirst()); // 0

Inversarea unei liste înlănțuite

// Algoritmul clasic cu 3 pointeri — O(n) timp, O(1) spațiu
public function reverse(): void {
    $prev    = null;
    $current = $this->head;
    $this->tail = $this->head;  // capul devine coada

    while ($current !== null) {
        $next          = $current->next;  // salvăm următorul
        $current->next = $prev;           // inversăm pointerul
        $prev          = $current;        // avansăm prev
        $current       = $next;           // avansăm current
    }
    $this->head = $prev;  // noul cap e fostul coadă
}
Când să folosești Linked List
Linked List e avantajoasă când: inserezi/ștergi frecvent la începutul listei, dimensiunea variază extrem de mult și nu vrei realocări, sau implementezi un Stack/Queue fără a folosi array. În PHP/JS modern, array-urile native sunt adesea mai rapide în practică datorită cache-ului CPU, chiar și pentru inserare.

  2. Implementează metoda reverse() în JavaScript
  3. Detectează dacă o listă înlănțuită are un ciclu (Floyd's Cycle Detection — doi pointeri, unul rapid și unul lent)
  4. Găsește al N-lea nod de la final fără a cunoaște lungimea listei

Verifică-ți înțelegerea

Întrebarea 1 din 5 De ce accesul prin index în o Linked List este O(n)?
Corect. Într-un array, adresa elementului se calculează direct: baza + i * size. Într-o Linked List, nu există adresă directă — trebuie să urmezi pointerii nod cu nod de la head: head → next → next → ... de i ori.
Întrebarea 2 din 5 Care este avantajul principal al Linked List față de un Array?
Corect. Schimbarea pointerilor la head sau tail este O(1) și nu deplasează niciun alt element. Un array trebuie să mute toate elementele la inserare/ștergere la început — O(n). Acesta e trade-off-ul principal: acces lent vs modificare rapidă la extremități.
Întrebarea 3 din 5 Câți pointeri modifici la ștergerea primului nod (removeFirst) dintr-o Singly Linked List?
Corect. Muți head să pointeze la al doilea nod. Vechiul prim nod nu mai are referințe și va fi colectat de Garbage Collector. Cazul special: dacă lista devine goală după ștergere, trebuie să setezi și tail = null.
Întrebarea 4 din 5 Ce avantaj adaugă o Doubly Linked List față de o Singly Linked List?
Corect. Cu pointer prev, poți șterge nodul tail în O(1) — actualizezi tail.prev.next = null și muți tail. Fără el, trebuie să parcurgi toată lista pentru a găsi penultimul nod — O(n). Costul: memorie dublă pentru pointeri.
Întrebarea 5 din 5 Algoritmul Floyd's Cycle Detection folosește două pointeri cu viteze diferite. De ce funcționează?
Corect. Pointerul lent avansează 1 nod/pas, cel rapid 2. Fără ciclu, rapid ajunge la null. Cu ciclu, ambii intră în el și se rotesc — rapid câștigă tur după tur și în final rapid = lent. Nici un nod nu e marcat — O(1) spațiu.
