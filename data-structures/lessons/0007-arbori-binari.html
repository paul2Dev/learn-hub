Arbori Binari și BST

Structuri ierarhice — Binary Search Tree, traversal în 3 moduri și aplicații în baze de date, sisteme de fișiere și expresii matematice.

Terminologie

           10          ← rădăcină (root)
          /  \
         5    15       ← noduri interne
        / \     \
       3   7    20     ← frunze (leaves)

- Nod (Node): elementul de bază cu o valoare
- Rădăcină (Root): nodul de sus, fără părinte
- Frunză (Leaf): nod fără copii
- Adâncime (Depth): distanța de la rădăcină
- Înălțime (Height): cea mai lungă cale de la rădăcină la frunză
- Subarbore (Subtree): orice nod cu descendenții lui
- Arbore binar: fiecare nod are maxim 2 copii (stâng și drept)

Binary Search Tree (BST)

Un BST respectă invarianta: pentru orice nod, toți copiii stângi au valori mai mici și toți copiii drepți au valori mai mari. Aceasta permite căutare în O(log n) pe un arbore echilibrat. 

           10
          /  \
         5    15      Căutăm 7:
        / \     \     - 7 < 10 → mergi stânga
       3   7    20    - 7 > 5  → mergi dreapta
                      - 7 = 7  → găsit! (3 comparații vs 6 linear)

Implementare în PHP

class BSTNode {
    public function __construct(
        public int      $value,
        public ?BSTNode $left  = null,
        public ?BSTNode $right = null
    ) {}
}

class BST {
    private ?BSTNode $root = null;

    // Inserare — O(log n) mediu, O(n) worst case (arbore degenerat)
    public function insert(int $value): void {
        $this->root = $this->insertNode($this->root, $value);
    }

    private function insertNode(?BSTNode $node, int $value): BSTNode {
        if ($node === null) return new BSTNode($value);
        if ($value < $node->value)      $node->left  = $this->insertNode($node->left,  $value);
        elseif ($value > $node->value)  $node->right = $this->insertNode($node->right, $value);
        return $node;  // valori duplicate ignorate
    }

    // Căutare — O(log n)
    public function search(int $value): bool {
        return $this->searchNode($this->root, $value);
    }

    private function searchNode(?BSTNode $node, int $value): bool {
        if ($node === null)              return false;
        if ($value === $node->value)     return true;
        if ($value < $node->value)       return $this->searchNode($node->left,  $value);
        return $this->searchNode($node->right, $value);
    }

    // Traversal In-Order (stânga, rădăcină, dreapta) → returnează sortat!
    public function inOrder(): array {
        $result = [];
        $this->inOrderTraversal($this->root, $result);
        return $result;
    }

    private function inOrderTraversal(?BSTNode $node, array &$result): void {
        if ($node === null) return;
        $this->inOrderTraversal($node->left, $result);
        $result[] = $node->value;
        $this->inOrderTraversal($node->right, $result);
    }
}

$bst = new BST();
foreach ([10, 5, 15, 3, 7, 20] as $val) $bst->insert($val);
print_r($bst->inOrder());   // [3, 5, 7, 10, 15, 20] — sortat!
var_dump($bst->search(7));  // true
var_dump($bst->search(9));  // false

Traversal — cele 3 moduri

// Arborele:    10
//             /  \
//            5    15
//           / \
//          3   7

// In-Order   (L → Root → R): 3, 5, 7, 10, 15  ← returnează valorile sortate
// Pre-Order  (Root → L → R): 10, 5, 3, 7, 15  ← util pentru copiere/serializare arbore
// Post-Order (L → R → Root): 3, 7, 5, 15, 10  ← util pentru ștergere (ștergi copiii înainte de părinte)

// JavaScript — in-order iterativ cu stack (fără recursivitate)
function inOrderIterative(root) {
    const result = [];
    const stack  = [];
    let current  = root;

    while (current || stack.length) {
        while (current) {
            stack.push(current);
            current = current.left;
        }
        current = stack.pop();
        result.push(current.value);
        current = current.right;
    }
    return result;
}

Înălțimea unui arbore și echilibrul

// Înălțimea — O(n)
function height(?BSTNode $node): int {
    if ($node === null) return -1;
    return 1 + max(height($node->left), height($node->right));
}

BST în practică
Nu implementezi de obicei BST de la zero — bazele de date folosesc B-Tree (variantă), motoarele JavaScript folosesc arbori interni pentru obiecte. Valoarea exercițiului BST este înțelegerea recursivității, traversalului și a complexității O(log n) pe structuri ierarhice.

Verifică-ți înțelegerea

Întrebarea 1 din 5 Care este invarianta unui Binary Search Tree?
Corect. Invarianta BST: pentru orice nod N, toate nodurile din subarborele stâng au valori < N.value, toate din subarborele drept au valori > N.value. Această proprietate permite căutarea binară — eliminăm jumătate din arbore la fiecare pas.
Întrebarea 2 din 5 Ce returnează traversalul In-Order pe un BST valid?
Corect. In-Order (Stânga → Rădăcină → Dreapta) pe un BST produce elementele în ordine ascendentă. Aceasta este o proprietate cheie: BST In-Order = array sortat. De aceea indexurile B-Tree din bazele de date permit scanare în ordine eficientă.
Întrebarea 3 din 5 Când devine căutarea într-un BST O(n) în loc de O(log n)?
Corect. Inserând valorile 1, 2, 3, 4, 5 în ordine, fiecare nod nou merge la dreapta celui precedent — arborele devine o linie (liste înlănțuită). Înălțimea = n-1, căutarea = O(n). Soluția: arbori auto-echilibrați (AVL, Red-Black) care mențin O(log n).
Întrebarea 4 din 5 Traversalul Post-Order (L → R → Root) este util mai ales pentru:
Corect. Post-Order procesează copiii complet înainte de rădăcină — esențial pentru ștergere (nu poți șterge un părinte dacă copiii sunt încă referiți) și pentru evaluarea expresiilor matematice în formă de arbore (operanzii se evaluează înainte de operator).
Întrebarea 5 din 5 Care traversal folosește o Coadă (Queue) în loc de recursivitate sau Stack?
Corect. BFS (Breadth First Search) pe arbore vizitează toate nodurile de pe nivelul curent înainte de a avansa la nivelul următor. O coadă menține nodurile în ordinea corectă: adaugi copiii unui nod în coadă și procesezi în ordinea FIFO.
