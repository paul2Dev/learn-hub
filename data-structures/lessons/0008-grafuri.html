Grafuri

Noduri și muchii — rețele sociale, hărți, dependențe de pachete, rutare. BFS și DFS aplicate pe grafuri reale.

Ce este un graf?

Un graf constă din noduri (vertices) conectate prin muchii (edges). Spre deosebire de arbori, grafurile pot forma cicluri și nu au o rădăcină. 

Graf neorientat (undirected):         Graf orientat (directed):
  A --- B --- D                          A → B → D
  |     |                                ↓   ↓
  C --- E                                C → E

Graf ponderat (weighted) — muchii au costuri:
  A --3-- B --1-- D
  |               |
  2               5
  |               |
  C ------4------ E
Exemple din lumea reală
Rețele sociale — noduri: utilizatori, muchii: conexiuni.
Hărți — noduri: locații, muchii: drumuri cu distanțe (graf ponderat).
Dependențe npm/Composer — noduri: pachete, muchii: dependențe (graf orientat).
Web crawling — noduri: pagini, muchii: link-uri.

Reprezentare — Adjacency List

Cea mai comună reprezentare: un hash map unde cheia e nodul și valoarea e lista vecinilor. Eficientă pentru grafuri rare (puține muchii față de noduri): 

// PHP — Adjacency List
$graph = [
    'A' => ['B', 'C'],
    'B' => ['A', 'D', 'E'],
    'C' => ['A', 'E'],
    'D' => ['B'],
    'E' => ['B', 'C'],
];

// JS — Adjacency List cu Map
const graph = new Map([
    ['A', ['B', 'C']],
    ['B', ['A', 'D', 'E']],
    ['C', ['A', 'E']],
    ['D', ['B']],
    ['E', ['B', 'C']],
]);

// Adjacency Matrix — alternativă pentru grafuri dense
//   A  B  C  D  E
// A[0, 1, 1, 0, 0]
// B[1, 0, 0, 1, 1]
// C[1, 0, 0, 0, 1]
// D[0, 1, 0, 0, 0]
// E[0, 1, 1, 0, 0]
// Avantaj: verificare muchie O(1). Dezavantaj: O(V²) memorie

BFS — Breadth First Search

Explorează nodurile nivel cu nivel — cel mai aproape primul. Găsește cea mai scurtă cale (în număr de muchii) între două noduri. 

// PHP
function bfs(array $graph, string $start, string $target): ?array {
    $queue   = new SplQueue();
    $visited = [$start => true];
    $parent  = [];  // pentru reconstrucția căii

    $queue->enqueue($start);

    while (!$queue->isEmpty()) {
        $node = $queue->dequeue();
        if ($node === $target) return reconstructPath($parent, $start, $target);

        foreach ($graph[$node] as $neighbor) {
            if (!isset($visited[$neighbor])) {
                $visited[$neighbor] = true;
                $parent[$neighbor]  = $node;
                $queue->enqueue($neighbor);
            }
        }
    }
    return null;  // nu există cale
}

function reconstructPath(array $parent, string $start, string $target): array {
    $path = [];
    $node = $target;
    while ($node !== $start) {
        array_unshift($path, $node);
        $node = $parent[$node];
    }
    array_unshift($path, $start);
    return $path;
}

$path = bfs($graph, 'A', 'D');
print_r($path);  // ['A', 'B', 'D']

DFS — Depth First Search

Explorează cât mai adânc pe o ramură înainte de a reveni. Util pentru detectarea ciclurilor, topological sort, componente conexe. 

// JS — DFS recursiv
function dfs(graph, node, visited = new Set(), result = []) {
    visited.add(node);
    result.push(node);

    for (const neighbor of (graph.get(node) ?? [])) {
        if (!visited.has(neighbor)) {
            dfs(graph, neighbor, visited, result);
        }
    }
    return result;
}

// JS — DFS iterativ cu Stack (evită stack overflow pe grafuri adânci)
function dfsIterative(graph, start) {
    const visited = new Set();
    const stack   = [start];
    const result  = [];

    while (stack.length) {
        const node = stack.pop();
        if (visited.has(node)) continue;
        visited.add(node);
        result.push(node);

        for (const neighbor of (graph.get(node) ?? []).reverse()) {
            if (!visited.has(neighbor)) stack.push(neighbor);
        }
    }
    return result;
}

const g = new Map([
    ['A', ['B', 'C']],
    ['B', ['D', 'E']],
    ['C', []],
    ['D', []],
    ['E', []],
]);
console.log(dfs(g, 'A'));          // ['A', 'B', 'D', 'E', 'C']
console.log(dfsIterative(g, 'A')); // ['A', 'B', 'D', 'E', 'C']

Verifică-ți înțelegerea

Întrebarea 1 din 5 Care este diferența principală dintre un graf și un arbore?
Corect. Un arbore este un caz special de graf — un graf conex și aciclic cu n noduri și n-1 muchii. Grafurile generale pot forma cicluri (A → B → C → A), pot fi neconexe, orientate, ponderate etc.
Întrebarea 2 din 5 De ce BFS găsește cea mai scurtă cale (în muchii) și DFS nu?
Corect. BFS extinde "frontiera" uniform — mai întâi toți vecinii la distanța 1, apoi la distanța 2 etc. Prima dată când atingi destinația, ai parcurs calea cu cele mai puține muchii posibile. DFS poate găsi destinația pe o cale lungă înainte de a explora calea scurtă.
Întrebarea 3 din 5 Ce structură de date folosește intern algoritmul BFS?
Corect. BFS adaugă vecinii în coadă (enqueue) și îi procesează în ordinea adăugării (dequeue FIFO). Aceasta garantează că nodurile de la distanța k sunt procesate complet înainte de nodurile de la distanța k+1. DFS folosește un stack (sau recursivitate, care folosește call stack-ul).
Întrebarea 4 din 5 Care este avantajul Adjacency List față de Adjacency Matrix?
Corect. Adjacency Matrix alocă V×V spațiu indiferent de câte muchii există. Pentru un graf cu 1000 noduri și 1000 muchii (rar), Matrix aloca 1.000.000 celule, List aloca ~2000. Invers, Matrix e mai bună pentru grafuri dense și pentru verificare rapidă O(1) a existenței unei muchii.
Întrebarea 5 din 5 De ce marcăm nodurile ca "vizitate" în BFS și DFS?
Corect. Fără marcarea nodurilor vizitate, un graf cu ciclu (A → B → A → B → ...) ar face BFS/DFS să ruleze la infinit. Marcând nodurile, garantăm că fiecare nod e procesat o singură dată — complexitate O(V + E) în loc de infinit.
