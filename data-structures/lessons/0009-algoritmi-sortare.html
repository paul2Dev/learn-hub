Algoritmi de Sortare

De la Bubble Sort O(n²) la Merge Sort O(n log n) — cum funcționează, când le folosești și ce sortare folosesc PHP și JavaScript internaly.

Comparație algoritmi de sortare

AlgoritmBestAverageWorstSpațiuStabil?

Selection SortO(n²)O(n²)O(n²)O(1)Nu

Bubble Sort — O(n²)

Compară perechi adiacente și le schimbă dacă sunt în ordine greșită. "Bule" mari ajung la final.

// PHP
function bubbleSort(array $arr): array {
    $n = count($arr);
    for ($i = 0; $i < $n - 1; $i++) {
        $swapped = false;
        for ($j = 0; $j < $n - $i - 1; $j++) {
            if ($arr[$j] > $arr[$j + 1]) {
                [$arr[$j], $arr[$j + 1]] = [$arr[$j + 1], $arr[$j]];
                $swapped = true;
            }
        }
        if (!$swapped) break;  // optimizare: deja sortat
    }
    return $arr;
}

// Trace: [5, 3, 8, 1]
// Pass 1: [3,5,8,1] → [3,5,8,1] → [3,5,1,8]
// Pass 2: [3,5,1,8] → [3,1,5,8]
// Pass 3: [1,3,5,8] ← sortat

Insertion Sort — O(n²) dar excelent pe date mici sau aproape sortate

// JS — construiești array-ul sortat câte un element
function insertionSort(arr) {
    for (let i = 1; i < arr.length; i++) {
        const key = arr[i];
        let j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
    return arr;
}

// Trace: [5, 3, 8, 1]
// i=1: key=3, [5,5,8,1] → [3,5,8,1]
// i=2: key=8, nicio mutare → [3,5,8,1]
// i=3: key=1, [3,5,8,8] → [3,5,5,8] → [3,3,5,8] → [1,3,5,8]
Insertion Sort pe date aproape sortate
Insertion Sort are complexitate O(n) pe date deja sortate sau cu puține inversiuni. De aceea TimSort (folosit în JS și Python) combină Merge Sort cu Insertion Sort: aplică Insertion Sort pe segmente mici (mai eficient în practică), apoi le combină cu Merge Sort.

Merge Sort — O(n log n) garantat

Divide et impera: împarte array-ul în jumătăți, sortează recursiv fiecare jumătate, combină (merge) rezultatele sortate. 

// PHP
function mergeSort(array $arr): array {
    if (count($arr) <= 1) return $arr;

    $mid   = intdiv(count($arr), 2);
    $left  = mergeSort(array_slice($arr, 0, $mid));
    $right = mergeSort(array_slice($arr, $mid));

    return merge($left, $right);
}

function merge(array $left, array $right): array {
    $result = [];
    $i = $j = 0;

    while ($i < count($left) && $j < count($right)) {
        if ($left[$i] <= $right[$j]) {
            $result[] = $left[$i++];
        } else {
            $result[] = $right[$j++];
        }
    }
    return array_merge($result, array_slice($left, $i), array_slice($right, $j));
}

// Trace: [5, 3, 8, 1]
// Split: [5,3] și [8,1]
// Split: [5],[3] și [8],[1]
// Merge: [3,5] și [1,8]
// Merge: [1,3,5,8]

Quick Sort — O(n log n) mediu, in-place

// JS — Quick Sort cu pivot la final
function quickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        const pi = partition(arr, low, high);
        quickSort(arr, low,   pi - 1);
        quickSort(arr, pi + 1, high);
    }
    return arr;
}

function partition(arr, low, high) {
    const pivot = arr[high];  // pivot = ultimul element
    let i = low - 1;

    for (let j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            [arr[i], arr[j]] = [arr[j], arr[i]];  // swap
        }
    }
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
    return i + 1;
}

// Quick Sort e O(n²) în worst case dacă pivotul e mereu min/max
// Soluție: pivot aleatoriu sau median-of-three

Sortarea nativă — ce folosim în producție

// PHP — sort() nativ, Timsort / Introsort
$arr = [5, 3, 8, 1, 4];
sort($arr);                                    // ascendent, reindexare
rsort($arr);                                   // descendent
usort($arr, fn($a, $b) => $a - $b);            // custom comparator

// Sortare stabilă pe obiecte (PHP 8)
$users = [
    ['name' => 'Ana',  'age' => 30],
    ['name' => 'Paul', 'age' => 25],
    ['name' => 'Ion',  'age' => 30],
];
usort($users, fn($a, $b) => $a['age'] - $b['age']);
// [Paul(25), Ana(30), Ion(30)] — stabila: Ana înaintea lui Ion (ordine inițială)

// JS — .sort() cu TimSort (stabil din ES2019)
const numbers = [5, 3, 8, 1];
numbers.sort((a, b) => a - b);    // [1, 3, 5, 8]

const users2 = [{ name: 'Ana', age: 30 }, { name: 'Paul', age: 25 }];
users2.sort((a, b) => a.age - b.age);

Verifică-ți înțelegerea

Întrebarea 1 din 5 Ce înseamnă un algoritm de sortare "stabil"?
Corect. Dacă sortezi [Ana(30), Paul(25), Ion(30)] după vârstă și algoritmul e stabil, Ana rămâne înaintea lui Ion în rezultat (ambii au 30). Un algoritm instabil ar putea schimba ordinea lor. Contează când sortezi după chei multiple.
Întrebarea 2 din 5 De ce Merge Sort consumă O(n) memorie extra față de Quick Sort O(log n)?
Corect. La fiecare merge, creăm un array temporar de dimensiunea celor două jumătăți combinate — în total O(n) spațiu extra. Quick Sort rearanjează elementele în loc (in-place) prin schimburi — folosește numai O(log n) pentru stiva de recursivitate.
Întrebarea 3 din 5 Quick Sort are worst case O(n²). Când se întâmplă asta?
Corect. Dacă pivotul e mereu extremul, o partiție are 0 elemente și cealaltă n-1 — arbore de recursivitate degenerează în listă. Pe un array deja sortat cu pivot la final, asta se întâmplă la fiecare pas: O(n²). Soluție: pivot aleatoriu sau "median of three".
Întrebarea 4 din 5 Ce algorim de sortare folosește JavaScript internaly pentru .sort()?
Corect. TimSort (creat de Tim Peters pentru Python, adoptat de V8/SpiderMonkey) combină Insertion Sort pe segmente mici (eficient în practică) cu Merge Sort pentru combinare. E stabil și garantat O(n log n). PHP folosește Timsort din PHP 8.0 (anterior Introsort).
Întrebarea 5 din 5 Pe ce tip de input excelează Insertion Sort față de Merge Sort?
Corect. Insertion Sort are O(n) pe date deja sortate (nicio inversiune) și performanță excelentă pe arrays mici (<~32 elemente) datorită constantei mici și localism la cache. De aceea TimSort îl folosește pentru segmente mici — combinând avantajele ambilor algoritmi.
