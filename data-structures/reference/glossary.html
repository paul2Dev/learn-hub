referință Data Structures — Glosar

Glosar Data Structures

Termenii esențiali din cursul de structuri de date — definiții concise pentru PHP și JavaScript.

Complexitate și analiză

Big O Notation

Notație matematică pentru descrierea comportamentului unui algoritm la scală — câte operații face în funcție de dimensiunea inputului (n). Descrie worst case: O(1) constant, O(log n) logaritmic, O(n) linear, O(n log n) linearitmic, O(n²) pătratic.

→ Lecția 1
Space Complexity

Cantitatea de memorie extra folosită de un algoritm, exprimată în Big O. Un algoritm in-place folosește O(1) spațiu (nu alocă structuri proporționale cu n). Merge Sort folosește O(n) pentru array-urile temporare de combinare.

→ Lecția 1, Lecția 9
Time Complexity

Numărul de operații executate de un algoritm în funcție de dimensiunea inputului. Distinge între best case (Ω), average case (Θ) și worst case (O). În practică, worst case e cel mai relevant pentru estimarea performanței.

→ Lecția 1
Amortized O(1)

Complexitate medie pe o secvență de operații. Array append e "amortized O(1)": ocazional declanșează un resize O(n), dar rar suficient încât media pe toate operațiile rămâne O(1).

→ Lecția 2

Structuri liniare

Array

Colecție de elemente stocate în memorie contiguă. Acces aleatoriu O(1) prin index. Insert/delete la mijloc O(n) — trebuie mutat tot ce urmează. PHP arrays sunt hash maps ordonate; JS arrays sunt obiecte cu indici numerici.

→ Lecția 2
Stack (Stivă)

Structură LIFO (Last In, First Out): ultimul element adăugat e primul eliminat. Operații: push (adaugă), pop (elimină vârful), peek (citește vârful). Folosit în call stack, undo/redo, evaluarea expresiilor. Implementat cu array în PHP și JS.

→ Lecția 3
Queue (Coadă)

Structură FIFO (First In, First Out): primul element adăugat e primul eliminat. Operații: enqueue (adaugă la final), dequeue (elimină din față). Folosit în job queues, BFS, rate limiting. PHP: SplQueue; JS: implementat cu index de head pentru O(1).

→ Lecția 4
Linked List

Colecție de noduri unde fiecare nod stochează valoarea și un pointer la nodul următor. Nu există index — acces O(n). Insert/delete la extremități O(1). Singly linked: pointer next. Doubly linked: next și prev.

→ Lecția 5
Nod (Node)

Unitatea de bază a structurilor înlănțuite (Linked List, arbori, grafuri). Stochează o valoare și unul sau mai mulți pointeri spre alte noduri. În PHP/JS implementat ca obiect/clasă cu proprietăți.

→ Lecția 5, Lecția 7
Floyd's Cycle Detection

Algoritmul "turtle and hare": doi pointeri cu viteze diferite (slow +1, fast +2) detectează un ciclu într-o Linked List în O(n) timp și O(1) spațiu. Dacă se întâlnesc, există ciclu.

→ Lecția 5

Hash Maps și seturi

Hash Map (Dicționar)

Structură care mapează chei la valori printr-o funcție hash. Operații în O(1) mediu: get, set, delete, has. PHP arrays sunt hash maps native. JS: Object (chei string), Map (orice cheie), Set (valori unice).

→ Lecția 6
Funcție Hash

Funcție care transformă o cheie de dimensiune arbitrară într-un index dintr-un spațiu finit. O funcție hash bună distribuie cheile uniform pentru a minimiza coliziunile. Determinist: aceeași cheie → același index.

→ Lecția 6
Coliziune (Hash Collision)

Când două chei diferite produc același hash index. Rezolvat prin chaining (bucket = listă înlănțuită cu mai multe perechi cheie-valoare) sau open addressing (caută alt bucket liber). PHP folosește chaining.

→ Lecția 6
Memoization

Tehnică de caching a rezultatelor funcțiilor cu aceleași argumente. Implementat cu hash map: dacă rezultatul pentru argumentele date există în cache, returnează-l direct fără recalcul. Transformă Fibonacci recursiv din O(2ⁿ) în O(n).

→ Lecția 6

Arbori

Binary Search Tree (BST)

Arbore binar cu invarianta: toți copiii stângi au valori mai mici, toți copiii drepți valori mai mari. Permite căutare O(log n) pe arbore echilibrat. In-Order traversal returnează elementele sortate.

→ Lecția 7
Traversal In-Order

Traversare BST în ordinea: subarbore stâng → rădăcină → subarbore drept. Returnează valorile în ordine sortată crescătoare. Poate fi implementat recursiv sau iterativ cu un Stack.

→ Lecția 7
Traversal Pre-Order

Traversare în ordinea: rădăcină → subarbore stâng → subarbore drept. Util pentru serializarea/copierea structurii arborelui — procesezi rădăcina înainte de copii.

→ Lecția 7
Traversal Post-Order

Traversare în ordinea: subarbore stâng → subarbore drept → rădăcină. Util pentru ștergerea arborelui (ștergi copiii înainte de părinte) și evaluarea expresiilor matematice.

→ Lecția 7
Arbore Degenerat

BST unde toate nodurile au un singur copil — degenerat în listă înlănțuită. Se produce dacă inserezi valori în ordine sortată. Înălțime O(n), căutare O(n). Rezolvat prin arbori auto-echilibrați (AVL, Red-Black).

→ Lecția 7

Grafuri

Graf (Graph)

Structură de date formată din noduri (vertices) conectate prin muchii (edges). Poate fi orientat (directed — muchiile au direcție) sau neorientat, ponderat (muchiile au cost) sau neponderat. Poate conține cicluri.

→ Lecția 8
Adjacency List

Reprezentare graf: hash map unde cheia e nodul și valoarea e lista vecinilor. Spațiu O(V+E) — eficient pentru grafuri rare. Traversal vecini O(grad nod). Alternativă: Adjacency Matrix O(V²) spațiu, verificare muchie O(1).

→ Lecția 8
BFS — Breadth First Search

Traversal graf nivel cu nivel folosind o Queue. Găsește cea mai scurtă cale (în nr. muchii) între două noduri. Complexitate O(V+E). Nodurile vizitate sunt marcate pentru a evita ciclurile infinite.

→ Lecția 8
DFS — Depth First Search

Traversal graf adânc pe o ramură înainte de a explora altele, folosind Stack sau recursivitate. Util pentru detectarea ciclurilor, componente conexe, topological sort. Complexitate O(V+E).

→ Lecția 8

Algoritmi de sortare

Sortare stabilă

Algoritm de sortare care păstrează ordinea relativă a elementelor cu valori egale din input. Merge Sort și Insertion Sort sunt stabile. Quick Sort nu e stabil în implementarea clasică. JS .sort() (TimSort) e stabil din ES2019.

→ Lecția 9
Merge Sort

Algoritm divide-et-impera: împarte array-ul în jumătăți recursiv, sortează fiecare, combină (merge) rezultatele. O(n log n) garantat în toate cazurile. Stabil. Dezavantaj: O(n) spațiu extra pentru array-urile temporare de combinare.

→ Lecția 9
Quick Sort

Alege un pivot, partiționează array-ul în elemente mai mici și mai mari, sortează recursiv fiecare partiție. In-place, O(log n) spațiu. O(n log n) mediu, O(n²) worst case (pivot extremal). PHP sort() și JS .sort() folosesc variante hibride.

→ Lecția 9
TimSort

Algoritm hybrid Merge Sort + Insertion Sort — aplică Insertion Sort pe segmente mici, combină cu Merge Sort. O(n log n) garantat, stabil. Folosit de Python, JavaScript (V8/SpiderMonkey) și PHP 8+. Performanță excelentă pe date parțial sortate.

→ Lecția 9
Insertion Sort

Construiește array-ul sortat câte un element: ia elementul curent și îl inserează la poziția corectă în porțiunea deja sortată. O(n²) mediu, O(n) best case (date deja sortate). Excelent pe array-uri mici sau aproape sortate.

→ Lecția 9
