Glosar Next.js

Termenii esențiali din cursul Next.js — definiții concise pentru referință rapidă.

Arhitectură și randare

App Router

Sistemul modern de routing al Next.js (v13+) bazat pe directorul app/. Suportă Server Components, layouts nested și fișiere speciale (page.tsx, layout.tsx, loading.tsx). Înlocuiește Pages Router.

→ Lecția 1, Lecția 2
CSR — Client-Side Rendering

Randare în browser. Serverul trimite HTML minim; JavaScript generează conținutul în browser. Comportamentul implicit al React fără framework. Dezavantaj: SEO slab, slow first load.

→ Lecția 1
Edge Network

Rețea de servere distribuite global ale Vercel. Când deployezi pe Vercel, aplicația rulează pe serverul cel mai aproape geografic de utilizator, reducând latența.

→ Lecția 7
Hydration

Procesul prin care React „preia controlul" HTML-ului pre-generat de server. Adaugă event listeners și activează interactivitatea Client Components în browser, fără a re-randa HTML-ul vizibil.

→ Lecția 1, Lecția 3
ISR — Incremental Static Regeneration

Pagini statice care se regenerează automat în background la un interval configurat (ex: revalidate: 3600). Combină viteza paginilor statice cu prospețimea SSR.

→ Lecția 1
Pages Router

Sistemul de routing legacy al Next.js bazat pe directorul pages/. Folosește getServerSideProps și getStaticProps. Compatibil în continuare dar nu mai e recomandat pentru proiecte noi.

→ Lecția 1
SSG — Static Site Generation

HTML generat o singură dată la build time. Extrem de rapid — servit direct din CDN. Ideal pentru conținut care nu se schimbă des: pagini de prezentare, documentație, blog cu actualizări rare.

→ Lecția 1
SSR — Server-Side Rendering

HTML generat la fiecare request, pe server, cu date proaspete. Similar cu PHP/Laravel clasic. Ideal pentru pagini cu date care se schimbă frecvent sau personalizate per utilizator (dashboard, profil).

→ Lecția 1

Componente și fișiere

Client Component

Componentă React cu directiva 'use client' la primul rând. Rulează în browser — poate folosi useState, useEffect, event handlers, window, localStorage.

→ Lecția 3
error.tsx

Fișier special care definește UI-ul pentru erori runtime dintr-o rută. Trebuie să fie obligatoriu un Client Component ('use client') deoarece React error boundaries funcționează numai în browser.

→ Lecția 2
layout.tsx

UI wrapper persistent aplicat paginii curente și tuturor sub-rutelor. Nu se remontează la navigarea între paginile copil — starea internă supraviețuiește navigării. app/layout.tsx este obligatoriu (include <html> și <body>).

→ Lecția 2
loading.tsx

UI afișat automat în timp ce un Server Component face fetch (skeleton, spinner). Creează automat un Suspense boundary în jurul paginii. Zero boilerplate față de useState(isLoading).

→ Lecția 6
not-found.tsx

UI pentru paginile 404. Activat când chemi funcția notFound() din next/navigation într-un Server Component.

→ Lecția 2
page.tsx

Singurul fișier dintr-un folder app/ care creează o rută accesibilă public în browser. Fără page.tsx, ruta nu există chiar dacă folderul există.

→ Lecția 2
Route Group

Folder cu paranteze în app/ — ex: (auth). Nu adaugă prefix la URL. Folosit pentru organizarea fișierelor și pentru a aplica layout-uri diferite unor grupuri de pagini.

→ Lecția 2
Server Component

Comportamentul implicit al tuturor componentelor în App Router. Rulează pe server, poate folosi async/await direct, accesa DB/fișiere/variabile secrete. Nu trimite JS în browser. Nu poate folosi hooks React sau event handlers.

→ Lecția 3

Data fetching și mutații

generateStaticParams()

Funcție exportată dintr-un page.tsx cu rute dinamice. Returnează lista de parametri pentru care Next.js va pre-genera pagini statice la build. Ex: toate slug-urile de blog.

→ Lecția 4
revalidatePath()

Funcție importată din next/cache. Invalidează cache-ul pentru o rută sau un layout, forțând re-randarea Server Components la următoarea vizită. Apelată din Server Actions după mutații.

→ Lecția 6
revalidateTag()

Similar cu revalidatePath dar invalidează prin tag-uri. Mai granular — poți invalida toate fetch-urile cu un anumit tag fără a specifica rutele.

→ Lecția 6
Server Action

Funcție async marcată cu directiva 'use server'. Rulează exclusiv pe server dar poate fi apelată din Client Components sau formulare HTML. Elimină necesitatea unui API REST separat pentru mutații simple.

→ Lecția 6
Suspense

Componenta React care afișează un fallback UI în timp ce componenta copil face operații async. Next.js o folosește intern pentru loading.tsx. Componentele în Suspense separate fac fetch în paralel.

→ Lecția 6

Supabase

@supabase/ssr

Pachetul Supabase specializat pentru SSR (Next.js, Nuxt, SvelteKit). Oferă createServerClient și createBrowserClient care gestionează corect cookie-urile de sesiune pe server și client.

→ Lecția 5
anon key

Cheia publică Supabase — sigură să fie în browser (prefix NEXT_PUBLIC_). Permite operații de date filtrate de politicile RLS. Nu bypasează securitatea.

→ Lecția 5
RLS — Row Level Security

Mecanism PostgreSQL care definește politici de acces la nivel de rând. Utilizatorii văd/modifică numai rândurile permise de politici. Activat cu ALTER TABLE x ENABLE ROW LEVEL SECURITY.

→ Lecțiile 5, 8
service_role key

Cheia privată Supabase cu acces total la DB, ocolind RLS. Niciodată cu prefix NEXT_PUBLIC_. Folosită numai pe server pentru operații administrative (migrații, import date, webhooks).

→ Lecția 5

Navigare și routing

Dynamic Route

Rută cu segment variabil definit prin paranteze pătrate în numele folderului: [id], [slug]. Valoarea din URL devine disponibilă ca params.id sau params.slug.

→ Lecțiile 2, 4
Link (componenta)

Componenta Next.js pentru navigare internă. Extinde <a> cu navigare client-side (fără full reload) și prefetching automat al paginilor vizibile în viewport.

→ Lecția 4
Middleware (Next.js)

Cod care rulează înaintea fiecărui request, la edge. Definit în middleware.ts la rădăcina proiectului. Folosit cu Supabase pentru reîmprospătarea automată a sesiunilor.

→ Lecțiile 5, 8
notFound()

Funcție din next/navigation care activează not-found.tsx. Returnează un 404 HTTP fără a arunca o eroare vizibilă. Utilizată în Server Components când un resurs nu există.

→ Lecțiile 2, 4
redirect()

Funcție din next/navigation pentru redirect HTTP în Server Components. Aruncă o excepție internă — nu pune niciodată în try/catch. Generează HTTP 307 (temporary redirect).

→ Lecția 4

Variabile de mediu

.env.local

Fișier pentru variabile de mediu locale. Gitignored implicit. Nu se comite niciodată — conține chei secrete. Pe Vercel, variabilele se adaugă manual în dashboard sau prin vercel env pull.

→ Lecțiile 5, 7
NEXT_PUBLIC_ prefix

Prefixul care face o variabilă de mediu accesibilă în browser (inclusă în bundle-ul JS). Fără prefix, variabila e disponibilă numai pe server. Niciodată nu pune chei secrete cu acest prefix.

→ Lecțiile 3, 5
