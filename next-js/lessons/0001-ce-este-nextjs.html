Ce este Next.js și de ce îl folosim

Mental model-ul complet: React, SSR, SSG, ISR și de ce Next.js rezolvă probleme pe care React singur nu le rezolvă.

Problema: React singur nu este de ajuns

React este o librărie pentru UI — construiești componente, gestionezi stare, randezi în DOM. Dar React singur nu îți spune: unde rulează codul (server sau client)? Cum faci routing? Cum optimizezi pentru SEO? Cum faci fetch la date înainte de randare?

Next.js este un framework construit peste React care răspunde la toate aceste întrebări. Face alegerile arhitecturale în locul tău și furnizează convenții clare. Dacă Laravel este „PHP with opinionated conventions", Next.js este exact asta pentru React.

Analogie Laravel
React = PHP pur. Next.js = Laravel. Framework-ul adaugă routing, server-side rendering, optimizări automate, structură de foldere și deployment integrat.

Modurile de randare: SSR, SSG, ISR, CSR

Cea mai importantă decizie în Next.js este unde și când se generează HTML-ul paginii. Next.js suportă 4 moduri:

SSR

Server-Side Rendering

HTML generat la fiecare request pe server. Date mereu proaspete. Similar cu PHP/Laravel clasic.

dinamic
SSG

Static Site Generation

HTML generat o singură dată la build time. Extrem de rapid. Ideal pentru pagini care nu se schimbă des.

static
ISR

Incremental Static Regeneration

Static, dar se regenerează în background la un interval configurat. Combină viteza staticului cu prospețimea SSR.

hibrid
CSR

Client-Side Rendering

HTML gol trimis din server; JavaScript randează în browser. Comportamentul React „clasic".

client

În Next.js modern (App Router), SSR este implicit pentru toate componentele server. Nu trebuie să alegi explicit — Next.js deduce modul de randare din codul tău.

App Router vs Pages Router

Next.js are două sisteme de routing. Dacă începi azi un proiect nou, folosești App Router (introdus în Next.js 13, stabil din 14). Pages Router este moștenire.

  • App Router — directorul app/, Server Components implicit, layouts nested, mai puternic
  • Pages Router — directorul pages/, mai simplu dar cu mai puțin control, deprecat conceptual
Atenție
Tutorialele mai vechi (pre-2023) folosesc Pages Router. Dacă găsești getServerSideProps sau getStaticProps în exemple, e Pages Router — nu e abordarea recomandată pentru proiecte noi.

De ce Next.js pentru integrări cu Supabase și Vercel?

Next.js, Supabase și Vercel formează un stack complet fără fricțiune:

  • Vercel — compania care creează Next.js. Deploy cu un git push, zero configurare.
  • Supabase — PostgreSQL ca serviciu + Auth + Storage + Realtime. SDK oficial pentru Next.js SSR.
  • Next.js Server Actions — mutații pe server fără a scrie un API separat, ideal cu Supabase.
Stack-ul acestui curs
Next.js (App Router) + TypeScript + Supabase (DB + Auth) + Vercel (deploy). Nu ai nevoie de server propriu, nu ai nevoie de API separat, nu ai nevoie de Docker local.
Exercițiu — Identificare mod randare

Pentru fiecare caz de mai jos, gândește-te care mod de randare ar fi cel mai potrivit:

  1. O pagină de prețuri care se schimbă rar (lunar)
  2. Un dashboard cu date în timp real ale utilizatorului autentificat
  3. Un blog cu 500 de articole, actualizate zilnic
  4. Un widget de coș de cumpărături (vizibil doar în browser)

Răspuns așteptat: 1→SSG, 2→SSR, 3→ISR (revalidare zilnică), 4→CSR.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Ce adaugă Next.js față de React pur?
Corect. Next.js este un framework peste React — adaugă routing bazat pe fișiere, SSR/SSG/ISR, optimizări de imagini/fonturi, și convenții clare. React rămâne librăria pentru UI.
Întrebarea 2 din 5 Care mod de randare generează HTML la fiecare request, pe server?
Corect. SSR generează HTML la fiecare request, pe server, cu date proaspete. Este echivalentul PHP/Laravel clasic — fiecare request produce un răspuns HTML complet.
Întrebarea 3 din 5 Care director este folosit de App Router (sistemul modern al Next.js)?
Corect. App Router folosește directorul app/. Structura de foldere definește rutele — un folder app/dashboard/ cu un page.tsx creează automat ruta /dashboard.
Întrebarea 4 din 5 Ce este ISR (Incremental Static Regeneration)?
Corect. ISR combină avantajele staticului (viteză, caching) cu prospețimea datelor: paginile sunt statice dar Next.js le regenerează în background la intervalul specificat (ex. revalidate: 3600 = o oră).
Întrebarea 5 din 5 De ce Next.js, Supabase și Vercel formează un stack bun împreună?
Corect. Vercel este compania din spatele Next.js — deployment-ul este nativ integrat. Supabase oferă @supabase/ssr special pentru Next.js App Router. Împreună elimină aproape orice fricțiune de infrastructură.
