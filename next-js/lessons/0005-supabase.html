Conectare la Supabase

Setup complet: creezi proiectul Supabase, configurezi variabilele de mediu, inițializezi clientul pentru server și client, și faci primele query-uri.

Ce este Supabase?

Supabase este „Firebase open-source" — oferă o bază de date PostgreSQL, autentificare, storage, realtime și un API REST generat automat din schema bazei de date. Pentru Next.js, Supabase oferă pachetul @supabase/ssr special optimizat pentru App Router.

DB

PostgreSQL

Baza de date relațională completă. Poți scrie SQL direct, sau folosești Supabase Query Builder.

principal
AUTH

Supabase Auth

Email/password, magic link, OAuth (Google, GitHub, etc.). Sesiunile sunt gestionate prin cookies.

integrat
RLS

Row Level Security

Politici de securitate la nivel de rând — utilizatorii văd numai datele lor, definit direct în DB.

securitate
RT

Realtime

Subscripții WebSocket la schimbări din DB — poți actualiza UI-ul instant când datele se schimbă.

opțional

Setup: 1. Creare proiect Supabase

Mergi la supabase.com, creează cont și un proiect nou. Din Settings → API copiezi două valori esențiale:

  • Project URL — ex: https://xyzcompany.supabase.co
  • anon (public) key — cheie publică, sigură să fie în browser
  • service_role key — cheie privată, NUMAI pe server, niciodată în browser

Setup: 2. Variabile de mediu în Next.js

Creează fișierul .env.local în rădăcina proiectului (gitignored implicit): 

# .env.local
NEXT_PUBLIC_SUPABASE_URL=https://xyzcompany.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# NUMAI pe server — nu adaugă NEXT_PUBLIC_ la service_role!
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Securitate critică
SUPABASE_SERVICE_ROLE_KEY bypasează toate politicile RLS — are acces total la DB. Niciodată în variabile cu prefix NEXT_PUBLIC_, niciodată în Client Components.

Setup: 3. Instalare pachet

npm install @supabase/supabase-js @supabase/ssr

@supabase/ssr este pachetul specializat pentru SSR (Next.js, SvelteKit, Nuxt). Gestionează cookie-urile de sesiune corect pe server și client.

Configurare: Client Supabase pentru Server Components

Creează utilitarul lib/supabase/server.ts: 

// lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr';
import { cookies } from 'next/headers';

export function createClient() {
  const cookieStore = cookies();

  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return cookieStore.getAll();
        },
        setAll(cookiesToSet) {
          try {
            cookiesToSet.forEach(({ name, value, options }) =>
              cookieStore.set(name, value, options)
            );
          } catch {
            // Server Components nu pot seta cookies direct — ignorăm
          }
        },
      },
    }
  );
}

Configurare: Client Supabase pentru Client Components

// lib/supabase/client.ts
import { createBrowserClient } from '@supabase/ssr';

export function createClient() {
  return createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  );
}

Exemple de query-uri

Server Component — fetch date cu autentificarea utilizatorului:

// app/dashboard/page.tsx
import { createClient } from '@/lib/supabase/server';
import { redirect } from 'next/navigation';

export default async function DashboardPage() {
  const supabase = createClient();

  // Verifică autentificarea
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) redirect('/login');

  // Query cu RLS — returnează NUMAI datele utilizatorului autentificat
  const { data: tasks, error } = await supabase
    .from('tasks')
    .select('*')
    .order('created_at', { ascending: false });

  if (error) throw new Error(error.message);

  return (
    <ul>
      {tasks?.map(task => (
        <li key={task.id}>{task.title}</li>
      ))}
    </ul>
  );
}

Client Component — operații la eveniment (ex: login):

'use client';

import { createClient } from '@/lib/supabase/client';
import { useRouter } from 'next/navigation';

export default function LoginForm() {
  const supabase = createClient();
  const router = useRouter();

  async function handleLogin(email: string, password: string) {
    const { error } = await supabase.auth.signInWithPassword({
      email,
      password
    });

    if (error) {
      alert(error.message);
      return;
    }

    router.push('/dashboard');
    router.refresh();  // reîncarcă Server Components cu noua sesiune
  }

  // ... return JSX cu formular
}

Middleware pentru sesiuni (recomandat)

Creează middleware.ts în rădăcina proiectului pentru a reîmprospăta sesiunile automat: 

// middleware.ts
import { createServerClient } from '@supabase/ssr';
import { NextResponse, type NextRequest } from 'next/server';

export async function middleware(request: NextRequest) {
  let supabaseResponse = NextResponse.next({ request });

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() { return request.cookies.getAll(); },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, options)
          );
        },
      },
    }
  );

  // Reîmprospătează sesiunea
  await supabase.auth.getUser();

  return supabaseResponse;
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
};

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 De ce ai nevoie de două utilitare Supabase diferite (server.ts și client.ts)?
Corect. Pe server, Next.js trebuie să citească și să scrie cookies manual din next/headers. Pe browser, createBrowserClient gestionează cookies automat via document.cookie. Mecanismele diferă fundamental.
Întrebarea 2 din 5 Care cheie Supabase TREBUIE să rămână doar pe server, niciodată în browser?
Corect. Cheia service_role are acces complet la baza de date, ignorând toate politicile RLS. Dacă ajunge în browser, oricine poate citi/modifica/șterge orice date. Pune-o NUMAI în variabile de server (fără prefix NEXT_PUBLIC_).
Întrebarea 3 din 5 Ce face Row Level Security (RLS) în Supabase?
Corect. RLS permite politici ca „un utilizator poate vedea numai rândurile unde user_id = auth.uid()". Aceste reguli sunt aplicate de PostgreSQL la nivel de DB — nu de aplicație. Chiar dacă un bug din cod face un SELECT greșit, DB-ul returnează numai rândurile permise.
Întrebarea 4 din 5 De ce apelezi router.refresh() după un login reușit în Client Component?
Corect. Server Components citesc sesiunea din cookies la runtime. După login, cookie-ul de sesiune e setat. router.refresh() forțează Next.js să re-randeze Server Components pe pagina curentă — acestea vor vedea noua sesiune și pot randeze conținut autentificat.
Întrebarea 5 din 5 Ce rol are middleware-ul Supabase configurat în middleware.ts?
Corect. Token-urile JWT de sesiune expiră. Middleware-ul apelează supabase.auth.getUser() la fiecare request, care reîmprospătează automat token-ul dacă e pe cale să expire. Fără middleware, sesiunile utilizatorilor ar expira după câteva minute de inactivitate.
