Routing și navigare

Rute dinamice, componenta Link, hooks de navigare — tot ce ai nevoie pentru a construi navigarea unei aplicații reale.

Rute dinamice și cum accesezi parametrii

Un segment dinamic se creează cu paranteze pătrate în numele folderului: [id]. Componenta paginii primește parametrii ca prop params: 

// app/blog/[slug]/page.tsx
interface Props {
  params: { slug: string };
}

export default async function BlogPost({ params }: Props) {
  const { slug } = params;
  const post = await getPostBySlug(slug);

  if (!post) {
    notFound();  // afișează not-found.tsx
  }

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}

// Generare statică a rutelor cunoscute (opțional)
export async function generateStaticParams() {
  const posts = await getAllPosts();
  return posts.map(post => ({ slug: post.slug }));
}

generateStaticParams spune Next.js ce slugs există — va pre-genera paginile la build. Dacă un slug nu e în listă, Next.js va genera pagina la runtime (ISR comportament).

Search params și query string

URL-ul /search?q=nextjs&page=2 — parametrii după ? sunt search params. În Server Components îi primești ca prop searchParams: 

// app/search/page.tsx
interface Props {
  searchParams: { q?: string; page?: string };
}

export default async function SearchPage({ searchParams }: Props) {
  const query = searchParams.q ?? '';
  const page  = Number(searchParams.page ?? 1);

  const results = await search(query, page);

  return (
    <div>
      <h1>Rezultate pentru: {query}</h1>
      {results.map(r => <div key={r.id}>{r.title}</div>)}
    </div>
  );
}

Componenta Link — navigare client-side

<Link> din Next.js extinde tag-ul <a> cu prefetching automat și navigare client-side (fără full page reload). Este singura modalitate corectă de a naviga intern. 

import Link from 'next/link';

export default function Navbar() {
  return (
    <nav>
      <Link href="/">Home</Link>
      <Link href="/about">Despre</Link>
      <Link href="/blog/primul-post">Post fix</Link>

      {/* Rută dinamică construită cu template string */}
      <Link href={`/users/${userId}/profile`}>Profil</Link>

      {/* href ca obiect pentru search params */}
      <Link href={{ pathname: '/search', query: { q: 'nextjs' } }}>
        Caută Next.js
      </Link>
    </nav>
  );
}
Prefetching automat
Când un <Link> intră în viewport, Next.js prefetchează automat pagina destinație. Navigarea va fi instant. Poți dezactiva cu prefetch={false} dacă pagina e mare sau costisitoare.

Navigare programatică cu useRouter

Pentru navigare declanșată din cod (ex: după submit form), folosești hook-ul useRouter. Funcționează numai în Client Components: 

'use client';

import { useRouter } from 'next/navigation';

export default function LoginForm() {
  const router = useRouter();

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    // ... logică autentificare
    router.push('/dashboard');      // navighează și adaugă în history
    // router.replace('/dashboard'); // navighează fără a adăuga în history
    // router.back();                // înapoi în history
    // router.refresh();             // reîncarcă Server Components pe pagină curentă
  }

  return <form onSubmit={handleSubmit}>...</form>;
}

usePathname și useSearchParams

Pentru a citi ruta curentă sau search params în Client Components: 

'use client';

import { usePathname, useSearchParams } from 'next/navigation';

export default function Breadcrumb() {
  const pathname = usePathname();      // '/dashboard/settings'
  const searchParams = useSearchParams();
  const query = searchParams.get('q'); // 'nextjs' din ?q=nextjs

  return <p>Ești pe: {pathname}</p>;
}

Redirects programatice din Server Components

Din Server Components poți redirecta fără JavaScript în browser: 

import { redirect, notFound } from 'next/navigation';

export default async function ProtectedPage() {
  const user = await getUser();

  if (!user) {
    redirect('/login');  // HTTP 307 redirect — instant, fără JS client
  }

  if (!user.isAdmin) {
    notFound();  // afișează not-found.tsx
  }

  return <div>Pagina protejată</div>;
}
Atenție
redirect() aruncă o excepție internă pentru a opri execuția — nu pune niciodată un try/catch care ar prinde redirect(). Va bloca redirectarea.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Cum accesezi params.id dintr-o rută dinamică /users/[id] într-un Server Component?
Corect. Server Components primesc params ca prop direct: export default function Page({'{'} params {'}'}: {'{'} params: {'{'} id: string {'}'} {'}'}) {'{'} ... {'}'}. Hook-urile (useParams) funcționează numai în Client Components.
Întrebarea 2 din 5 De ce <Link> este preferată față de <a href="..."> pentru navigare internă?
Corect. <Link> interceptează click-ul și navighează client-side — fără a reîncărca pagina. Plus prefetchează paginile vizibile în viewport. Un <a> obișnuit face un full page reload, pierzând starea aplicației.
Întrebarea 3 din 5 Care este diferența dintre router.push() și router.replace()?
Corect. push adaugă o intrare nouă în browser history — utilizatorul poate da Back. replace suprascrie intrarea curentă — Back duce la pagina de dinaintea celei suprascrise. Util pentru flow-uri de autentificare unde nu vrei ca utilizatorul să dea Back la pagina de login.
Întrebarea 4 din 5 Funcția generateStaticParams() dintr-un page.tsx cu rută dinamică ce face?
Corect. generateStaticParams returnează array-ul de parametri pentru care Next.js va genera HTML static la build. Ex: return posts.map(p => ({'{'} slug: p.slug {'}'})). Paginile respective vor fi instant — HTML pre-generat, fără DB la runtime.
Întrebarea 5 din 5 Ce se întâmplă dacă pui un try/catch în jurul funcției redirect()?
Corect. redirect() funcționează aruncând o excepție specială pe care Next.js o prinde. Dacă pui try/catch, codul tău prinde excepția și redirectarea nu mai are loc. Soluția: pune redirect() în afara blocului try, sau re-aruncă excepția dacă e de tip RedirectError.
