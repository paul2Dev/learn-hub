referință AI Harness — Curs

Glosar

Vocabularul de bază pentru harness-ul AI Claude Code. Termenii sunt folosiți în mod consecvent în toate lecțiile.

A

Agent
O instanță LLM care operează în cadrul unui harness — primește o fereastră de context, poate apela tool-uri și produce răspunsuri sau apeluri de tool ulterioare. În Claude Code, tu ești agentul când rulezi claude; sub-agenții sunt agenți pe care îi lansezi prin tool-ul Agent. Vezi și: Subagent, Harness

C

Context Window (Fereastră de context)
Totalul inputului pe care modelul îl vede la momentul inferenței: system prompt, istoricul conversației, rezultatele tool-urilor și documentele injectate. Harness-ul gestionează ce intră în fereastra de context și o comprimă când devine prea mare. Vezi și: Session

H

Harness
Un strat runtime care mediază între un om (sau sistem) și un LLM. Oferă tool-uri pe care modelul le poate apela, context pe care îl poate citi, permisiuni care controlează acțiunile și hooks care se declanșează la evenimente. Fără un harness, ai un apel brut de API. Cu unul, ai un agent. Vezi și: Tool, Hook, Permission Layer
Hook
O comandă shell înregistrată în settings.json pe care harness-ul o execută când se declanșează un eveniment specific. Evenimentele includ PreToolUse, PostToolUse, Stop și Notification. Hooks-urile rulează în afara modelului — sunt shell pur, nu AI. Vezi și: settings.json, PreToolUse, PostToolUse

M

MCP (Model Context Protocol)
Un standard deschis (creat de Anthropic) pentru conectarea serverelor de tool-uri externe la agenți AI. Un server MCP expune unul sau mai multe tool-uri printr-o schemă JSON; harness-ul le înregistrează alături de tool-urile built-in astfel încât modelul să le poată apela în mod transparent. Vezi și: Tool, MCP Server
MCP Server
Un proces (Node.js, Python etc.) care implementează protocolul MCP și expune tool-uri personalizate harness-ului. Înregistrat în settings.json sub mcpServers. Vezi și: MCP

P

Permission Layer
Configurarea din settings.json (și settings.local.json) care controlează ce tool-uri și comenzi shell agentul are voie să execute. Constă dintr-o listă allow și o listă deny. Verificată înainte de fiecare apel de tool. Vezi și: settings.json, Hook
PostToolUse
Un eveniment de hook care se declanșează după finalizarea execuției unui tool. Hook-ul primește numele tool-ului și outputul său. Folosit pentru: logging, trimitere de notificări, declanșarea de efecte secundare bazate pe rezultate. Vezi și: PreToolUse, Hook
PreToolUse
Un eveniment de hook care se declanșează înainte de execuția unui tool. Hook-ul primește numele tool-ului și argumentele de input. Folosit pentru: validare, logging, limitare a ratei sau blocarea unor comenzi specifice. Vezi și: PostToolUse, Hook

S

Session (Sesiune)
Un singur ciclu de viață al harness-ului Claude Code, de la pornire (claude) până la ieșire. Fiecare sesiune are propria fereastră de context, stare de permisiuni și înregistrări de hooks. Vezi și: Context Window
settings.json
Fișierul de configurare principal pentru harness-ul Claude Code. Poate fi la nivel de proiect (.claude/settings.json) sau la nivel de utilizator (~/.claude/settings.json). Definește tool-urile permise/blocate, hooks-urile, serverele MCP și variabilele de mediu. Vezi și: Permission Layer, Hook, MCP Server
Skill
Un set de instrucțiuni cu nume, reutilizabil, invocat prin un slash command (ex. /teach). Skills-urile pot împacheta resurse, fișiere de format și prompturi cu mai mulți pași într-o singură comandă. Definite ca fișiere în ~/.claude/skills/. Vezi și: Agent
Subagent
Un agent lansat de un alt agent prin tool-ul Agent. Sub-agenții pornesc cu un context proaspăt, fără memorie a sesiunii părintelui. Pot rula în foreground (blochează până la finalizare) sau background (paralel). Rezultatele sunt returnate ca un singur mesaj. Vezi și: Agent, Tool

T

Tool
O funcție pe care agentul o poate apela în timpul unei sesiuni. Tool-urile built-in includ Read, Write, Edit, Bash, Grep, Glob, WebSearch, Agent și altele. Tool-urile personalizate sunt adăugate prin servere MCP. Vezi și: MCP, Permission Layer
Tool Dispatch
Procesul prin care harness-ul primește un apel de tool de la model, verifică permisiunile, declanșează hooks-urile PreToolUse, execută tool-ul, declanșează hooks-urile PostToolUse și returnează rezultatul în contextul modelului. Vezi și: Tool, Hook, Permission Layer
