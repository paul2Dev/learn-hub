Scheduling & Automatizare

Rulează agenți pe un program — chiar și când IDE-ul tău este închis.

De ce să planifici agenți?

Până acum, fiecare interacțiune cu un agent a fost reactivă — tu întrebi, Claude răspunde. Scheduling întoarce acest lucru: definești o sarcină o dată, iar harness-ul o rulează automat la momentul sau intervalul pe care îl specifici, fără să fii prezent.

Agenții planificați rulează în cloud, nu pe mașina ta. Asta înseamnă:

  • Laptopul tău poate fi oprit — programul tot se declanșează
  • Agentul are același acces la tool-uri ca sesiunea ta normală
  • Rezultatele sunt livrate ca notificări când deschizi din nou Claude Code
Cazuri de utilizare bune
Remindere zilnice de standup · Rezumate ale rulărilor de test nocturne · Audituri săptămânale de dependențe · Monitorizare orară a unui pipeline CI · Un singur "amintește-mi să revizuiesc acest PR vineri"

Sintaxa cron

O expresie cron are cinci câmpuri: minut oră zi-din-lună lună zi-din-săptămână. * înseamnă "orice valoare." Iată pattern-urile pe care le vei folosi cel mai des: 

┌─ minut  (0–59)
│  ┌─ oră   (0–23, UTC)
│  │  ┌─ zi din lună  (1–31)
│  │  │  ┌─ lună  (1–12)
│  │  │  │  ┌─ zi din săptămână  (0=Dum, 1=Lun … 6=Sâm)
│  │  │  │  │
*  *  *  *  *

Exemple:
"0 9 * * 1-5"    →  09:00 UTC, Luni până Vineri
"0 8 * * 1"      →  08:00 UTC, în fiecare Luni
"*/30 * * * *"   →  la fiecare 30 de minute
"0 0 * * *"      →  miezul nopții UTC, în fiecare zi
"0 9 1 * *"      →  09:00 UTC în prima zi din fiecare lună
Fus orar
Orele cron sunt în UTC. Dacă ești în UTC+3, scade 3 ore din ora locală pentru a obține echivalentul UTC. 09:00 UTC+3 = 06:00 UTC.

Crearea unui program

Când îi ceri lui Claude Code să creeze un agent planificat, acesta invocă CronCreate cu o expresie cron și un prompt care descrie ce ar trebui să facă agentul de fiecare dată când se declanșează. 

// Ce apelează harness-ul intern:
CronCreate({
  schedule: "0 9 * * 1-5",
  prompt: `
    Rulează suita de teste a proiectului cu: npm test
    Rezumă rezultatele: câte au trecut, au eșuat sau au fost sărite.
    Dacă vreun test a eșuat, listează numele testelor și mesajele de eroare.
    Trimite rezumatul ca notificare.
  `
})

Promptul este briefing-ul complet pentru agent — exact ca un prompt de subagent. Scrie-l să fie autonom; agentul care rulează luni dimineața nu are nicio memorie a conversației în care ai configurat programul.

Rulări unice cu ScheduleWakeup

ScheduleWakeup este folosit în interiorul unei sesiuni în buclă pentru a face pauză și a relua după o întârziere. Este tool-ul potrivit când vrei să:

  • Sondezi o rulare CI la fiecare 5 minute până se termină
  • Verifici un deploy după 10 minute
  • Îți amintești ceva în 2 ore
ScheduleWakeup({
  delaySeconds: 600,          // 10 minute
  prompt: "Verifică starea deploy-ului la staging.example.com și raportează.",
  reason: "așteptând finalizarea deploy-ului"
})

Spre deosebire de CronCreate, un ScheduleWakeup se declanșează o dată și dispare. Menține fereastra de context caldă dacă întârzierea este sub 270 de secunde (în fereastra de cache de 5 minute).

Exercițiu — Creează și gestionează un program
  1. Cere lui Claude Code: "Planifică un agent zilnic să ruleze la 09:00 UTC în zilele lucrătoare. Ar trebui să citească CHANGELOG.md și să rezume ultimele 3 intrări ca briefing de dimineață."
  2. Claude va apela CronCreate. Notează ID-ul programului pe care îl returnează.
  3. Întreabă: "Listează toți agenții mei planificați." — acesta apelează CronList. Ar trebui să-ți vezi noul program cu momentul următoarei rulări.
  4. Întreabă: "Anulează programul de digest zilnic." — Claude apelează CronDelete cu ID-ul programului.
  5. Verifică că a dispărut cu un alt apel CronList.
Exercițiu — Reminder unic
  1. Pornește o sesiune în buclă: cere lui Claude să "îmi amintești în 5 minute să verific review-ul PR-ului."
  2. Claude va folosi ScheduleWakeup cu delaySeconds: 300.
  3. După 5 minute, bucla se trezește și livrează reminderul ca notificare.

Verifică-ți înțelegerea

Întrebarea 1 din 5 Poate un agent planificat (via CronCreate) să ruleze cât timp laptopul tău este oprit?
Corect. Agenții planificați sunt executați în cloud. Nu depind de mașina locală, IDE sau conexiunea la rețea. Rezultatele te așteaptă ca notificări când deschizi din nou Claude Code.
Întrebarea 2 din 5 Ce înseamnă expresia cron "0 8 * * 1"?
Corect. Câmpurile: 0=minutul 0, 8=ora 8, *=orice zi, *=orice lună, 1=Luni. Ziua din săptămână folosește 0=Duminică până la 6=Sâmbătă. 1-5 ar fi zilele lucrătoare.
Întrebarea 3 din 5 Promptul unui agent planificat ar trebui să fie…
Corect. Agentul care se declanșează conform programului pornește proaspăt — nu are nicio memorie a când sau de ce ai creat programul. Promptul trebuie să îi spună totul: ce să facă, unde să găsească lucrurile și ce output să producă.
Întrebarea 4 din 5 Ce tool folosești pentru o rulare viitoare unică (non-recurentă)?
Corect. ScheduleWakeup se declanșează o dată după o întârziere în secunde, apoi se curăță singur. Folosește CronCreate numai pentru programe recurente. Nu există flag "once" pe CronCreate.
Întrebarea 5 din 5 Care este strategia optimă pentru delaySeconds la ScheduleWakeup?
Corect. Cache-ul de prompt Anthropic are un TTL de 5 minute (300s). A dormi sub 270s menține cache-ul cald (reintegrare rapidă, ieftină). A dormi peste 300s generează un cache miss — deci dacă trebuie să aștepți mai mult, angajează-te la 1200s+ pentru a amortiza costul. Exact 300s este cel mai rău scenariu din ambele.
