MCP Servers

Extinde harness-ul cu tool-uri personalizate — conectează baze de date, API-uri interne și orice altceva are nevoie modelul tău.

Ce este MCP?

Model Context Protocol (MCP) este un standard deschis, creat de Anthropic, pentru conectarea agenților AI la tool-uri și surse de date externe. Un server MCP este orice proces care implementează acest protocol — expune unul sau mai multe tool-uri printr-o schemă JSON definită, iar harness-ul le înregistrează alături de tool-urile built-in astfel încât modelul să le poată apela în mod transparent.

Din perspectiva modelului, nu există nicio diferență între apelarea tool-ului built-in Read și apelarea tool-ului tău MCP personalizat query_database. Ambele apar în tool registry; ambele trec prin permission layer.

De ce MCP în loc de hooks?
Hooks-urile interceptează apelurile de tool existente. MCP adaugă tool-uri noi pe care modelul le poate apela. Hook: "loghează fiecare apel Bash." MCP: "oferă modelului un nou tool pentru a interoga Postgres direct."

Tipuri de transport

1

stdio (cel mai comun)

Harness-ul lansează serverul tău ca proces copil și comunică prin stdin/stdout. Simplu, rapid și local. Nu este necesară o rețea.

recomandat
2

SSE (Server-Sent Events)

Serverul tău rulează independent și harness-ul se conectează prin HTTP. Folosit pentru servere remote sau tooling partajat de echipă.

remote / partajat
3

HTTP (Streamable)

Transport mai nou cu suport complet de streaming. Cel mai bun pentru tool-uri cu throughput mare care returnează rezultate voluminoase.

avansat

Începe cu stdio. Nu necesită gestionarea serverului — harness-ul pornește și oprește procesul automat.

Înregistrarea unui server MCP

Serverele MCP sunt înregistrate în settings.json sub "mcpServers": 

{
  "mcpServers": {
    "my-tools": {
      "type": "stdio",
      "command": "node",
      "args": ["/path/to/my-mcp-server/index.js"],
      "env": {
        "DATABASE_URL": "postgres://localhost:5432/mydb"
      }
    },
    "remote-api": {
      "type": "sse",
      "url": "https://tools.example.com/mcp",
      "headers": {
        "Authorization": "Bearer $MY_API_TOKEN"
      }
    }
  }
}

Câmpuri cheie:

  • "type" — "stdio", "sse" sau "http"
  • "command" + "args" — pentru stdio: comanda de lansare a serverului
  • "env" — variabile de mediu injectate în procesul serverului
  • "url" — pentru SSE/HTTP: endpoint-ul serverului

Cum sunt definite tool-urile

Fiecare tool expus de serverul tău MCP este descris printr-o schemă JSON care îi spune modelului: cum se numește tool-ul, ce face și ce inputuri acceptă. 

{
  "name": "query_database",
  "description": "Execută o interogare SQL SELECT read-only față de baza de date a aplicației. Returnează rânduri ca array JSON.",
  "inputSchema": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "O instrucțiune SQL SELECT validă. Numai SELECT este permis."
      },
      "limit": {
        "type": "number",
        "description": "Numărul maxim de rânduri de returnat. Implicit 100.",
        "default": 100
      }
    },
    "required": ["query"]
  }
}

Descrierea este critică — modelul o folosește pentru a decide când să apeleze tool-ul tău. Scrie-o din perspectiva modelului: ce voi obține dacă apelezi asta?

Un server MCP minimal

Iată cel mai mic server MCP complet în Node.js — un tool care returnează ora curentă: 

// mcp-time-server/index.js
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({ name: "time-server", version: "1.0.0" });

server.tool(
  "get_current_time",
  "Returnează ora curentă UTC ca șir ISO 8601.",
  { timezone: z.string().optional().describe("Numele fusului orar IANA, ex. 'Europe/Bucharest'") },
  async ({ timezone }) => {
    const date = new Date();
    const display = timezone
      ? date.toLocaleString("ro-RO", { timeZone: timezone })
      : date.toISOString();
    return { content: [{ type: "text", text: display }] };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);
// package.json
{
  "name": "mcp-time-server",
  "type": "module",
  "dependencies": {
    "@modelcontextprotocol/sdk": "^1.0.0",
    "zod": "^3.0.0"
  }
}

Înregistrează-l în settings.json:

{
  "mcpServers": {
    "time": {
      "type": "stdio",
      "command": "node",
      "args": ["./mcp-time-server/index.js"]
    }
  }
}

Acum poți întreba Claude: "Ce oră este în București?" și va apela get_current_time({ timezone: "Europe/Bucharest" }).

Debugging la serverele MCP

Când un server MCP nu se încarcă sau un tool eșuează, verifică acestea în ordine:

  1. Stderr-ul serverului — Serverele MCP scriu erorile pe stderr. Verifică panoul de output al Claude Code.
  2. Pornirea — testează serverul în izolare: echo '{}' | node index.js
  3. Schema — modelul nu va apela un tool cu o descriere vagă sau câmpuri required lipsă
  4. Permissions — tool-urile MCP sunt supuse acelorași reguli allow/deny ca tool-urile built-in. Adaugă "mcp__my-tools__query_database" la lista ta allow dacă este necesar.
MCP Inspector
Anthropic oferă un tool oficial MCP Inspector pentru testarea interactivă a serverelor: npx @modelcontextprotocol/inspector node index.js. Îți permite să apelezi tool-uri și să inspectezi răspunsurile fără a rula o sesiune Claude completă.

Practic: construiește și conectează un tool

Exercițiu — Primul tău tool MCP
  1. Creează directorul serverului și instalează SDK-ul: 
    mkdir mcp-demo && cd mcp-demo
npm init -y
npm install @modelcontextprotocol/sdk zod
  2. Creează index.js cu codul serverului de timp din secțiunea de mai sus. Actualizează package.json cu "type": "module".
  3. Testează-l cu MCP Inspector: 
    npx @modelcontextprotocol/inspector node index.js
    Apelează get_current_time cu {"timezone": "UTC"} pentru a verifica că răspunde.
  4. Înregistrează-l în .claude/settings.json și repornește Claude Code.
  5. Întreabă Claude: "Ce oră este în Tokyo?" Ar trebui să apeleze tool-ul tău cu timezone: "Asia/Tokyo".
Extensie — Adaugă un tool de numărat fișiere

Adaugă un al doilea tool la același server care numără fișierele care se potrivesc cu un pattern glob: 

server.tool(
  "count_files",
  "Numără fișierele care se potrivesc cu un pattern glob în directorul curent.",
  { pattern: z.string().describe("Pattern glob, ex. '**/*.ts'") },
  async ({ pattern }) => {
    const { glob } = await import("glob");
    const files = await glob(pattern);
    return {
      content: [{
        type: "text",
        text: `${files.length} fișier(e) se potrivesc cu '${pattern}'`
      }]
    };
  }
);

Instalează glob (npm install glob), apoi testează cu: "Câte fișiere TypeScript sunt în acest proiect?"

Verifică-ți înțelegerea

Întrebarea 1 din 5 Din perspectiva modelului, cum diferă un tool MCP de un tool built-in ca Read?
Corect. Harness-ul normalizează toate tool-urile — built-in și MCP — în același registry. Modelul vede numele, descrierea și schema tool-ului și îl apelează fără să știe (sau să-i pese) dacă este un built-in sau un server extern.
Întrebarea 2 din 5 Ce tip de transport este recomandat pentru serverele MCP locale, private?
Corect. Cu stdio, harness-ul lansează și gestionează procesul serverului automat. Niciun port, niciun firewall, niciun daemon de menținut. Folosește SSE sau HTTP când ai nevoie ca serverul să fie partajat între mașini sau echipe.
Întrebarea 3 din 5 De ce este câmpul description din schema unui tool atât de important?
Corect. Modelul nu are altă modalitate de a ști ce face un tool. O descriere slabă duce la faptul că tool-ul nu este niciodată apelat (modelul nu recunoaște cazul de utilizare) sau este apelat incorect (modelul înțelege greșit contractul). Scrie descrierile din perspectiva modelului: ce voi obține dacă apelezi asta?
Întrebarea 4 din 5 Un tool MCP pe care l-ai adăugat nu este apelat de model. Ce ar trebui să verifici mai întâi?
Corect. Majoritatea problemelor MCP sunt eșecuri la pornirea serverului — o dependență lipsă, o cale greșită sau o eroare de sintaxă. Verifică mai întâi outputul stderr în panoul de output al Claude Code. Dacă serverul nu pornește, tool-ul nu se înregistrează niciodată și modelul nu îl poate apela.
Întrebarea 5 din 5 Tool-urile MCP trec prin permission layer înainte de execuție?
Corect. Tool-urile MCP sunt supuse acelorași reguli de permisiune ca built-in-urile. Le poți permite sau bloca folosind pattern-ul "mcp__server-name__tool-name" în configurarea permisiunilor. Asta îți permite să adaugi un tool MCP puternic menținând în același timp restricții asupra operațiunilor pe care le poate efectua.
