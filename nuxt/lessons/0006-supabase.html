Supabase în Nuxt 3

Integrarea completă cu modulul oficial @nuxtjs/supabase — autentificare, query-uri, RLS și diferența față de integrarea cu Next.js.

De ce @nuxtjs/supabase în loc de @supabase/ssr?

În Next.js ai folosit @supabase/ssr și ai creat manual clientul de server și de browser. În Nuxt, modulul oficial @nuxtjs/supabase face totul automat:

MOD

@nuxtjs/supabase

Configurează automat clientul Supabase, injectează composable-uri, gestionează sesiunile via cookies.

recomandat
AUTO

Auto-imports incluse

useSupabaseClient(), useSupabaseUser(), serverSupabaseClient(), serverSupabaseUser() — disponibile fără import.

zero config
SES

Sesiuni automate

Refresh automat al token-urilor de sesiune, sincronizare server-client, middleware inclus.

integrat

Setup: instalare și configurare

# Instalare modul
npm install @nuxtjs/supabase
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@nuxtjs/supabase'],

  supabase: {
    // Redirect după login — implicit /
    redirect: true,
    redirectOptions: {
      login: '/login',
      callback: '/confirm',
      exclude: ['/', '/about', '/blog/*'],  // rute publice
    }
  }
})
# .env
SUPABASE_URL=https://xyzcompany.supabase.co
SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...   # anon key

# NUMAI server — fără prefix NUXT_PUBLIC_
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Față de Next.js
Next.js: NEXT_PUBLIC_SUPABASE_URL și NEXT_PUBLIC_SUPABASE_ANON_KEY în .env.local.
Nuxt: SUPABASE_URL și SUPABASE_KEY în .env. Modulul le citește automat și le expune public prin runtimeConfig.public.

useSupabaseClient — query-uri pe client

<!-- pages/dashboard.vue -->
<script setup lang="ts">
const supabase = useSupabaseClient()
const user = useSupabaseUser()

// Redirect dacă nu e autentificat
if (!user.value) {
  await navigateTo('/login')
}

// Query cu RLS — returnează numai datele utilizatorului
const { data: tasks } = await useAsyncData('tasks', async () => {
  const { data, error } = await supabase
    .from('tasks')
    .select('*')
    .order('created_at', { ascending: false })

  if (error) throw error
  return data
})
</script>

<template>
  <div>
    <h1>Bun venit, {{ user?.email }}</h1>
    <ul>
      <li v-for="task in tasks" :key="task.id">{{ task.title }}</li>
    </ul>
  </div>
</template>

useSupabaseUser — starea autentificării

useSupabaseUser() returnează un Ref<User | null> reactiv. Se actualizează automat la login/logout: 

<!-- components/Navbar.vue -->
<script setup lang="ts">
const user = useSupabaseUser()
const supabase = useSupabaseClient()

async function logout() {
  await supabase.auth.signOut()
  await navigateTo('/login')
}
</script>

<template>
  <nav>
    <NuxtLink to="/">Home</NuxtLink>

    <template v-if="user">
      <NuxtLink to="/dashboard">Dashboard</NuxtLink>
      <button @click="logout">Logout ({{ user.email }})</button>
    </template>
    <template v-else>
      <NuxtLink to="/login">Login</NuxtLink>
    </template>
  </nav>
</template>

Autentificare — email și password

<!-- pages/login.vue -->
<script setup lang="ts">
definePageMeta({ layout: 'auth' })

const supabase = useSupabaseClient()
const email    = ref('')
const password = ref('')
const error    = ref<string | null>(null)
const loading  = ref(false)

async function handleLogin() {
  loading.value = true
  error.value   = null

  const { error: authError } = await supabase.auth.signInWithPassword({
    email: email.value,
    password: password.value,
  })

  if (authError) {
    error.value = authError.message
  } else {
    await navigateTo('/dashboard')
  }

  loading.value = false
}

async function handleRegister() {
  loading.value = true
  const { error: authError } = await supabase.auth.signUp({
    email: email.value,
    password: password.value,
  })
  if (authError) error.value = authError.message
  else await navigateTo('/dashboard')
  loading.value = false
}
</script>

<template>
  <form @submit.prevent="handleLogin">
    <input v-model="email"    type="email"    placeholder="Email"    required />
    <input v-model="password" type="password" placeholder="Parolă"   required />
    <p v-if="error" style="color:red">{{ error }}</p>
    <button type="submit" :disabled="loading">
      {{ loading ? 'Se procesează...' : 'Login' }}
    </button>
    <button type="button" @click="handleRegister" :disabled="loading">
      Înregistrare
    </button>
  </form>
</template>

serverSupabaseClient — query-uri pe server (API routes)

// server/api/tasks.get.ts
import { serverSupabaseUser, serverSupabaseClient } from '#supabase/server'

export default defineEventHandler(async (event) => {
  // Verificare autentificare pe server
  const user = await serverSupabaseUser(event)
  if (!user) {
    throw createError({ statusCode: 401, message: 'Neautorizat' })
  }

  // Client cu sesiunea utilizatorului — RLS activ
  const client = await serverSupabaseClient(event)
  const { data, error } = await client
    .from('tasks')
    .select('*')
    .order('created_at', { ascending: false })

  if (error) throw createError({ statusCode: 500, message: error.message })
  return data
})
serverSupabaseClient vs useSupabaseClient
useSupabaseClient() — în componente Vue (client-side).
serverSupabaseClient(event) — în server routes Nitro (server-side).
Nu le confunda: useSupabaseClient nu există în contextul de server Nitro.

OAuth — Google, GitHub, etc.

<script setup lang="ts">
const supabase = useSupabaseClient()

async function loginWithGoogle() {
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'google',
    options: {
      redirectTo: `${window.location.origin}/confirm`,
    }
  })
  if (error) console.error(error)
}
</script>

<template>
  <button @click="loginWithGoogle">Login cu Google</button>
</template>
Exercițiu — Setup complet

Configurează Supabase în proiectul tău Nuxt:

  1. Instalează @nuxtjs/supabase și adaugă în nuxt.config.ts
  2. Adaugă SUPABASE_URL și SUPABASE_KEY în .env
  3. Creează un tabel notes în Supabase cu RLS activat
  4. Creează pagina pages/login.vue cu formular de email + password
  5. Creează pagina pages/dashboard.vue care afișează notele utilizatorului
  6. Testează că login-ul funcționează și că datele sunt filtrate de RLS

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Care este principalul avantaj al modulului @nuxtjs/supabase față de @supabase/ssr?
Corect. @nuxtjs/supabase este un modul Nuxt complet integrat — configurezi URL și key în .env, adaugi în modules[] și gata. Primești automat useSupabaseClient, useSupabaseUser etc. fără a scrie niciun utilitar manual.
Întrebarea 2 din 5 Ce returnează useSupabaseUser()?
Corect. useSupabaseUser() returnează un Ref Vue reactiv. Când utilizatorul face login sau logout, user.value se actualizează automat și toate componentele care îl folosesc se re-randează. Accesezi userul cu user.value în JavaScript sau user direct în template.
Întrebarea 3 din 5 Când folosești serverSupabaseClient(event) în loc de useSupabaseClient()?
Corect. useSupabaseClient() este un composable Vue — există numai în contextul componentelor (.vue). În server/api/ ești în contextul Nitro (Node.js), unde composable-urile Vue nu există. Folosești serverSupabaseClient(event) care citește sesiunea din cookies request-ului.
Întrebarea 4 din 5 Cum specifici variabila de mediu pentru cheia anon Supabase în Nuxt?
Corect. Modulul @nuxtjs/supabase are numele fixe: SUPABASE_URL și SUPABASE_KEY pentru anon key (publice), și SUPABASE_SERVICE_ROLE_KEY pentru service role (privată, server only). Nu ai nevoie de prefixe speciale.
Întrebarea 5 din 5 Ce opțiune din nuxt.config.ts configurează unde se redirecționează utilizatorul neautentificat?
Corect. Modulul @nuxtjs/supabase include un middleware de autentificare automat. Configurezi unde se duce la login (login: '/login'), unde se callback-uiește după OAuth (callback: '/confirm') și ce rute sunt publice (exclude: ['/']).
