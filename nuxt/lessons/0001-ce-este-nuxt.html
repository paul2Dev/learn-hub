Ce este Nuxt 3 și de ce îl folosim

Mental model complet: Vue vs Nuxt, Nitro server engine, moduri de randare și unde se plasează față de Next.js.

Nuxt este Next.js-ul ecosistemului Vue

Dacă știi deja Next.js, ai înțeles deja 70% din Nuxt — rezolvă aceleași probleme, cu aceleași soluții conceptuale. Diferența esențială: Nuxt folosește Vue 3 în loc de React.

AspectNuxt 3Next.js 15
Framework UI Vue 3 (Composition API) React 19
Server engine Nitro — portabil pe orice runtime Node.js (optimizat pentru Vercel)
Template syntax <template> HTML-like în SFC .vue JSX în fișiere .tsx
Auto-imports Da — componente și composables automat Nu — import-uri explicite
File routing pages/about.vue → /about app/about/page.tsx → /about
Dacă știi Next.js
Vei recunoaște imediat: SSR, SSG, file-based routing, middleware, layouts, fetch pe server, deploy cu zero config. Adaptarea principală este sintaxa Vue (template + composables) față de JSX + hooks React.

Cele două inovații cheie ale Nuxt 3

1. Nitro — server engine universal

Nitro este motorul de server al Nuxt 3. Spre deosebire de Next.js care e strâns legat de Node.js (și optimizat pentru Vercel), Nitro produce output care rulează pe:

NOD

Node.js

Serverul clasic — VPS, Docker, Railway, Render

implicit
VRL

Vercel / Netlify

Serverless functions — același cod, alt preset de build

preset: vercel
CF

Cloudflare Workers

Edge computing global — latență minimă, fără cold starts

preset: cloudflare
STC

Static (SSG complet)

HTML pur — GitHub Pages, Netlify, orice CDN

nuxi generate

2. Auto-imports — zero boilerplate

În Nuxt, nu trebuie să importi manual componente sau composables. Dacă există components/Button.vue, poți folosi <Button /> în orice template. Dacă există composables/useUser.ts, poți apela useUser() oriunde. 

<!-- pages/dashboard.vue — FĂRĂ import-uri manuale -->
<script setup lang="ts">
// Nu trebuie: import { ref } from 'vue'
// Nu trebuie: import MyCard from '~/components/MyCard.vue'
// Nu trebuie: import { useUser } from '~/composables/useUser'

const user = useUser()       // composable auto-importat
const count = ref(0)         // ref() auto-importat din Vue
</script>

<template>
  <MyCard :title="user.name">  <!-- componentă auto-importată -->
    {{ count }}
  </MyCard>
</template>

Modurile de randare în Nuxt 3

La fel ca Next.js, Nuxt suportă mai multe moduri — dar decizia se ia diferit:

SSR

Universal (SSR + Hydration) — implicit

Pagina se randează pe server la fiecare request, apoi React/Vue preia controlul în browser. SEO perfect.

renderMode: universal
SPA

SPA — Client-only

HTML minim, tot se randează în browser. Util pentru dashboard-uri autentificate fără nevoi SEO.

renderMode: client
SSG

Static (SSG)

HTML pre-generat la build cu nuxi generate. Cel mai rapid — servit din CDN.

nuxi generate
HYB

Hybrid — per rută

Nuxt 3 permite configurarea modului de randare per rută: unele statice, altele SSR, altele SPA.

routeRules
// nuxt.config.ts — Hybrid rendering cu routeRules
export default defineNuxtConfig({
  routeRules: {
    '/':           { prerender: true },    // static
    '/blog/**':    { isr: 3600 },          // ISR: regenerare la oră
    '/dashboard':  { ssr: true },          // SSR la fiecare request
    '/account/**': { ssr: false },         // SPA (client only)
  }
})

Single File Component (.vue) — formatul de bază

Spre deosebire de Next.js unde fiecare fișier e TSX cu React, Nuxt folosește Single File Components — un format .vue cu trei secțiuni distincte: 

<!-- components/UserCard.vue -->
<script setup lang="ts">
// Logica componentei (Composition API)
interface Props {
  name: string
  email: string
}
const props = defineProps<Props>()
const initials = computed(() => props.name.slice(0, 2).toUpperCase())
</script>

<template>
  <!-- Template HTML + Vue directives -->
  <div class="card">
    <span class="avatar">{{ initials }}</span>
    <div>
      <strong>{{ props.name }}</strong>
      <p>{{ props.email }}</p>
    </div>
  </div>
</template>

<style scoped>
/* CSS scoped — aplicat numai acestei componente */
.card { display: flex; gap: 1rem; padding: 1rem; border: 1px solid #eee; border-radius: 8px; }
.avatar { width: 40px; height: 40px; background: #00856d; color: white; border-radius: 50%; display: grid; place-items: center; }
</style>
SFC vs JSX
SFC (Single File Component) este mai familiară pentru cei cu background HTML/CSS — template-ul arată ca HTML cu atribute speciale (v-if, v-for, :prop, @event). JSX (React/Next.js) este JavaScript cu sintaxă asemănătoare HTML-ului — mai flexibil programatic, dar mai diferit față de HTML pur.
Exercițiu — Identificare echivalențe

Mapează fiecare concept Next.js la echivalentul Nuxt:

  1. 'use client' în Next.js → ? în Nuxt
  2. app/page.tsx în Next.js → ? în Nuxt
  3. Server Actions → ? în Nuxt
  4. next.config.js → ? în Nuxt
  5. useRouter() din next/navigation → ? în Nuxt

Răspuns: 1→nu există (Vue componente sunt isomorfice implicit), 2→pages/index.vue, 3→Server Routes în server/api/, 4→nuxt.config.ts, 5→useRouter() din Vue Router (auto-importat).

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Care este principala diferență între Nuxt 3 și Next.js?
Corect. Ambele framework-uri rezolvă aceleași probleme (SSR, file routing, optimizări, deploy) dar pentru ecosisteme diferite. Nuxt → Vue. Next.js → React. Conceptele sunt identice, sintaxa diferă.
Întrebarea 2 din 5 Ce avantaj oferă Nitro față de serverul Next.js?
Corect. Nitro este server engine-ul Nuxt 3 — abstractizează runtime-ul. Schimbi un preset în nuxt.config.ts și același cod se deployează pe Node.js, Vercel, Netlify sau Cloudflare Workers fără modificări.
Întrebarea 3 din 5 Ce sunt auto-imports în Nuxt 3?
Corect. Nuxt scanează directoarele components/ și composables/ și face disponibile toate fișierele fără import manual. Avantaj: mai puțin boilerplate. Dezavantaj: mai greu de urmărit de unde vine o funcție sau componentă.
Întrebarea 4 din 5 Ce conține un Single File Component (.vue)?
Corect. SFC-urile sunt formatul caracteristic Vue: logică, template și stiluri colocate în același fișier .vue. CSS scoped se aplică numai componentei respective — fără coliziuni. Toate trei secțiunile sunt opționale, dar template-ul e de obicei prezent.
Întrebarea 5 din 5 Cum configurezi o pagină să se randeze static (SSG) iar alta SSR în același proiect Nuxt?
Corect. routeRules este una dintre cele mai puternice funcții Nuxt 3 — permite „Hybrid Rendering". Poți seta prerender: true pentru pagini de marketing, isr: 3600 pentru blog, ssr: true pentru dashboard, totul în același proiect.
