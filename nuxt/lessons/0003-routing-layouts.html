Routing și Layouts în Nuxt 3

Navigare cu NuxtLink, rute dinamice, accesul la parametri, middleware de rute și layouts personalizate.

NuxtLink — navigare client-side

<NuxtLink> este echivalentul lui <Link> din Next.js. Extinde tag-ul <a> cu navigare SPA (fără full reload) și prefetching automat. 

<template>
  <nav>
    <NuxtLink to="/">Home</NuxtLink>
    <NuxtLink to="/about">Despre</NuxtLink>

    <!-- Rută dinamică -->
    <NuxtLink :to="`/blog/${post.slug}`">{{ post.title }}</NuxtLink>

    <!-- Cu obiect -->
    <NuxtLink :to="{ name: 'blog-slug', params: { slug: 'hello' } }">
      Post named route
    </NuxtLink>

    <!-- Dezactivare prefetch -->
    <NuxtLink to="/heavy-page" :prefetch="false">Pagină mare</NuxtLink>

    <!-- Link extern -->
    <NuxtLink href="https://nuxt.com" target="_blank">Nuxt.com</NuxtLink>
  </nav>
</template>
NuxtLink vs <a>
Folosește întotdeauna <NuxtLink> pentru navigare internă. <a href> face full page reload. <NuxtLink> navighează SPA, păstrând starea aplicației și evitând refetch-ul inutil.

useRoute — accesul la parametrii rutei

Composable-ul useRoute() oferă acces la parametrii, query string-ul și metadatele rutei curente: 

<!-- pages/blog/[slug].vue -->
<script setup lang="ts">
const route = useRoute()

// Parametrul dinamic din URL /blog/hello-world → slug = 'hello-world'
const slug = route.params.slug as string

// Query string: /blog/hello-world?page=2 → page = '2'
const page = route.query.page as string

// Fetch date cu parametrul
const { data: post } = await useFetch(`/api/posts/${slug}`)
</script>

<template>
  <article v-if="post">
    <h1>{{ post.title }}</h1>
    <div v-html="post.content" />
  </article>
</template>

useRouter — navigare programatică

<script setup lang="ts">
const router = useRouter()

async function handleLogin() {
  // ... logică autentificare
  await router.push('/dashboard')        // navighează, adaugă în history
  // await router.replace('/dashboard')  // navighează fără a adăuga în history
  // router.back()                       // înapoi
  // router.go(-2)                       // înapoi 2 pași
}
</script>

definePageMeta — metadata per pagină

definePageMeta() configurează comportamentul paginii: layout, middleware, titlu, etc.: 

<!-- pages/dashboard/index.vue -->
<script setup lang="ts">
definePageMeta({
  layout: 'dashboard',    // folosește layouts/dashboard.vue
  middleware: 'auth',     // rulează middleware/auth.ts înainte
  title: 'Dashboard',     // metadata pentru <head>
})
</script>

Middleware de rute

Middleware-ul de rute rulează pe client la navigare (nu pe server). Folosit pentru protecția rutelor, redirecturi condiționare, logging: 

// middleware/auth.ts
export default defineNuxtRouteMiddleware((to, from) => {
  const { loggedIn } = useSupabaseUser()  // sau orice altă verificare de auth

  if (!loggedIn.value) {
    return navigateTo('/login')
  }
})

Aplici middleware-ul pe o pagină prin definePageMeta sau global în nuxt.config.ts: 

// Aplicat pe o singură pagină:
definePageMeta({ middleware: 'auth' })

// Aplicat pe mai multe pagini:
definePageMeta({ middleware: ['auth', 'subscription'] })

// Global (toate paginile) — în nuxt.config.ts:
router: {
  middleware: ['auth']
}
Client vs Server Middleware
Middleware-ul din middleware/ rulează pe client la navigare. Middleware-ul din server/middleware/ rulează pe server la fiecare request HTTP. Sunt concepte diferite cu directoare diferite.

Layouts — mai mult control

Poți schimba layout-ul dinamic chiar și în funcție de date sau stare: 

<!-- pages/post/[id].vue -->
<script setup lang="ts">
const route = useRoute()
const { data: post } = await useFetch(`/api/posts/${route.params.id}`)

// Layout diferit pentru articolele premium
definePageMeta({
  layout: false  // dezactivează layout-ul automat
})
</script>

<template>
  <NuxtLayout :name="post?.isPremium ? 'premium' : 'default'">
    <article>{{ post?.title }}</article>
  </NuxtLayout>
</template>

NuxtPage și app.vue

Fișierul app.vue este root-ul aplicației. Implicit conține <NuxtPage /> care randează pagina curentă (similară cu {children} din root layout Next.js): 

<!-- app.vue — root component -->
<template>
  <div>
    <!-- NuxtLayout aplică layout-ul activ din layouts/ -->
    <NuxtLayout>
      <!-- NuxtPage randează pagina din pages/ corespunzătoare URL-ului -->
      <NuxtPage />
    </NuxtLayout>
  </div>
</template>
Exercițiu — Protejarea rutelor

Implementează protecția rutelor pentru un dashboard:

  1. Creează middleware/auth.ts care redirecționează la /login dacă nu e autentificat
  2. Aplică middleware-ul pe pages/dashboard/index.vue
  3. Creează layouts/dashboard.vue cu un sidebar simplu
  4. Asociază layout-ul dashboard paginii de dashboard prin definePageMeta

Hint: pentru verificarea autentificării, poți folosi temporar const user = useState('user') și verifica dacă e truthy.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 Cum accesezi parametrul id din URL-ul /users/42 în pagina pages/users/[id].vue?
Corect. useRoute() este composable-ul standard Vue Router — auto-importat în Nuxt. Returnează obiectul rutei curente cu params, query, path, hash etc. Nu ai nevoie de import.
Întrebarea 2 din 5 Ce face return navigateTo('/login') dintr-un middleware Nuxt?
Corect. navigateTo() în middleware face un redirect. Nuxt anulează navigarea la ruta protejată și navighează la /login. Poți pasa { replace: true } pentru a nu adăuga în history: navigateTo('/login', { replace: true }).
Întrebarea 3 din 5 Care este diferența dintre middleware din middleware/ și cel din server/middleware/?
Corect. middleware/auth.ts rulează în browser la fiecare navigare Vue Router. server/middleware/log.ts rulează în Nitro la fiecare request HTTP, inclusiv la cereri API directe. Scopuri diferite, contexte diferite.
Întrebarea 4 din 5 Ce rol are <NuxtPage /> în app.vue?
Corect. <NuxtPage /> este echivalentul lui {children} din Next.js root layout. Nuxt înlocuiește această componentă cu pagina activă din pages/ pe baza URL-ului. Fără ea, nicio pagină nu se randează.
Întrebarea 5 din 5 Cum aplici layout-ul layouts/admin.vue pe pagina pages/admin/users.vue?
Corect. definePageMeta({'{'} layout: 'admin' {'}'}) spune Nuxt să folosească layouts/admin.vue pentru acea pagină. Valoarea este numele fișierului din layouts/ fără extensie. Layout-ul default se aplică implicit dacă nu specifici.
