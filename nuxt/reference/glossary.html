Glosar Nuxt 3

Termenii esențiali din cursul Nuxt 3 — definiții concise pentru referință rapidă.

Arhitectură și randare

Auto-imports

Funcționalitate Nuxt prin care componentele din components/, composable-urile din composables/ și utilitarele Vue (ref, computed, watch) sunt disponibile în orice fișier fără import explicit. Eliminate nevoia de boilerplate.

→ Lecția 1, Lecția 2
Hydration

Procesul prin care Vue „preia controlul" HTML-ului pre-generat de server pe Nitro. Adaugă reactivitatea și event listeners în browser fără a re-randa vizibil conținutul. Esențial pentru SSR.

→ Lecția 1
Nitro

Motorul de server universal care alimentează Nuxt 3. Compilează aplicația într-un output adaptat platformei de hosting prin presets. Rulează server routes, middleware de server și SSR. Înlocuiește necesitatea unui server Express/Fastify separat.

→ Lecția 1, Lecția 5, Lecția 8
Nitro Preset

Configurare care transformă output-ul nuxt build în formatul specific platformei de hosting: funcții serverless Vercel, Edge Functions Netlify, un singur Worker Cloudflare, sau server Node.js clasic. Auto-detectat pe Vercel și Netlify.

→ Lecția 8
routeRules

Configurare în nuxt.config.ts care permite specificarea modului de randare per rută: ssr: true, prerender: true, isr: { revalidate: 60 }, redirect: '/new'. O singură aplicație poate mixa SSR, SSG și ISR.

→ Lecția 1
SSG — Static Site Generation

HTML generat la build time cu nuxt generate. Output-ul merge în .output/public/ și poate fi servit de orice CDN fără server. Ideal pentru site-uri de prezentare, documentație și blog-uri cu actualizări rare.

→ Lecția 1, Lecția 8
SSR — Server-Side Rendering

Comportamentul implicit Nuxt — HTML generat pe server la fiecare request cu date proaspete. Nitro rulează codul de server. Ideal pentru pagini cu date care se schimbă frecvent sau personalizate per utilizator.

→ Lecția 1

Vue 3 și SFC

Composition API

API-ul modern Vue 3 pentru scrierea logicii componentelor — ref(), computed(), watch(), onMounted() apelate direct în <script setup>. Înlocuiește Options API (Vue 2) și permite extragerea logicii în composables reutilizabile.

→ Lecția 1, Lecția 4
SFC — Single File Component

Formatul de fișier Vue (.vue) care reunește template HTML, script JS/TS și stiluri CSS în același fișier — în blocuri <template>, <script setup> și <style scoped>. Unitatea de bază a oricărui proiect Nuxt.

→ Lecția 1
<script setup>

Sintaxa modernă Vue 3 pentru Composition API — codul scris direct în blocul <script setup> este executat la crearea componentei. Variabilele și funcțiile declarate sunt automat disponibile în <template> fără return {}.

→ Lecția 1, Lecția 3
<style scoped>

Stiluri CSS limitate la componenta curentă — Nuxt adaugă automat un atribut unic pe elementele HTML și în selectori pentru a preveni conflictele de stiluri între componente.

→ Lecția 1

Routing și navigare

definePageMeta()

Macro Nuxt apelat în <script setup> al unei pagini pentru a configura meta-date: layout-ul folosit, middleware aplicat, titlul pentru SEO. Compilat la build time — nu poate conține valori reactive.

→ Lecția 3
Dynamic Route

Rută cu segment variabil definit prin paranteze pătrate în numele fișierului: [id].vue, [slug].vue. Valoarea din URL devine disponibilă prin useRoute().params.id. Catch-all cu [...slug].vue.

→ Lecția 3
navigateTo()

Funcție Nuxt pentru navigare programatică — funcționează identic pe server (redirect HTTP) și client (Vue Router push). Alternativa la useRouter().push(), recomandata pentru middleware și server routes.

→ Lecția 3
NuxtLink

Componenta Nuxt pentru navigare internă. Extinde <a> cu navigare client-side (fără full reload), prefetching automat și adăugarea clasei active pe link-ul curent. Echivalentul componentei <Link> din Next.js.

→ Lecția 3
NuxtPage

Componenta care randează pagina corespunzătoare rutei curente din directorul pages/. Folosită în app.vue sau în fișierele de layout. Echivalentul {children} din layout-urile Next.js.

→ Lecția 3
Route Middleware

Funcție care rulează înaintea navigării la o pagină, pe client (Vue Router). Definită în middleware/ sau inline cu definePageMeta({ middleware: 'auth' }). Diferit de server middleware (Nitro) care rulează pe server.

→ Lecția 3
useRoute()

Composable Vue Router pentru accesarea informațiilor despre ruta curentă: route.params (parametri dinamici), route.query (query string), route.path, route.name.

→ Lecția 3
useRouter()

Composable Vue Router pentru navigare programatică: router.push('/path'), router.replace(), router.back(). Pentru navigare simplă, navigateTo() este mai potrivit în Nuxt.

→ Lecția 3

Data fetching și composables

$fetch

Utilitar Nitro/ofetch pentru request-uri HTTP — funcționează identic pe server și client fără a verifica window. Parsează automat JSON, aruncă erori cu statusCode accesibil în catch. Folosit în stores Pinia și acțiuni one-shot.

→ Lecția 4
Composable

Funcție Vue 3 reutilizabilă cu prefix use care encapsulează logică reactivă. Poate conține ref, computed, lifecycle hooks. Fișierele din composables/ sunt auto-importate în tot proiectul.

→ Lecția 4
useAsyncData()

Composable Nuxt pentru data fetching cu control complet — caching prin cheie, refresh manual, watch pe dependențe. Returnează { data, pending, error, refresh }. Se execută pe server la SSR și este hidratat pe client fără a re-face request-ul.

→ Lecția 4
useFetch()

Shortcut care combină $fetch + useAsyncData — cel mai comun mod de data fetching în Nuxt. Returnează aceleași valori reactive. Cheia de cache este generată automat din URL și opțiuni.

→ Lecția 4
useState()

Composable Nuxt pentru state global simplu — un ref cu cheie unică, accesibil din orice componentă, sincronizat server-client la hydration. Alternativă la Pinia pentru state minimal fără actions sau getters.

→ Lecția 4

Server Routes — Nitro

createError()

Utilitar Nitro care creează erori HTTP standardizate: throw createError({ statusCode: 401, message: '...' }). Nitro le prinde și le transformă automat în răspunsuri HTTP cu statusCode și body JSON corespunzătoare.

→ Lecția 5
defineEventHandler()

Funcția Nitro cu care definești un handler pentru un server route. Primește un event care conține request, response și context. Valoarea returnată este serializată automat ca JSON.

→ Lecția 5
event.context

Obiect disponibil pe fiecare request Nitro, destinat partajării de date între middleware de server și endpoint-uri. Middleware setează event.context.user = user; endpoint-urile citesc event.context.user. Izolat per request.

→ Lecția 5
getQuery()

Utilitar Nitro pentru citirea parametrilor din query string-ul URL: const { page } = getQuery(event) pentru /api/tasks?page=2. Returnează un obiect cu toate perechile cheie-valoare.

→ Lecția 5
getRouterParam()

Utilitar Nitro pentru citirea parametrilor dinamici din ruta URL: const id = getRouterParam(event, 'id') în fișierul server/api/tasks/[id].get.ts.

→ Lecția 5
readBody()

Utilitar Nitro async care citește și parsează automat body-ul JSON al unui request POST/PATCH: const body = await readBody(event). Similar există readFormData(event) pentru formulare multipart.

→ Lecția 5
server/api/ vs server/middleware/

server/api/ — endpoint-uri accesibile la /api/*, fiecare fișier răspunde la o metodă HTTP specificată prin sufix (.get.ts, .post.ts). server/middleware/ — cod care rulează înaintea fiecărui request, pe server.

→ Lecția 5

Supabase

@nuxtjs/supabase

Modulul oficial Nuxt pentru Supabase. Configurează automat clientul Supabase, injectează composable-urile în auto-imports, gestionează sesiunile via cookies și include middleware de autentificare cu redirect configurabil.

→ Lecția 6
anon key

Cheia publică Supabase (SUPABASE_KEY în .env) — sigură să fie în browser. Permite operații de date filtrate de politicile RLS. Nu bypasează securitatea — RLS rămâne activ.

→ Lecția 6
RLS — Row Level Security

Mecanism PostgreSQL care definește politici de acces la nivel de rând. Utilizatorii văd și modifică numai rândurile permise de politici (ex: auth.uid() = user_id). Activat cu ALTER TABLE x ENABLE ROW LEVEL SECURITY.

→ Lecțiile 6, 9
serverSupabaseClient(event)

Utilitar @nuxtjs/supabase pentru obținerea unui client Supabase cu sesiunea utilizatorului în context server Nitro. Diferit de useSupabaseClient() care există numai în componente Vue.

→ Lecția 6
serverSupabaseUser(event)

Utilitar @nuxtjs/supabase care verifică și returnează utilizatorul autentificat în context server Nitro. Returnează null dacă nu există sesiune validă — folosit pentru a proteja server routes.

→ Lecția 6
service_role key

Cheia privată Supabase cu acces total la DB, ocolind RLS. Niciodată expusă pe client. Stocată în SUPABASE_SERVICE_ROLE_KEY în .env — folosită numai pe server pentru operații administrative.

→ Lecția 6
useSupabaseClient()

Composable injectat de @nuxtjs/supabase — disponibil în componente Vue pentru query-uri Supabase cu sesiunea utilizatorului curent și RLS activ. Nu există în contextul server Nitro.

→ Lecția 6
useSupabaseUser()

Composable injectat de @nuxtjs/supabase care returnează un Ref<User | null> reactiv — actualizat automat la login/logout. Poate fi observat cu watch pentru redirect automat.

→ Lecția 6

State management și module

Modul Nuxt

Pachet npm care se injectează în build-ul Nuxt — adaugă auto-imports, modifică configurația Vite, adaugă server routes sau configurează runtimeConfig. Înregistrat în nuxt.config.ts în array-ul modules: [].

→ Lecția 7
Pinia

Librăria oficială de state management Vue 3. Store-urile sunt definite cu defineStore(), state cu ref(), getters cu computed() și actions cu funcții async. Cu @pinia/nuxt, store-urile sunt auto-importate.

→ Lecția 7, Lecția 9

Configurare și variabile de mediu

.env

Fișier pentru variabile de mediu locale — gitignored, nu se comite niciodată. Nuxt citește automat variabilele la pornire. Pe Vercel sau Netlify, variabilele se adaugă manual în dashboard-ul platformei.

→ Lecțiile 6, 8
nuxt.config.ts

Fișierul central de configurare Nuxt — definește module, CSS global, runtimeConfig, app.head, opțiuni Nitro, TypeScript strict și configurări per-modul. Înlocuiește next.config.js din Next.js.

→ Lecția 7
runtimeConfig

Sistemul Nuxt pentru variabile de mediu — runtimeConfig.* sunt private (numai server), runtimeConfig.public.* sunt expuse și pe client. Variabilele .env cu prefix NUXT_ overrideaza automat valorile din config.

→ Lecția 7
NUXT_PUBLIC_ prefix

Prefixul variabilelor de mediu care overrideaza runtimeConfig.public.* — disponibile atât pe server cât și pe client. Echivalentul prefixului NEXT_PUBLIC_ din Next.js. Fără prefix, variabila suprascrie o valoare privată (server only).

→ Lecția 7
