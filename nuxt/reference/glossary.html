Glosar Nuxt 3

Termeni cheie din cursul Nuxt 3 — de la auto-imports la Nitro presets. Referință rapidă pentru review.

App Router
Sistemul de rutare bazat pe directorul pages/ — fiecare fișier .vue devine automat o rută URL. Lecția 3 — Routing și Layouts
Auto-imports
Funcționalitate Nuxt prin care composable-urile din composables/, componentele din components/ și utilitarele Vue (ref, computed) sunt disponibile fără import explicit. Lecția 1
Composable
Funcție Vue 3 reutilizabilă cu prefix use care encapsulează logică reactivă. Exemple: useFetch(), useRoute(), useSupabaseUser(). Lecția 4
createError()
Utilitar Nitro pentru a arunca erori HTTP standardizate din server routes: throw createError({ statusCode: 401, message: '...' }). Nitro le transformă automat în răspunsuri HTTP corespunzătoare. Lecția 5
defineEventHandler()
Funcția Nitro cu care definești un server route handler. Primește un event (request, response, context) și returnează datele serializate automat ca JSON. Lecția 5
definePageMeta()
Macro Nuxt apelat în <script setup> pentru a configura meta-date ale unei pagini: layout-ul folosit, middleware aplicat, titlul paginii etc. Lecția 3
event.context
Obiect disponibil pe fiecare request Nitro, destinat pentru a partaja date între middleware de server și endpoint-uri. Middleware setează date pe el (ex: user autentificat), endpoint-urile le citesc. Lecția 5
getQuery()
Utilitar Nitro pentru a citi parametrii din query string-ul URL-ului: const { page } = getQuery(event) pentru /api/tasks?page=2. Lecția 5
getRouterParam()
Utilitar Nitro pentru a citi parametrii dinamici din ruta URL-ului: const id = getRouterParam(event, 'id') pentru /api/tasks/[id]. Lecția 5
Nitro
Motorul de server universal care alimentează Nuxt 3. Generează output adaptat platformei de hosting prin presets — Vercel, Netlify, Cloudflare Workers, Node.js. Rulează server routes, middleware de server și SSR. Lecția 1, Lecția 8
Nitro Preset
Configurare care transformă output-ul nuxt build în formatul specific platformei de hosting. Detectat automat pe Vercel/Netlify sau specificat manual în nitro: { preset: 'cloudflare-workers' }. Lecția 8
NuxtLink
Componenta de navigare Nuxt — înlocuiește <a href> pentru rute interne. Activează client-side routing (SPA), prefetching automat și active class pe link-ul curent. Echivalentul <Link> din Next.js. Lecția 3
NuxtPage
Componenta care randează pagina curentă corespunzătoare rutei — folosită în app.vue sau în layout-uri. Echivalentul {children} din Next.js layout-uri. Lecția 3
Pinia
Librăria oficială de state management Vue 3. Definești store-uri cu defineStore(), state cu ref(), getters cu computed() și actions cu funcții async. Cu modulul @pinia/nuxt, store-urile sunt auto-importate. Lecția 7
readBody()
Utilitar Nitro async pentru a citi și parsa automat body-ul JSON dintr-un request POST/PATCH: const body = await readBody(event). Lecția 5
runtimeConfig
Sistemul Nuxt pentru variabile de mediu. runtimeConfig.* = privat (numai server); runtimeConfig.public.* = expus și pe client. Variabilele .env cu prefixul NUXT_ overrideaza automat config-ul. Lecția 7
server/api/
Directorul pentru server routes Nitro. Fișierele numite *.get.ts, *.post.ts, *.patch.ts, *.delete.ts răspund numai metodei HTTP corespunzătoare. Accesibile la /api/*. Lecția 5
server/middleware/
Middleware Nitro care rulează înaintea fiecărui request HTTP (inclusiv API routes). Diferit de middleware/ Vue Router (care rulează pe client la navigare SPA). Lecția 5
serverSupabaseClient(event)
Utilitar @nuxtjs/supabase pentru a obține un client Supabase cu sesiunea utilizatorului în context server (Nitro). Diferit de useSupabaseClient() care există numai în componente Vue. Lecția 6
serverSupabaseUser(event)
Utilitar @nuxtjs/supabase care verifică și returnează utilizatorul autentificat în context server. Returnează null dacă nu există sesiune validă. Lecția 6
SFC (Single File Component)
Formatul de fișier Vue (.vue) care conține template HTML, script JS/TS și stiluri CSS în același fișier — separat în blocuri <template>, <script setup>, <style scoped>. Lecția 1
$fetch
Utilitar Nitro/Nuxt pentru request-uri HTTP — funcționează identic pe server și client (nu necesită window check). Parsează automat JSON, aruncă erori cu statusCode accesibil. Lecția 4
useAsyncData()
Composable Nuxt pentru data fetching cu caching și control complet. Returnează { data, pending, error, refresh }. Se execută pe server la SSR și este hidratat pe client. Lecția 4
useFetch()
Combinație de $fetch + useAsyncData — shortcut pentru cel mai comun caz de data fetching în Nuxt. Returnează aceleași valori reactive și se execută isomorf (server + client). Lecția 4
useRoute()
Composable Vue Router pentru accesarea informațiilor despre ruta curentă: route.params, route.query, route.path. Lecția 3
useRouter()
Composable Vue Router pentru navigare programatică: router.push('/path'), router.replace(), router.back(). Alternativ, folosești navigateTo('/path') din Nuxt. Lecția 3
useSupabaseClient()
Composable injectat de @nuxtjs/supabase — disponibil în componente Vue pentru query-uri Supabase cu sesiunea utilizatorului curent și RLS activ. Lecția 6
useSupabaseUser()
Composable injectat de @nuxtjs/supabase care returnează un Ref<User | null> reactiv — actualizat automat la login/logout. Lecția 6
Vue 3 Composition API
API-ul modern Vue 3 pentru scrierea logicii componentelor — ref(), computed(), watch(), onMounted() etc. apelate direct în <script setup>. Alternativa la Options API (Vue 2). Lecția 1
useState()
Composable Nuxt pentru state global simplu — un ref cu cheie, accesibil din orice componentă, sincronizat server-client. Alternativă la Pinia pentru state simplu. Lecția 4
