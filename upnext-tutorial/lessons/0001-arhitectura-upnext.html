Ce este UpNext

UpNext este o aplicație personală de descoperire filme/seriale: recomandări, watchlist, rating-uri și căutare semantică. E scrisă cu Nuxt 4, folosește Nuxt UI v4 + Tailwind CSS v4 pentru interfață, Supabase pentru autentificare și bază de date, și TMDB (The Movie Database) ca sursă de date pentru filme/seriale. E deployată pe Vercel.

Structura reală de foldere (Nuxt 4)

Nuxt 4 mută sursa aplicației din rădăcina proiectului într-un folder app/ dedicat. Dacă ai văzut Nuxt 3 în cursul nuxt/, aceasta e diferența majoră — acolo pages/ era în rădăcină, aici e app/pages/.

Notă
Nu există folder layouts/ în acest proiect. Aplicația are o singură „coajă" vizuală (navbar + footer), definită direct în app/app.vue — nu are nevoie de layout-uri multiple. Vom reveni pe asta în Lecția 2.

Un detaliu important: ssr: false

Spre deosebire de comportamentul implicit Nuxt (server-side rendering), UpNext rulează în SPA mode — vezi ssr: false în nuxt.config.ts. Asta înseamnă:

  • Browserul primește un HTML gol la prima încărcare, apoi Vue randează totul client-side
  • Nu există hydration — nu e nevoie să te gândești la diferențe server/client în componente
  • server/api/ tot rulează pe server (Nitro) — SPA mode afectează doar randarea paginilor, nu API-ul
  • Motivul tipic pentru SPA într-o aplicație ca asta: conținutul e personalizat per utilizator (watchlist, recomandări) — SEO pe pagini publice de filme contează mai puțin decât simplitatea SPA

Cum pornești aplicația local

cd D:\projects\movies
pnpm install
cp .env.example .env    # completează SUPABASE_URL, SUPABASE_KEY, TMDB_API_KEY

Comenzi utile: pnpm build (build producție), pnpm lint, pnpm typecheck.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 În Nuxt 4, unde se află pages/, components/ și composables/ în UpNext?
Corect. Nuxt 4 introduce app/ ca director sursă dedicat pentru tot ce ține de client — o schimbare structurală față de Nuxt 3, unde aceleași foldere stăteau în rădăcină.
Întrebarea 2 din 5 Ce face concret ssr: false în nuxt.config.ts al UpNext?
Corect. ssr: false afectează doar randarea paginilor Vue (devin SPA, fără HTML server-rendered). Nitro și server/api/ nu sunt afectate — endpoint-urile tot rulează pe server, exact ca înainte.
Întrebarea 3 din 5 De ce nu există folder layouts/ în UpNext?
Corect. Sistemul de layouts există și funcționează normal în SPA mode — pur și simplu UpNext nu are nevoie de mai multe layout-uri, așa că navbar-ul și footer-ul sunt scrise direct în app.vue, care e mereu randat.
Întrebarea 4 din 5 Unde e stocată cheia TMDB API și de ce e sigură acolo?
Corect. runtimeConfig.tmdbApiKey (fără public) e disponibil doar prin useRuntimeConfig() apelat în cod de server — niciodată trimis către browser în bundle-ul JS.
Întrebarea 5 din 5 Ce se întâmplă când deschizi /movie/27205 — care e ordinea corectă?
Corect. Pagina apelează întotdeauna propriul server (/api/movies/[id]), niciodată TMDB direct. Handler-ul Nitro face request-ul real către TMDB, folosind cheia din runtimeConfig.
