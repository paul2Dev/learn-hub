MediaCard.vue — walkthrough complet

Această componentă e folosită peste tot: pe homepage, în discover, în „filme similare". Poate afișa un film, un serial, un actor sau o colecție, în funcție de item.media_type.

app/components/MediaCard.vue — <script setup>
interface MediaItem {
  id: number
  media_type?: 'movie' | 'tv' | 'person' | 'collection'
  title?: string
  name?: string
  release_date?: string
  first_air_date?: string
  poster_path?: string | null
  profile_path?: string | null
  vote_average?: number
  overview?: string
}

const { item, showWatchlistButton = true } = defineProps<{ item: MediaItem, showWatchlistButton?: boolean }>()

const { poster } = useTmdbImage()
const user = useSupabaseUser()
const { has, toggle } = useWatchlist()

const mediaType = computed(() => item.media_type ?? 'movie')
const displayTitle = computed(() => item.title ?? item.name ?? '—')
const displayYear = computed(() => (item.release_date ?? item.first_air_date)?.slice(0, 4) ?? '—')
const imagePath = computed(() => item.poster_path ?? item.profile_path ?? null)
const imageUrl = computed(() => poster(imagePath.value))
const link = computed(() => {
  if (mediaType.value === 'tv') return `/tv/${item.id}`
  if (mediaType.value === 'person') return `/person/${item.id}`
  if (mediaType.value === 'collection') return `/collection/${item.id}`
  return `/movie/${item.id}`
})

const isWatchable = computed(() => mediaType.value === 'movie' || mediaType.value === 'tv')
const inWatchlist = computed(() => has(item.id, mediaType.value))
const togglingWatchlist = ref(false)

async function onToggleWatchlist(event: Event) {
  event.preventDefault()
  event.stopPropagation()
  if (!user.value || togglingWatchlist.value) return
  togglingWatchlist.value = true
  try {
    await toggle(item.id, mediaType.value as 'movie' | 'tv', item)
  } finally {
    togglingWatchlist.value = false
  }
}

Câteva lucruri de observat, direct din codul de mai sus:

  • defineProps<{...}>() — props tipate cu generic TypeScript, fără runtime validator separat
  • title ?? name — filmele au title, serialele au name; componenta acoperă ambele fără v-if separat
  • useTmdbImage(), useSupabaseUser(), useWatchlist() — toate composables, apelate direct, fără import
  • link e un computed care alege ruta corectă în funcție de tip — de asta un singur component servește 4 tipuri de entități
Auto-import
Nici MediaCard.vue, nici pagina care îl folosește nu au vreun import pentru useTmdbImage, useSupabaseUser sau useWatchlist. Nuxt scanează app/composables/ și expune orice funcție cu prefix use global, în orice fișier din app/.

RatingButton.vue — pattern v-model

Componentele care se comportă ca un input (aici: rating 1-5, redus la 3 opțiuni) folosesc defineProps + defineEmits pentru v-model:

app/components/RatingButton.vue
defineProps<{ modelValue: number | null, loading?: boolean }>()
const emit = defineEmits<{ 'update:modelValue': [value: number] }>()

// în template:
// @click="emit('update:modelValue', option.value)"

Folosit dintr-o pagină de detalii film:

<RatingButton
  :model-value="userRating"
  :loading="loadingWatched"
  @update:model-value="setRating"
/>
Față de React
v-model e echivalentul Vue pentru un input controlat React (value + onChange), dar cu o singură directivă în loc de două props separate. Vue leagă automat modelValue (prop) cu update:modelValue (event) când scrii v-model="x" pe componentă.

Exercițiu — Citește codul

În MediaCard.vue, ce se întâmplă dacă item.media_type este undefined?

  1. Uită-te la linia const mediaType = computed(() => item.media_type ?? 'movie')
  2. Ce rută generează link pentru un item fără media_type?

Răspuns: mediaType devine 'movie' implicit, deci link devine /movie/{id}. E de ce MovieCard.vue (folosit doar pentru filme) poate omite complet câmpul media_type din datele pe care le primește.

Verifică-ți înțelegerea

Cinci întrebări. Dă clic pe un răspuns — feedback-ul este imediat.

Întrebarea 1 din 5 De ce nu apare niciun import MediaCard from ... în paginile care îl folosesc?
Corect. Auto-importul de componente e o funcționalitate Nuxt standard — orice fișier .vue din app/components/ devine disponibil global, indiferent de ssr.
Întrebarea 2 din 5 Cum decide MediaCard.vue spre ce rută să facă link — /movie, /tv, /person sau /collection?
Corect. Un singur computed(() => {...}) cu if-uri pe mediaType.value generează ruta corectă — nu există 4 componente separate.
Întrebarea 3 din 5 Ce înseamnă v-model="userRating" pe <RatingButton>?
Corect. v-model e sintaxă compactă pentru :model-value="userRating" @update:model-value="userRating = $event". Componenta trebuie doar să declare props/emits cu numele potrivite.
Întrebarea 4 din 5 De ce MediaCard.vue folosește item.title ?? item.name în loc de un singur câmp title?
Corect. TMDB folosește title pentru filme și name pentru seriale/persoane/colecții. ?? acoperă ambele cazuri într-o singură expresie.
Întrebarea 5 din 5 De ce onToggleWatchlist apelează event.preventDefault() și event.stopPropagation()?
Corect. Tot cardul (<NuxtLink :to="link">) e clickabil pentru navigare. Fără preventDefault/stopPropagation, click pe butonul de watchlist ar naviga și la pagina filmului, nu doar ar bifa watchlist-ul.
