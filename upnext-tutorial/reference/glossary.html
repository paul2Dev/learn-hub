Glosar UpNext

Termenii esențiali din codul real al aplicației — definiții concise pentru referință rapidă.

Structură și rute

app/ (Nuxt 4)

Director sursă dedicat introdus în Nuxt 4 — pages/, components/, composables/, middleware/ stau sub app/ în loc de rădăcina proiectului, ca în Nuxt 3.

→ Lecția 1
ssr: false

Opțiune din nuxt.config.ts care dezactivează server-side rendering pentru pagini — aplicația devine SPA (Single Page Application). Nu afectează server/api/, care rulează normal pe Nitro.

→ Lecția 1
definePageMeta()

Macro apelat în <script setup> al unei pagini pentru a configura middleware sau layout. În UpNext: { middleware: 'auth' } pe paginile protejate, { layout: false } pe login/confirm.

→ Lecția 2, Lecția 6
Dynamic Route ([id].vue)

Fișier de pagină cu segment variabil în paranteze pătrate. UpNext are patru: movie/[id].vue, tv/[id].vue, person/[id].vue, collection/[id].vue — toate urmează același pattern de citire a parametrului cu useRoute().params.id.

→ Lecția 2, Lecția 10
app.vue ca shell unic

UpNext nu are folder layouts/ — navbar-ul, footer-ul și <NuxtPage /> stau direct în app/app.vue, tratat ca singura „coajă" vizuală a aplicației.

→ Lecția 1, Lecția 2

Componente și composables

Auto-import

Componentele din app/components/ și composables din app/composables/ sunt disponibile în orice fișier fără import explicit — Nuxt le scanează la build time.

→ Lecția 3, Lecția 4
MediaCard.vue

Componentă universală pentru filme, seriale, actori și colecții — un singur computed decide ruta și afișarea corectă pe baza câmpului media_type.

→ Lecția 3, Lecția 10
useState() vs ref()

useState('key', ...) partajează aceeași valoare între toate componentele care o apelează cu aceeași cheie (ex: useWatchlist). ref() simplu creează o valoare izolată per apel (ex: useMediaActions).

→ Lecția 4
useWatchlist()

Composable cu state global (useState('watchlist-ids', ...)) — un Set de chei "tip:id", sincronizat automat între navbar și orice MediaCard de pe ecran.

→ Lecția 4, Lecția 8
useMediaActions()

Composable cu state local (per apel) pentru pagina de detalii a unui film/serial — gestionează watchlist, rating, stări de loading și toast-uri de eroare pentru acea entitate specifică.

→ Lecția 4

Server API — Nitro

tmdbFetch()

Utilitar de server (server/utils/tmdb.ts) care centralizează request-urile către TMDB — citește config.tmdbApiKey din runtimeConfig și adaugă parametrii comuni. Folosit de toate endpoint-urile movies/ și tv/.

→ Lecția 5
server/middleware/ vs server/api/

server/middleware/security.ts rulează pe orice request către server, indiferent de rută. server/api/*.get.ts răspunde la o rută și metodă HTTP specifice.

→ Lecția 5, Lecția 8
serverSupabaseUser(event)

Utilitar @nuxtjs/supabase care verifică sesiunea din context Nitro — returnează null dacă utilizatorul nu e autentificat. Prima verificare din orice endpoint server/api/user/.

→ Lecția 5, Lecția 6, Lecția 8
Validare Zod în endpoint

Fiecare endpoint server/api/user/*.post.ts definește o schemă Zod la începutul fișierului și apelează schema.safeParse(await readBody(event)) — date invalide → eroare 400, înainte de orice query Postgres.

→ Lecția 5, Lecția 8
Promise.allSettled()

Variantă de Promise.all care așteaptă toate promisiunile fără să eșueze la primul reject — folosită în motorul de recomandări pentru a tolera eșecul individual al unor cereri TMDB paralele.

→ Lecția 9

Supabase și date

RLS — Row Level Security

Mecanism Postgres care restricționează accesul la rânduri pe baza unor policies (ex: auth.uid() = user_id). A doua linie de apărare, independentă de verificările din codul serverului.

→ Lecția 7, Lecția 8
Trigger on_auth_user_created

Funcție Postgres care creează automat o linie în profiles imediat ce Supabase inserează un rând nou în auth.users — garantează că orice user autentificat are un profil, fără cod explicit în aplicație.

→ Lecția 6, Lecția 7
Migrație SQL versionată

Fișier numerotat în supabase/migrations/ care descrie o schimbare incrementală de schemă. Niciodată editat retroactiv — schimbările ulterioare (ex: 004 repară un constraint din 001) sunt fișiere noi.

→ Lecția 7
tmdb_data (jsonb)

Coloană jsonb pe watchlist/watched care cache-uiește o copie a răspunsului TMDB la momentul salvării — evită un fetch nou la fiecare afișare a watchlist-ului, cu compromisul unor date potențial ușor stale.

→ Lecția 7
pgvector / vector(1536)

Extensie Postgres care adaugă tipul de date vector și operatori de distanță (<=> pentru cosine distance). media_embeddings.embedding stochează un vector de 1536 dimensiuni per film/serial, generat de OpenAI.

→ Lecția 7, Lecția 11
RPC (Remote Procedure Call) Supabase

supabase.rpc('nume_funcție', params) apelează o funcție Postgres definită cu SECURITY DEFINER — folosit când o operație trebuie să fie atomică (ex: increment_search_usage) sau necesită logică SQL complexă (ex: search_media_by_embedding).

→ Lecția 11

Auth

middleware.global.ts

Fișier din app/middleware/ cu sufixul .global.ts care rulează automat pe orice navigare, fără să fie declarat explicit în definePageMeta. UpNext folosește onboarding.global.ts pentru redirect la primul login.

→ Lecția 6
supabase.redirectOptions.exclude

Listă de rute în nuxt.config.ts care rămân publice, exceptate de la middleware-ul implicit al modulului @nuxtjs/supabase care ar redirecționa altfel orice pagină neautentificată la login.

→ Lecția 6
signInWithOAuth()

Metodă a clientului Supabase care inițiază fluxul OAuth (aici: Google) — redirecționează browserul la providerul extern, apoi înapoi la redirectTo specificat (/confirm în UpNext).

→ Lecția 6

Configurare

runtimeConfig vs runtimeConfig.public

Valorile la rădăcina runtimeConfig sunt private (server-only) — ex: tmdbApiKey. Valorile sub public ajung și în bundle-ul client — ex: featureSemanticSearch.

→ Lecția 1, Lecția 11
Feature flag

Variabilă de mediu boolean (FEATURE_SEMANTIC_SEARCH) care activează/dezactivează o funcționalitate întreagă fără a elimina codul — permite rularea aceleiași baze de cod cu sau fără infrastructura suplimentară necesară (aici: embeddings + cheie OpenAI).

→ Lecția 11
