Explorarea și Înțelegerea Codului

Cum să înțelegi un codebase necunoscut în minute, nu ore — și cum să-l faci pe Claude Code mai util cu CLAUDE.md.

Prima zi pe un proiect nou

Ești nou pe un proiect. Există sute de fișiere, o arhitectură neclară și zero documentație. Fă asta:

cd /proiect-nou
claude
> Explică-mi structura acestui proiect. Ce face, cum e organizat și care sunt fișierele principale?

Claude Code va citi structura directoarelor, va inspecta fișierele cheie și îți va returna o hartă mentală a proiectului — în câteva secunde.

Urmărire: Dacă răspunsul e prea general, adâncește-te: "Explică-mi cum funcționează modulul de autentificare." sau "Trasează ce se întâmplă când un utilizator face login."

Întrebări utile de explorare

Iată patterns care funcționează bine în practică:

> Cum funcționează funcția `processPayment` din src/billing.js?

> Unde se validează input-ul utilizatorului în acest proiect?

> Ce dependențe externe folosim și la ce servesc?

> Există cod duplicat sau funcții similare în proiect?

> Explică-mi fluxul de date de la request HTTP până la baza de date.

Claude Code va căuta activ în fișierele proiectului, nu va ghici. Răspunsurile sunt ancorate în codul real.

CLAUDE.md — Memoria Permanentă a Proiectului

Fiecare sesiune nouă pornește fără memorie. Dar dacă creezi un fișier CLAUDE.md în rădăcina proiectului, Claude Code îl va citi automat la fiecare sesiune.

Acesta e locul unde documentezi ce nu se poate vedea din cod:

# CLAUDE.md

## Stack
- Laravel 11, PHP 8.3
- PostgreSQL 16
- Vite + Vue 3 pentru frontend

## Comenzi importante
- `php artisan serve` — pornește serverul local
- `npm run dev` — compilează assets
- `php artisan test` — rulează testele

## Convenții
- Toate API route-urile sunt versionate: /api/v1/...
- Nu modifica fișierele din vendor/
- Branch-urile de feature urmează formatul: feat/TICKET-123-descriere

## Lucruri de evitat
- Nu folosi `dd()` în producție
- Nu comite fișierele .env

Sfat: Cere chiar lui Claude Code să genereze un prim draft de CLAUDE.md pentru proiectul tău: "Creează un CLAUDE.md pentru acest proiect bazat pe ce ai văzut."

CLAUDE.md ierarhic

Poți avea CLAUDE.md la mai multe niveluri:

  • ~/.claude/CLAUDE.md — se aplică tuturor proiectelor (preferințele tale globale)
  • /proiect/CLAUDE.md — specific proiectului
  • /proiect/src/CLAUDE.md — specific unui submodul sau director

Toate sunt citite automat, de sus în jos. Global + proiect + submodul = context complet.

Căutarea în codebase

Când știi ce cauți, poți fi mai direct:

> Găsește toate locurile unde se apelează `sendEmail()`

> Care fișiere importă modulul `AuthService`?

> Există vreo referință la API-ul deprecat `v1/users` în cod?

Claude Code folosește grep și glob intern — e mai rapid decât să cauți manual în IDE.
