Git Workflow Integrat

Commit, PR, diff și review — cum Claude Code face din git un flux fluid, nu un obstacol.

Commit-uri cu mesaje bune, automat

Cel mai folosit pattern git cu Claude Code:

> Fă commit cu modificările curente

Claude Code va:

  1. Rula git status și git diff pentru a vedea ce s-a schimbat
  2. Citi ultimele commit-uri pentru a respecta stilul de mesaje al proiectului
  3. Propune un mesaj de commit descriptiv și contextual
  4. Stagea fișierele relevante și creeze commit-ul după aprobare

Avantaj real: Mesajele de commit generate de Claude Code descriu ce și de ce — nu doar "fix bug" sau "update files". Istoricul git devine lizibil.

Pull Requests

După ce ai terminat un feature pe un branch:

> Creează un Pull Request pentru branch-ul curent față de main.
  Descrie ce am implementat și adaugă un checklist de testare.

Claude Code va:

  • Analiza toate commit-urile de pe branch față de main
  • Scrie un titlu concis și un body structurat (summary + test plan)
  • Crea PR-ul via gh pr create (dacă ai GitHub CLI instalat)

Prerechizită: GitHub CLI (gh) trebuie instalat și autentificat: gh auth login. Descarcă de la cli.github.com.

Analiza diff-urilor și istoricului

> Ce s-a schimbat față de main în branch-ul curent?

> Explică-mi ce a făcut ultimul commit

> Care commit a introdus funcția validateUser?

> Ce s-a modificat în src/auth/ în ultimele 2 săptămâni?

Claude Code rulează comenzile git necesare și traduce output-ul tehnic în explicații utile.

Guardrails — ce Claude Code refuză să facă

Claude Code are protecții încorporate pentru operații git periculoase:

  • Nu face force push pe main/master fără avertisment explicit
  • Nu rulează git reset --hard fără confirmare
  • Nu face push la remote fără aprobare
  • Nu bypassează hooks (--no-verify) decât dacă îi ceri explicit

Aceste protecții există pentru că operațiile sunt greu de anulat. Dacă ai nevoie să le faci, Claude Code te va avertiza și va cere confirmare explicită.

Rezolvarea conflictelor de merge

> Am conflicte de merge în aceste fișiere. Ajută-mă să le rezolv
  păstrând logica din branch-ul meu dar integrând și schimbările din main:

  CONFLICT (content): Merge conflict in src/services/auth.js

Claude Code va citi fișierul cu markerii de conflict (<<<<<<<, =======, >>>>>>>), va înțelege intențiile ambelor versiuni și va propune o rezoluție care le îmbină corect.
