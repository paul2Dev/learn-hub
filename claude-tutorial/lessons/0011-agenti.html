Claude Agents — Instanțe Specializate în Paralel

Când o singură sesiune nu e suficientă — agenți care lucrează în paralel, cu contexte izolate, pentru task-uri complexe.

Ce este un Agent?

Un agent Claude Code este o instanță separată de Claude, cu propriul context și unelte, pornită din sesiunea principală pentru a rezolva o subsarcină. Agentul lucrează independent, returnează rezultatul, și sesiunea principală continuă cu aceste informații.

Gândește-te la o sesiune normală ca la un programator care lucrează singur. Un agent e ca și când ai delega un task unui coleg: îi dai instrucțiuni clare, el lucrează independent și îți aduce rezultatul, tu continui cu treaba ta în paralel.

Principiu cheie: Fiecare agent pornește cu un context gol — nu "știe" ce s-a discutat în sesiunea principală dacă nu îi trimiți explicit informațiile relevante. Asta e o caracteristică, nu un bug: contextul izolat = mai puțin zgomot, mai multă precizie.

De ce să folosești Agents?

  • Paralelism — Poți porni mai mulți agenți simultan pentru task-uri independente. Un agent face research în codebase, altul scrie teste, altul generează documentație. Economisești timp real.
  • Izolarea contextului — Un context prea lung devine imprecis. Un agent proaspăt, cu instrucțiuni clare și contextul exact necesar, produce rezultate mai bune decât o sesiune lungă "obosită".
  • Specializare — Agenți diferiți pentru task-uri diferite: Explore pentru localizat cod, Plan pentru design, code-reviewer pentru review. Fiecare e optimizat pentru scopul lui.
  • Protejarea sesiunii principale — Dacă task-ul ar "polua" sesiunea principală cu zeci de fișiere citite sau output-uri mari, delegarea la un agent ține sesiunea principală curată și rapidă.

Agenți built-in în Claude Code

Claude Code vine cu mai mulți agenți specializați pe care îi poți cere explicit:

  • Explore — Agent de căutare read-only. Localizează fișiere, simboluri, pattern-uri în codebase fără să modifice nimic. Rapid și precis pentru întrebări de tipul "unde e definit X?"
  • Plan — Arhitect de soluții. Primește un task, returnează un plan de implementare: fișiere afectate, ordine de modificare, riscuri. Nu scrie cod singur.
  • general-purpose — Agent universal pentru task-uri complexe multi-step care nu se potrivesc unui agent specific.
  • code-reviewer — Specializat în review de cod. Analizează diff-ul, găsește bug-uri, propune îmbunătățiri. Opinie independentă față de sesiunea principală — util ca "a doua pereche de ochi".

Cum îi invoci: Claude pornește automat agenții potriviți pe baza task-ului. Poți cere și explicit: "Folosește un agent Explore pentru a găsi toate endpoint-urile API din proiect."

Agenți în fundal (Background Agents)

Un agent poate rula în fundal în timp ce tu continui să lucrezi. Claude te notifică automat când agentul termină.

> Pornește un agent în fundal care să analizeze toate fișierele
   din src/ și să listeze funcțiile fără teste unit. Continuă
   când termină, nu te opri să aștepți.

# Claude pornește agentul și tu poți continua cu alte task-uri.
# Când agentul termină, primești notificare automată cu rezultatele.

Util pentru:

  • Analize de codebase care durează câteva minute
  • Generare de teste pentru module întregi
  • Refactorizări mari care nu blochează flow-ul tău curent

Worktrees — Izolarea Modificărilor

Când un agent trebuie să modifice fișiere, poate lucra într-un worktree izolat — o copie separată a repo-ului pe un branch dedicat. Modificările agentului nu ating branch-ul tău principal până când nu le accepți explicit.

> Pornește un agent cu worktree izolat care să refactorizeze
   modulul de autentificare să folosească hooks în loc de HOCs.
   Vreau să văd diff-ul complet înainte să accept ceva.

Worktree-ul permite:

  • Agenți care modifică fișiere fără niciun risc pentru branch-ul tău activ
  • Review complet al modificărilor agentului înainte de merge
  • Abandon complet dacă rezultatul nu e satisfăcător — șterg worktree-ul, zero urmări

Notă: Worktree-ul e curățat automat dacă agentul nu face nicio modificare. Dacă agentul modifică fișiere, primești calea și branch-ul cu modificările pentru review.

Agenți în paralel

Pentru task-uri independente, poți porni mai mulți agenți simultan și să economisești timp real:

> Am nevoie de două analize în paralel:
   1. Un agent care să auditeze performanța query-urilor din src/db/
   2. Un agent care să scrie documentație pentru endpoint-urile din src/api/
   Pornește ambii agenți simultan și spune-mi când ambii termină.

Claude pornește ambii agenți simultan, lucrează în paralel și te notifică când ambii termină. În loc de 2× timpul unui agent, obții rezultatele în aproximativ timpul unui singur agent.

Când nu folosi paralel: Dacă task-ul B depinde de output-ul task-ului A, trebuie să le faci secvențial. Paralelismul e util doar pentru task-uri cu adevărat independente.

Agenți personalizați prin Skills

Poți crea propriii agenți specializați combinator skills cu instrucțiuni de agent. Un skill poate descrie comportamentul unui subagent specializat:

# .claude/commands/db-review.md

Pornește un agent specializat pentru audit de bază de date.

Agentul trebuie să:
1. Caute toate fișierele de migrare din database/migrations/
2. Analizeze fiecare migrare pentru:
   - Migrări fără metodă rollback (down())
   - Modificări de coloană care pot cauza pierdere de date
   - Foreign keys fără index
3. Returneze un raport structurat cu severitate (critical / warning / info)

Abordare read-only — nu modifica niciun fișier.

Invoci cu /db-review și obții un agent specializat care analizează baza de date cu criterii precise, fără să îi mai explici de fiecare dată ce să caute.

Când să folosești Agent vs. Sesiune normală

Regula simplă pentru a alege:

  • Sesiune normală — Task-uri iterative unde vrei să controlezi fiecare pas: implementare cu review continuu, debugging, refactorizare mică unde vrei să aprobi fiecare modificare.
  • Agent simplu — Task-uri cu instrucțiuni clare și output definit care nu necesită intervenție în mijlocul execuției: research, analiză, generare boilerplate, scriere de teste.
  • Agenți în paralel — Când ai 2+ task-uri independente și vrei să le faci simultan.
  • Agent cu worktree — Când agentul trebuie să modifice fișiere, dar vrei să revizuiești complet înainte să accepți modificările.

Exemplu complet de flux cu agenți

Un flux real pentru adăugarea unui feature nou:

# Pasul 1 — Research paralel (2 agenți simultan)
> Pornește doi agenți în paralel:
  1. Explore: găsește toate locurile unde e gestionată autentificarea
  2. Explore: găsește toate testele existente pentru auth

# Pasul 2 — Plan (după ce primești rezultatele)
> Pe baza rezultatelor de mai sus, folosește un agent Plan
  pentru a proiecta implementarea 2FA prin TOTP.

# Pasul 3 — Implementare cu worktree (tu controlezi merge-ul)
> Pornește un agent cu worktree izolat care să implementeze
  planul de mai sus. Vreau diff complet înainte să accept.

# Pasul 4 — Review independent
> Pornește un agent code-reviewer care să analizeze
  diff-ul din worktree și să identifice probleme.

Rezultat: research, design, implementare și review — toate cu contexte proaspete, izolate, fără ca sesiunea principală să se încarce cu zeci de fișiere citite.
