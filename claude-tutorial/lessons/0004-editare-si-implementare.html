Editare și Implementare Features

Cum Claude Code modifică fișierele tale, ce poți controla și cum să revizuiești modificările înainte să le accepți.

Cum funcționează editarea

Când îi ceri lui Claude Code să modifice cod, el nu face asta orb. Procesul e:

  1. Citește fișierele relevante pentru a înțelege contextul existent
  2. Planifică modificările necesare
  3. Îți prezintă diff-ul propus și cere aprobare (implicit)
  4. Aplică modificările în fișierele reale de pe disk

Nu e magic: Claude Code modifică fișierele reale ale proiectului tău. Git e salvatorul tău — lucrează mereu pe un branch curat și fă commit frecvent.

Implementarea unui feature — exemplu complet

Să zicem că vrei să adaugi validare pentru un câmp de email:

> În fișierul src/validators/user.js, adaugă o funcție validateEmail
  care să verifice formatul adresei de email folosind un regex standard.
  Funcția să returneze { valid: true } sau { valid: false, error: "mesaj" }

Claude Code va:

  • Citi src/validators/user.js pentru a vedea stilul existent
  • Genera funcția în stilul fișierului (nu injecta un stil diferit)
  • Arăta diff-ul înainte să modifice

Dacă nu ești mulțumit de implementare, spune:

> Folosește în schimb librăria validator.js în loc de regex manual

Implementare pe mai multe fișiere

Cele mai utile features implică mai multe fișiere. Claude Code gestionează asta bine:

> Adaugă un endpoint POST /api/users/reset-password care să:
  1. Accepte email-ul în body
  2. Genereze un token temporar
  3. Salveze token-ul în baza de date
  4. Trimită un email cu link-ul de reset
  Urmează structura existentă din /api/users/login

Claude Code va citi endpoint-ul existent ca referință, va crea sau modifica controller-ul, route-ul, service-ul și testul corespunzător.

Tip esențial: Arată-i un exemplu existent ("urmează structura din X"). Claude Code reproduce stilul proiectului tău — nu inventează convenții noi.

Permission model — ce trebuie să aprobi

Claude Code clasifică acțiunile în două categorii:

  • Citire (automată, fără aprobare) — citit fișiere, listare directoare, git log, git diff
  • Scriere / Execuție (cu aprobare) — modificat fișiere, rulat comenzi, git push, ștergere

Când Claude Code vrea să facă o acțiune ce necesită aprobare, îți va cere explicit. Vei vedea ce vrea să facă și poți aproba sau refuza.

Atenție: Dacă operezi în auto-approve mode (flag --dangerously-skip-permissions), Claude Code va executa totul fără să te întrebe. Folosește doar în medii izolate sau scripturi bine testate.

Iterarea pe implementare

Implementarea rareori e perfectă din prima. Iată ciclul de lucru natural:

> Implementează funcția de paginare pentru lista de produse

# Claude implementează

> Testele de integrare eșuează, uite eroarea: [paste error]

# Claude analizează și corectează

> Acum adaugă și sortarea după preț

# Claude extinde implementarea existentă

Sesiunea interactivă păstrează contextul între mesaje — nu trebuie să re-explici ce ai implementat anterior în aceeași sesiune.
