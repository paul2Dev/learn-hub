Claude Skills — Slash Commands Personalizate

Automatizează fluxurile repetitive cu comenzi proprii — de la /code-review la /deploy, orice workflow devine o singură comandă.

Ce sunt Skills?

Skills sunt slash commands personalizate pe care tu le definești. Un skill e un fișier Markdown cu instrucțiuni pentru Claude, salvat în .claude/commands/ al proiectului sau în ~/.claude/commands/ pentru a fi disponibil global.

Când tastezi /code-review în Claude Code, de fapt rulezi un skill — un fișier Markdown cu instrucțiuni detaliate despre cum să facă code review-ul. Poți crea propriile tale slash commands exact la fel.

Structura: Fișierul skill = instrucțiunile pe care Claude le primește când invoci comanda. Ce scrii în fișier, aia face Claude.

De ce să folosești Skills?

Trei probleme reale pe care le rezolvă skills:

  • Repetitivitate — Dacă trimiți același prompt de 10 ori pe zi ("fă code review cu accent pe securitate și performanță"), transformă-l într-un skill. O dată definit, /review face tot ce ai scris.
  • Standardizare în echipă — Skills salvate în .claude/commands/ și commituite în git sunt disponibile tuturor din echipă. Toți fac code review la fel, cu aceleași criterii.
  • Instrucțiuni complexe, invocație simplă — Un skill poate conține 50 de linii de instrucțiuni, exemple de format, contexte de cod. Tu tastezi /deploy și obții tot comportamentul complex.

Cum creezi un Skill

Creează directorul și un fișier .md — numele fișierului devine numele comenzii:

mkdir -p .claude/commands
# Fișierul acesta se invocă cu /code-review
touch .claude/commands/code-review.md

Conținutul fișierului e exact prompt-ul pe care Claude îl primește:

# Code Review

Fă un code review al modificărilor din branch-ul curent față de main.

Structurează răspunsul în trei secțiuni:
1. **Probleme critice** — bug-uri, vulnerabilități de securitate, logică greșită
2. **Îmbunătățiri** — performanță, lizibilitate, refolosire cod
3. **Ce e bine** — pattern-uri corecte, cod curat (minim 2-3 observații pozitive)

Dacă nu există probleme critice, spune explicit asta.
Nu comenta stilul de formatare dacă există deja un linter configurat.

Acum invoci din orice sesiune Claude Code:

> /code-review

Argumente dinamice cu $ARGUMENTS

Skills pot accepta argumente dinamice cu placeholder-ul $ARGUMENTS:

# .claude/commands/new-feature.md

Creează un feature nou pentru: $ARGUMENTS

Pași:
1. Analizează codul existent pentru a înțelege pattern-urile folosite
2. Implementează feature-ul urmând convențiile din proiect
3. Adaugă teste unitare
4. Actualizează documentația relevantă dacă e necesar

Invocarea cu argument:

> /new-feature sistem de autentificare cu JWT

$ARGUMENTS devine "sistem de autentificare cu JWT". Claude primește instrucțiunile tale complete plus argumentul tău.

Pattern util: Skills fără $ARGUMENTS pentru fluxuri fixe (code review, standup). Skills cu $ARGUMENTS pentru fluxuri parametrizabile (new-feature, debug-issue, explain-module).

Skills de proiect vs. Skills globale

Skills pot fi stocate în două locuri, cu scopuri diferite:

  • De proiect — .claude/commands/ în rădăcina proiectului. Disponibile doar în acel proiect. Commituite în git → accesibile tuturor din echipă automat.
  • Globale — ~/.claude/commands/ în home directory. Disponibile în orice proiect, pe mașina ta. Invizibile pentru restul echipei.

Strategie recomandată: Skills legate de proiect (deploy, convenții specifice stack-ului, checklist de PR) → de proiect. Skills legate de tine ca programator (stilul tău de code review, template-ul tău de commit message) → globale.

Exemple concrete de Skills utile

1. Daily standup (.claude/commands/standup.md):

Uită-te la commit-urile mele din ultimele 24 de ore.
Generează un rezumat pentru standup-ul zilnic în format:
- Ce am terminat ieri
- Ce fac azi (bazat pe branch-ul curent și TODO-urile deschise)
- Blocanți (dacă există comentarii cu "blocked" sau "waiting")

2. Pregătire PR (.claude/commands/prep-pr.md):

Pregătește PR-ul pentru branch-ul curent:
1. Verifică că nu există console.log sau debugger lăsat în cod
2. Verifică că testele trec
3. Generează descrierea PR: Summary, Changes Made, How to Test
4. Propune un titlu scurt, clar, în format Conventional Commits

3. Audit securitate rapid (.claude/commands/security-check.md):

Auditează codul modificat în sesiunea curentă pentru:
- SQL injection și XSS
- Date sensibile expuse în logs sau răspunsuri API
- Validare input lipsă la granițele sistemului (input din request)
- Dependențe cu vulnerabilități cunoscute (verifică package.json dacă există)

4. Explain pentru onboarding (.claude/commands/explain.md):

Explică modulul sau fișierul: $ARGUMENTS

Structurează explicația pentru un programator nou în proiect:
1. Ce problemă rezolvă
2. Cum e structurat (fișiere cheie și rolul lor)
3. Cum se integrează cu restul sistemului
4. Ce trebuie să știe înainte să modifice ceva

Skills cu resurse atașate

Un skill poate include fișiere din proiect ca referință, folosind sintaxa de includere Markdown. Util pentru skills care trebuie să respecte un standard specific:

# .claude/commands/migration.md

Creează o nouă migrare de bază de date pentru: $ARGUMENTS

Urmează exact structura din fișierele existente de migrare.
Convenții obligatorii:
- Fiecare migrare are up() și down()
- Comentariu în header cu autorul și data
- Testează că down() revine exact la starea inițială
