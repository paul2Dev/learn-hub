Debugging cu Claude Code

Stack traces, teste care eșuează și erori de neînțeles — cum le rezolvi mai rapid cu Claude Code.

Pattern de bază: paste eroarea, cere analiza

Cel mai simplu și eficient pattern de debugging:

> Primesc această eroare când rulez aplicația. Ajută-mă să înțeleg cauza
  și cum o rezolv:

  TypeError: Cannot read properties of undefined (reading 'id')
      at getUserProfile (src/services/user.js:47:23)
      at async handleRequest (src/controllers/auth.js:112:18)
      at Layer.handle [as handle_request] (node_modules/express/...)

Claude Code va:

  • Citi src/services/user.js la linia 47
  • Citi src/controllers/auth.js la linia 112
  • Urma lanțul de apeluri pentru a găsi rădăcina problemei
  • Propune fix-ul cu context

Teste care eșuează

Acesta e unul din cele mai puternice use case-uri. Nu mai citi manual output-ul testelor:

> Rulează testele și repară ce eșuează

Claude Code va rula comanda de teste, va citi output-ul, va identifica fișierele cu probleme și va propune fix-uri. Poți itera:

> Testele de autentificare încă eșuează, uite output-ul:
  [paste output]

> Acum rulează din nou doar testele de auth: npm test -- --grep "auth"

Sfat: Dacă ai o comandă specifică de test, pune-o în CLAUDE.md. Astfel Claude Code știe exact cum să ruleze testele proiectului tău fără să îi explici de fiecare dată.

Bug de logică — când codul "funcționează" dar dă rezultate greșite

Erorile de logică sunt mai greu de prins. Descrie comportamentul așteptat vs. cel real:

> Funcția `calculateDiscount` din src/pricing.js returnează discount greșit
  pentru comenzi peste 500 RON. Ar trebui să aplice 15% dar aplică 10%.
  Uite câteva exemple:
  - order: 600 RON → discount returnat: 60 RON (greșit, ar trebui 90)
  - order: 1000 RON → discount returnat: 100 RON (greșit, ar trebui 150)

Cu exemple concrete (input → output așteptat vs. real), Claude Code poate identifica mult mai rapid unde e logica defectă.

Debugging interactiv — când nu știi de unde să începi

Uneori nu ai un stack trace clar. Descrie simptomul:

> Aplicația devine lentă după câteva ore de funcționare și trebuie restartată.
  Nu am erori în log-uri. Unde aș putea căuta cauzele?

Claude Code va analiza codul și va sugera locuri unde s-ar putea produce memory leaks, conexiuni nedeschise sau query-uri costisitoare — și îți va propune ce să verifici și în ce ordine.

Adăugarea de logging temporar

Când ai nevoie să înțelegi fluxul de execuție:

> Adaugă console.log-uri temporare în funcția `processOrder` pentru a
  vedea valoarea variabilelor la fiecare pas. Marchează-le cu // DEBUG
  ca să le găsesc ușor ulterior.

Și după ce ai înțeles problema:

> Acum șterge toate comentariile // DEBUG din proiect

Compararea cu o versiune anterioară

Dacă bug-ul a apărut recent:

> Funcționalitatea X funcționa săptămâna trecută. Ce s-a schimbat în
  ultimele 5 commit-uri în fișierele legate de modulul de plăți?

Claude Code va rula git log și git diff pentru a identifica modificările relevante.
