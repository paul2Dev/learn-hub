Mental Model — Cum Gândește Claude Code

Dacă îl tratezi ca pe un chatbot, îl vei folosi la 20% din capacitate. Iată cum să gândești cu el.

Nu e un chatbot. E un agent.

Un chatbot primește text și returnează text. Claude Code primește o sarcină și o execută: citește fișiere reale, scrie cod real, rulează comenzi reale în terminalul tău.

Gândește-l ca pe un coleg programator senior care are acces deplin la mașina ta și poate:

  • Citi orice fișier din proiectul tău
  • Scrie și modifica fișiere
  • Rula comenzi în terminal (npm test, git log, etc.)
  • Interacționa cu git
  • Căuta în codebase cu grep/glob

Schimbarea de paradigmă: Nu îi explica cum să facă — spune-i ce vrei să obții. El va găsi calea.

Context Window — Memoria lui Claude Code

Claude Code are o fereastră de context — o cantitate maximă de informație pe care o poate "ține minte" într-o sesiune. Această fereastră se umple cu:

  • Conversația voastră (mesajele tale + răspunsurile lui)
  • Fișierele pe care le-a citit
  • Output-ul comenzilor rulate

Când contextul se umple, Claude Code îl comprimă automat (/compact). Implicație practică: sesiunile lungi devin mai puțin precise. Soluția? Sesiuni scurte, focusate pe o singură sarcină.

Regula de aur: O sesiune = o sarcină. Termină, fă commit, pornește o sesiune nouă pentru sarcina următoare.

Modul interactiv vs. one-shot

Claude Code are două moduri de utilizare:

  • Interactiv (claude) — o sesiune continuă cu mai multe mesaje. Ideal pentru sarcini complexe care necesită iterații: "implementează X", "acum testează", "acum refactorizează".
  • One-shot (claude "prompt") — un singur schimb, fără sesiune. Ideal pentru întrebări rapide sau automatizări.
# Interactiv
claude

# One-shot
claude "Câte linii de cod are src/?"

Când Claude Code strălucește

  • Explorarea unui codebase necunoscut ("Ce face această funcție?")
  • Implementarea unui feature cu mai multe fișiere implicate
  • Debugging pe un stack trace de neînțeles
  • Refactorizare cu impact larg în proiect
  • Automatizarea fluxului git (commit, PR, review)

Când NU e potrivit

  • Sarcini care necesită browsing web sau date în timp real
  • Generarea de conținut creativ neconectat la codul tău
  • Întrebări rapide fără legătură cu proiectul — pentru asta e mai rapid un chatbot simplu

Permisiunile — Ce poate face fără să te întrebe

Claude Code funcționează cu un sistem de permisiuni. Implicit, te va întreba înainte de acțiuni potențial distructive (ștergere fișiere, push la remote, etc.). Acțiunile de citire sunt permise automat.

Poți vedea și ajusta permisiunile cu comanda /settings din sesiunea interactivă. Mai multe detalii în Lecția 8.
